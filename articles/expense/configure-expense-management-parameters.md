# <a name="configure-expense-management-parameters"></a>Настройка параметров управления расходами

_**Относится к:** Project Operations для сценариев на основе ресурсов/без запасов_

В этой теме описываются параметры, которые управляют общим поведением в модуле управления расходами.

## <a name="general"></a>Общие

| Поле                                                    | Описание: |
|----------------------------------------------------------|-------------|
| Стандартная норма проездных денег                                 | Введите ставку возмещения расходов за пробег. Эта ставка умножается на пробег, указанный для расхода, чтобы рассчитать сумму, которая возмещается для путевых расходов. |
| Проверить цель расходов                                 | Включите этот параметр, чтобы ограничить пользователей существующим набором значений, настроенным в поле **Назначение отчета по расходам** при создании отчетов о расходах. |
| Личные расходы, оплачиваемые                                | Выберите, кто несет ответственность за оплату любых сумм транзакций по кредитной карте, которые относятся к категории личных. |
| Отобразить полный отчет об издержках с детализацией               | Выберите этот параметр, чтобы отображать все расходы в отчете о расходах при просмотре деталей исходного документа для определенного ваучера, который был создан при разноске отчета о расходах. |
| Предварительная авторизация командировки обязательна                 | Выберите этот параметр, чтобы потребовать, чтобы заявка на командировку была отправлена и утверждена, прежде чем сотрудник сможет отправить отчет о расходах. |
| Разрешить редактирование распределений перед разноской            | Выберите, можно ли редактировать распределения отчета о расходах до его разноски. |
| Оценить политики управления расходами                     | Выберите, когда оцениваются расходы, чтобы определить, была ли нарушена политика расходов. Расходы можно оценить на предмет нарушений при вводе и сохранении записи расхода или при отправке расхода на утверждение. Правила политики для требуемых квитанций будут оценены при сохранении строки расходов. |
| Разрешать строки внутрихолдинговых расходов                         | Выберите, можно ли вводить расходы для других юридических лиц в отчете о расходах. |
| Разрешить изменение валютного курса для расходов по кредитной карте | Выберите этот параметр, чтобы разрешить пользователю редактировать обменный курс для импортированных расходов по кредитной карте. |
| Поля многоуровневой иерархии для отображения                  | Выберите, какие поля утверждающего будут отображаться, если тип назначения рабочего процесса установлен как иерархия, а выбор **Иерархия** настроен на использование иерархии утверждения, многоуровневого утверждения расходов. Когда для рабочего процесса используется многоуровневая иерархия утверждения, выбранные поля будут отображаться в отчете о расходах и их можно будет редактировать. |
| Введите номер кредитной карты сотрудника                        | Выберите, можно ли ввести 15-значный или 16-значный номер и сохранить его в поле **Код карты** на странице **Кредитные карты** для сотрудника. |
| Проверить назначение заявки на командировку                      | Включите этот параметр, чтобы ограничить пользователей существующим набором значений, настроенным в поле **Назначение отчета по расходам** при создании заявки на командировку. |

## <a name="financial"></a>Финансовая деятельность

| Поле                                                                                    | Описание: |
|------------------------------------------------------------------------------------------|-------------|
| Наименование ежедневного журнала ГК                                                                | Выберите имя журнала книги учета, в который разносятся утвержденные отчеты о расходах. |
| Включить возмещение налогов из расходов                                                        | Выберите этот параметр, чтобы включить возмещение налога на расходы для соответствующих расходов. Этот параметр нельзя выбрать, если включены правила налога с продаж и налога на использование в США. |
| Разнести денежные авансы немедленно                                                           | Выберите этот вариант, чтобы разнести утвержденный денежный аванс после завершения процесса оплаты и перевода. Если этот параметр не выбран, в процессе "Оплата и перевод" создается неразнесенный общий журнал. |
| Исправлять дату учета во время разноски                                                   | Выберите этот параметр, чтобы обновлять дату учета при разноске расходов, если текущий период приостановлен или закрыт. Дата учета будет установлена на следующий открытый период. |
| Разрешить группировку проводок на основании корр. счета, указанного в способе оплаты       | Выберите этот параметр, чтобы суммировать проводки по расходам для отчета о расходах на основе корреспондирующего счета, указанного в способе оплаты для расходов. |
| Налог включен                                                                             | Выберите этот параметр, чтобы по умолчанию включить налог с продаж в сумму расходов. |
| Выпуск бюджетных обязательств при закрытии заявок на командировку                                      | Выберите этот параметр, чтобы разблокировать обремененные средства при закрытии утвержденной заявки на командировку. |
| Разрешить отправку заявок на командировку при перерасходе бюджета для регистра бюджета и бюджета проекта | Выберите этот параметр, чтобы разрешить сотрудникам отправлять заявки на командировки на утверждение, даже если они превышают разрешенный бюджет, установленный в реестре бюджета, или разрешенный бюджет для проекта. |
| Разрешить отправку отчета о расходах при перерасходе бюджета для регистра бюджета и бюджета проекта     | Выберите этот параметр, чтобы разрешить сотрудникам отправлять отчеты о расходах на утверждение, даже если они превышают разрешенный бюджет, установленный в реестре бюджета, или разрешенный бюджет для проекта. |
| Разрешить утверждение отчета о расходах при перерасходе бюджета только для регистра бюджета                 | Выберите этот параметр, чтобы разрешить утверждающим утверждать отчеты о расходах, превышающие разрешенный бюджет, установленный в реестре бюджета. |

## <a name="per-diem"></a>В день

| Поле                                 | Описание: |
|---------------------------------------|-------------|
| Минимальное количество часов для суточных            | Введите минимальное количество часов по умолчанию, которое сотрудник должен отработать в день, чтобы иметь право на получение суточных для покрытия командировочных расходов. Это значение используется как значение по умолчанию только для поля **Минимальное количество часов** для уровней ставки суточных. |
| Процент на питание                          | Введите процент суточных для питания по умолчанию, который используется в первый и последний дни командировочных расходов, когда в поле **Рассчитать снижение расходов на питание по** установлено значение **Тип питания за день** или **Число приемов пищи в день**. Рабочий день в первый и последний дни может быть короче стандартного рабочего дня. Поэтому размер суточных в эти дни может отличаться от стандартного. Если процент установлен на **0** (ноль), вычеты за первый и последний дни будут равны 0,00. |
| Процент на гостиницу                         | Введите процент суточных для отелей по умолчанию, который используется в первый и последний дни командировочных расходов. Рабочий день в первый и последний дни может быть короче стандартного рабочего дня. Поэтому размер суточных в эти дни может отличаться от стандартного. Если процент установлен на **0** (ноль), вычеты за первый и последний дни будут равны 0,00. |
| Др. проценты                         | Введите процент суточных по умолчанию для прочих расходов, который используется в первый и последний дни командировочных расходов. Рабочий день в первый и последний дни может быть короче стандартного рабочего дня. Поэтому размер суточных в эти дни может отличаться от стандартного. Если процент установлен на **0** (ноль), вычеты за первый и последний дни будут равны 0,00. |
| Сокращение в процентах для завтрака | Введите сумму, на которую уменьшаются суточные за завтрак. Например, если служащий получает бесплатный завтрак, вы можете уменьшить суточные на 10 процентов. |
| Сокращение в процентах для обеда     | Введите сумму, на которую уменьшаются суточные за обед. Например, если служащий получает бесплатный обед, вы можете уменьшить суточные на 15 процентов. |
| Сокращение в процентах для ужина    | Введите сумму, на которую уменьшаются суточные за ужин. Например, если служащий получает бесплатный ужин, вы можете уменьшить суточные на 25 процентов. |
| Рассчитать снижение расходов на питание по           | Укажите, как система должна рассчитывать сокращение суточных на питание, выбрав, основано ли сокращение на типе питания за поездку или в день, или на основе количества приемов пищи, разрешенных в день. |
| Округление суточных                     | Выберите тип округления, который используется для суточных. Если вы выберете обычное округление, любые суточные расходы, которые имеют размер 0,50, округляются до 1,00, а любые суточные расходы, размер которых меньше 0,50, округляется до 0,00. |
| Основывать расчет суточных на          | Выберите, основывается ли сумма суточных на календарном дне или 24-часовом периоде. |

## <a name="fax-cover-pages"></a>Титульные страницы факса

| Поле                          | Описание: |
|--------------------------------|-------------|
| Инструкции                   | Введите инструкции, которым должны следовать сотрудники при создании титульного листа для факса, который используется для отправки квитанций, связанных с отчетом о расходах. Чтобы ввести текст для конкретного языка, который будет отображаться в зависимости от языка пользователя, выберите **Переводы**. |
| ИД пользователя (информация о штрих-коде) | Выберите этот параметр, чтобы сохранить уникальный идентификатор сотрудника в штрих-коде, который используется на титульной странице факса. |
| Штрих-код                       | Выберите штрих-код, который используется на титульной странице факса. Штрих-коды 39 и 128 поддерживаются в управлении расходами. |

## <a name="anti-corruption"></a>Противодействие коррупции

| Поле                                 | Описание: |
|---------------------------------------|-------------|
| Показывать аттестацию по противодействию коррупции   | Выберите этот параметр, чтобы отображать антикоррупционный текст при создании отчета о расходах. Затем можно включить определенные категории расходов, которые потребуют выбора антикоррупционного подтверждения в отчете о расходах. Например, категория подарков, связанная с государственными официальными расходами, может требовать от сотрудника подтверждения того, что расходы соответствуют политике компании, связанной с государственными служащими. |
| Сообщение по противодействию коррупции для отправителя | Введите текст, который должен быть показан сотруднику, создающему отчет о расходах. Чтобы ввести текст для конкретного языка, который будет отображаться в зависимости от языка пользователя, выберите **Переводы**. |
| Сообщение по противодействию коррупции для утверждающего  | Введите текст, который должен быть показан утверждающему при создании отчета о расходах. Чтобы ввести текст для конкретного языка, который будет отображаться в зависимости от языка пользователя, выберите **Переводы**. |
