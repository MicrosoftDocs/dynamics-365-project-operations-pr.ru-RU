# <a name="mobile-expense-app"></a>Мобильное приложение для учета расходов

_**Относится к:** Project Operations для сценариев на основе ресурсов/нескладируемых запасов, упрощенное развертывание — от сделки до выставления счетов-фактур_

В этой статье приводятся сведения о мобильной рабочей области **Управление расходами**. Эта рабочая область позволяет пользователям записывать и отправлять квитанцию, чтобы потом прикрепить ее к отчету о расходах. Пользователи также могут быстро создать строку расходов с помощью прикрепленной квитанции, а также создавать свои отчеты о расходах и управлять ими. Кроме того, утверждающие могут использовать мобильную рабочую область **Управление расходами** для просмотра назначенных им отчетов о расходах и утверждения или отклонения этих отчетов о расходах.

Это мобильное рабочее пространство предназначено для использования вместе с мобильным приложением Dynamics 365 Unified Ops.

Многие организации требуют, чтобы копия квитанции была приложена к отчету о расходах, связанных с поездкой или бизнесом, который сотрудник отправляет на возмещение. Мобильная рабочая область **Управление расходами** позволяет пользователям быстро создавать новые строки расходов на мобильном устройстве по своему выбору, используя прикрепленную фотографию квитанции. Кроме того, пользователи могут сделать снимок квитанции, а затем прикрепить ее к отчету о расходах позже. Сотрудники также могут создавать отчеты о расходах и управлять ими, а затем отправлять их на утверждение и возмещение с помощью своего мобильного устройства.

В частности, мобильная рабочая область **Управление расходами** позволяет пользователям выполнять следующие задачи:

- Сфотографируйте квитанцию. Отправьте фотографию квитанции и приложите ее к отчету о расходах позже.
- Отправьте файл как снятую квитанцию. Позже вы можете прикрепить этот файл к отчету о расходах.
- Создайте новую строку расходов, используя прикрепленную квитанцию. Позже вы можете добавить эту строку в отчет о расходах и отправить ее на утверждение и возмещение.

Можно также использовать следующие функции:

- Создайте новый отчет о расходах.
- Прикрепите к отчету о расходах транзакции по кредитным картам и другие ранее созданные расходы.
- Создайте новые расходы для отчета о расходах.
- Прикрепите квитанцию к любым расходам для отчета о расходах, сфотографировав квитанцию или отправив файл в качестве полученной квитанции.
- В зависимости от политики компании в отношении расходов добавьте список гостей к расходам.
- Детализируйте расходы в зависимости от политики компании в отношении расходов.
- Отправьте отчет о расходах на утверждение и возмещение.
- Утверждайте или отклоняйте отчеты о расходах, для которых вы назначены в качестве утверждающего.

4. При первом входе в систему вам будет предложено ввести имя пользователя и пароль. Введите свои учетные данные.

## <a name="capture-a-receipt-by-using-the-expense-management-mobile-workspace"></a>Получение чека с помощью мобильной рабочей области "Управление расходами"

1. На мобильном устройстве откройте рабочую область **Управление расходами**.
2. Выберите **Получение квитанции**.
3. Выберите **Сфотографировать** или **Выбрать изображение**.
4. Выполните одно из перечисленных ниже действий.

    - Если вы выбрали пункт **Сфотографировать**, выполните следующие действия:

        1. Вы будете переключены на камеру вашего мобильного устройства, чтобы вы могли сфотографировать чек. 
        2. Когда вы закончите делать фотографию, выберите **ОК**, чтобы принять фотографию.
        3. Необязательно: введите имя фотографии и введите заметки.

    - Если вы выбрали пункт **Выбрать изображение**, выполните следующие действия:

        1. Выберите изображение в списке.
        2. Необязательно: введите имя изображения и введите заметки.

5. Нажмите кнопку **Готово**.

## <a name="quickly-enter-expenses-by-using-the-expense-management-mobile-workspace"></a>Быстрый ввод расходов с помощью мобильной рабочей области "Управление расходами"

1. На мобильном устройстве откройте рабочую область **Управление расходами**.
2. Выберите **Быстрый ввод расходов**.
3. Выберите категорию расхода. Вы увидите список категорий расходов, которые загружены в ваше приложение для автономного использования. По умолчанию загружено 50 элементов, но разработчик может изменить это число. Для получения дополнительной информации разработчики должны обратиться к разделу [Мобильная платформа](/dynamics365/fin-ops-core/dev-itpro/mobile-apps/platform/mobile-platform-getting-started). Если вашей категории нет в списке, выберите **Поиск** для выполнения сетевого поиска. Выполните поиск по категории расходов или переключитесь на поиск по типу расходов.
4. Введите дату транзакции расхода.
5. Необязательно: введите продавца для расхода.
6. Введите сумму расхода.
7. Выберите валюту расхода. Вы увидите список кодов валют, которые загружены в ваше приложение для автономного использования. По умолчанию загружено 400 валют, но разработчик может изменить это число. Для получения дополнительной информации разработчики должны обратиться к разделу [Мобильная платформа](/dynamics365/fin-ops-core/dev-itpro/mobile-apps/platform/mobile-platform-getting-started). Если вашей валюты нет в списке, выберите **Поиск** для выполнения сетевого поиска. Выполните поиск по валюте или переключитесь на поиск по названию.
8. Выберите **Сфотографировать** или **Выбрать изображение**.
9. Выполните одно из перечисленных ниже действий.

    - Если выбран пункт **Сфотографировать**, вы будете переключены на камеру вашего мобильного устройства, чтобы вы могли сфотографировать чек. Когда вы закончите делать фотографию, выберите **ОК**, чтобы принять фотографию.
    - Если вы выбрали **Выбрать изображение**, выберите изображение в списке.

10. Нажмите кнопку **Готово**.

## <a name="approve-an-expense-report-by-using-the-expense-management-mobile-workspace"></a>Утверждение отчета о расходах с помощью мобильной рабочей области "Управление расходами"

1. На мобильном устройстве откройте рабочую область **Управление расходами**.
2. **Утверждения расходов** показывают количество отчетов о расходах, которые назначены вам на утверждение. Количество обновляется примерно каждые 30 минут. Выберите **Утверждения расходов**.

    Отображается список отчетов о расходах, которые назначены вам на утверждение.

3. Выберите отчет о расходах, чтобы просмотреть подробную информацию о расходах.
4. Выберите расход, чтобы просмотреть подробные сведения для него. Информация, отображаемая для расходов, включает все детали квитанции, гостя и детализации.
5. Вернувшись на страницу **Отчет о затратах**, выберите утверждение или отклонение отчета о расходах.
6. Введите любые комментарии к действию утверждения.
7. Нажмите кнопку **Готово**.

## <a name="create-a-new-expense-report-and-submit-it-for-approval-by-using-the-expense-management-mobile-workspace"></a>Создание нового отчета по расходам и отправка его на утверждение с помощью мобильной рабочей области "Управление расходами"

1. На мобильном устройстве откройте рабочую область **Управление расходами**.
2. Выберите **Ввод расходов**.
3. Выберите **Создать отчет** или выберите существующий отчет о расходах в списке.
4. Для новых отчетов о расходах введите цель и любую доступную дополнительную информацию. Эта информация различается в зависимости от того, как настроено управление расходами в вашей компании.
5. Нажмите кнопку **Готово**.
6. Чтобы добавить в отчет о расходах существующие расходы, например операции по кредитной карте, выберите **Вложить**.
7. Выберите один или несколько расходов в списке.
8. Нажмите кнопку **Готово**.
9. Чтобы добавить новый расход в отчет о расходах, выберите **Создать расход**.
10. Выберите категорию для расхода. Вы увидите список категорий расходов, которые загружены в ваше приложение для автономного использования. По умолчанию загружено 50 элементов, но разработчик может изменить это число. Для получения дополнительной информации разработчики должны обратиться к разделу [Мобильная платформа](/dynamics365/fin-ops-core/dev-itpro/mobile-apps/platform/mobile-platform-getting-started). Если вашей категории нет в списке, выберите **Поиск** для выполнения сетевого поиска. Выполните поиск по категории расходов или переключитесь на поиск по типу расходов.
11. Необязательно: введите продавца для расхода.
12. Введите дату транзакции расхода.
13. Введите сумму расхода.
14. Выберите валюту расхода. Вы увидите список кодов валют, которые загружены в ваше приложение для автономного использования. По умолчанию загружено 400 валют, но разработчик может изменить это число. Для получения дополнительной информации разработчики должны обратиться к разделу [Мобильная платформа](/dynamics365/fin-ops-core/dev-itpro/mobile-apps/platform/mobile-platform-getting-started). Если вашей валюты нет в списке, выберите **Поиск** для выполнения сетевого поиска. Выполните поиск по валюте или переключитесь на поиск по названию.
15. Нажмите кнопку **Готово**.
16. Чтобы добавить дополнительные сведения к расходу, выберите **Добавить дополнительные сведения**. Доступные поля зависят от конфигурации управления расходами в вашей компании.
17. Если в соответствии с политикой компании требуется квитанция о расходах, выберите **Квитанции**, затем выполните следующие действия:

    1. Выберите **Получить квитанцию** или **Прикрепить квитанцию**.
    2. Выполните одно из перечисленных ниже действий.

        - Если вы выбрали пункт **Получить квитанцию**, выполните следующие действия:

            1. Выберите **Сфотографировать** или **Выбрать изображение**.
            2. Выполните одно из перечисленных ниже действий.

                - Если вы выбрали пункт **Сфотографировать**, выполните следующие действия:

                    1. Вы будете переключены на камеру вашего мобильного устройства, чтобы вы могли сфотографировать чек. Когда вы закончите делать фотографию, выберите **ОК**, чтобы принять фотографию.
                    2. Необязательно: введите имя фотографии и введите заметки.

                - Если вы выбрали пункт **Выбрать изображение**, выполните следующие действия:

                    1. Выберите изображение в списке.
                    2. Необязательно: введите имя изображения и введите заметки.

            3. Нажмите кнопку **Готово**.

        - Если вы выбрали пункт **Прикрепить квитанцию**, выполните следующие действия:

            1. Выберите в списке одно или несколько изображений.
            2. Нажмите кнопку **Готово**.

    3. Выберите кнопку **Назад**, чтобы вернуться к сведениям о расходах.

18. Если в соответствии с политикой компании требуются гости для расходов, выберите **Гости**, затем выполните следующие действия:

    1. Выберите **Гость**, **Предыдущие гости** или **Коллеги**.
    2. Выполните одно из перечисленных ниже действий.

        - Если вы выбрали пункт **Гости**, выполните следующие действия:

            1. Введите имя гостя.
            2. Необязательно: введите организацию и/или страну гостя.
            3. Необязательно: введите должность гостя.
            4. Нажмите кнопку **Готово**.

        - Если вы выбрали пункт **Предыдущие гости**, выполните следующие действия:

            1. Выберите в списке одного или несколько предыдущих гостей. Вы увидите список предыдущих гостей, которых вы добавили в предыдущие отчеты о расходах, которые загружены в ваше приложение для автономного использования. По умолчанию загружено 50 элементов, но разработчик может изменить это число. Для получения дополнительной информации разработчики должны обратиться к разделу [Мобильная платформа](/dynamics365/fin-ops-core/dev-itpro/mobile-apps/platform/mobile-platform-getting-started). Если вашего предыдущего гостя нет в списке, выберите **Поиск** для выполнения сетевого поиска. Выполните поиск по названию или переключитесь на поиск по организации, стране или должности.
            2. Нажмите кнопку **Готово**.

        - Если вы выбрали пункт **Коллеги**, выполните следующие действия:

            1. В списке выберите одного или нескольких коллег. Вы увидите список коллег, которые загружены в ваше приложение для автономного использования. По умолчанию загружено 50 элементов, но разработчик может изменить это число. Для получения дополнительной информации разработчики должны обратиться к разделу [Мобильная платформа](/dynamics365/fin-ops-core/dev-itpro/mobile-apps/platform/mobile-platform-getting-started). Если вашего коллеги нет в списке, выберите **Поиск** для выполнения сетевого поиска. Выполните поиск по названию или переключитесь на поиск по компании или должности.
            2. Нажмите кнопку **Готово**.

    3. Выберите кнопку **Назад**, чтобы вернуться к сведениям о расходах.

19. Если в соответствии с политикой компании требуется детализация расхода, выберите **Детализировать**, затем выполните следующие действия:

    1. Выберите первую дату для детализации.
    2. Выберите **Добавить детализацию**.
    3. Выберите подкатегорию для детализации расходов. Вы увидите список подкатегорий расходов, которые загружены в ваше приложение для автономного использования. По умолчанию загружено 50 элементов, но разработчик может изменить это число. Для получения дополнительной информации разработчики должны обратиться к разделу [Мобильная платформа](/dynamics365/fin-ops-core/dev-itpro/mobile-apps/platform/mobile-platform-getting-started). Если вашей подкатегории нет в списке, выберите **Поиск** для выполнения сетевого поиска. Выполните поиск по названию подкатегории расходов.
    4. Введите сумму транзакции для детализации.
    5. При необходимости измените дату транзакции.
    6. Нажмите кнопку **Готово**.
    7. Повторяйте предыдущие шаги, пока не закончите добавлять все элементы детализации на выбранную дату.
    8. Для дополнительных дней вы можете выбрать **Копировать на следующий день**, чтобы скопировать детализацию на следующий день. Кроме того, вы можете выбрать дату для детализации, а затем добавить элементы детализации, как вы это делали для первой даты.
    9. После того, как вы закончите детализацию расхода, выберите кнопку **Назад**, чтобы вернуться к сведениям о расходах.

20. Выберите кнопку **Назад**, чтобы вернуться на страницу **Отчет о расходах**.
21. Повторяйте предыдущие шаги, пока не закончите добавлять все расходы.
22. Выберите **Отправить**.
23. Введите любые комментарии для утверждающего.
24. Нажмите кнопку **Готово**.

## <a name="frequently-asked-questions"></a>Вопросы и ответы

### <a name="why-doesnt-the-expense-mobile-app-enter-the-payment-method-by-default"></a>Почему мобильное приложение "Расходы" не указывает способ оплаты по умолчанию?

Организации могут настроить параметр **Способ оплаты по умолчанию** для каждой категории расходов при ее создании. Кроме того, при настройке способов оплаты вы можете установить в поле **Способ оплаты по умолчанию** значение **Только импорт**.

Когда значение **Только импорт** включено для способа оплаты, по умолчанию метод оплаты не вводится. Он будет пустым в категориях расходов, в которых настроен этот способ оплаты. Такое поведение согласовано как в Интернет-интерфейсе, так и в интерфейсе для мобильных устройств.
    
Когда параметр **Только импорт** не включен для метода оплаты, установленное значение вводится по умолчанию для категорий расходов, для которых настроен этот метод оплаты. Однако существует известная проблема, когда значение по умолчанию не вводится в мобильном приложении "Расходы". Чтобы обойти эту проблему, выберите вручную способ оплаты перед сохранением отчета о расходах. 

### <a name="why-cant-i-add-or-edit-financial-dimensions-in-the-expense-mobile-app"></a>Почему я не могу добавлять или редактировать финансовые аналитики в мобильном приложении "Расходы"?

Ввод измерений и распределений не поддерживается. Чтобы обойти это ограничение, вы можете настроить эти поля по умолчанию в мобильном приложении, настроив финансовые аналитики по умолчанию для каждого проекта или сотрудника.

### <a name="why-do-i-sometimes-see-a-synchronization-error-in-the-expense-mobile-app"></a>Почему я иногда вижу ошибку синхронизации в мобильном приложении "Расходы"?

Если строки расходов не соответствуют требованиям политики и пользователь отправляет отчет о расходах, не обращая внимания на предупреждение политики, мобильные данные не синхронизируются с сервером, и происходит сбой синхронизации. Все отчеты о расходах, отправленные после сбоя синхронизации, останутся в состоянии сбоя и вызовут больше сбоев синхронизации. Единственный способ исправить эту ситуацию — вручную удалить уведомления о синхронизации. Эта проблема решена путем остановки отправки отчетов о расходах, когда предупреждения политики не были учтены, чтобы избежать ошибок синхронизации.

### <a name="why-isnt-project-and-category-validation-correctly-reflected-in-the-expense-mobile-app"></a>Почему проверка проекта и категории не отражается правильно в мобильном приложении "Расходы"?

Эта проверка в настоящий момент не поддерживается. Однако поддержка может быть добавлена в будущем. 

### <a name="what-document-types-are-supported-in-the-expense-mobile-app"></a>Какие типы документов поддерживаются в мобильном приложении "Расходы"?

Мобильное приложение "Расходы" поддерживает только изображения. В настоящее время оно не поддерживает PDF-файлы или другие документы.
