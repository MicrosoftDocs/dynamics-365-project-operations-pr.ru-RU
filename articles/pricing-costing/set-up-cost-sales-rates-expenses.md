# <a name="set-up-cost-and-sales-rates-for-expenses"></a>Настройка ставок себестоимости и продаж для расходов

_**Относится к:** Project Operations для сценариев на основе ресурсов/нескладируемых запасов, упрощенное развертывание — от сделки до выставления счетов-фактур_

Вы можете настроить себестоимость и продажную цену для категорий транзакций в Dynamics 365 Project Operations. Поскольку себестоимость и продажные цены предназначены для расходов, каждая категория транзакций, которая их включает, также должна быть настроена как категория расходов. Такая настройка обеспечивает точность последующих функций. Себестоимость и цены продажи для категорий транзакций могут быть указаны только в одной валюте, которая должна быть валютой в заголовке прайс-листа.

Чтобы настроить ставки себестоимости и продаж для категорий транзакций, выполните следующие действия. 

1. Перейдите **Продажи** > **Клиенты** > **Прайс-листы**.
2. Выберите **Создать** для создания нового прайс-листа. 
3. В разделе **Цены категорий** в меню вложенной сетки выберите **Создать цену категории**. 
4. На странице **Быстрое создание** введите категорию транзакций и единицу, для которой вы создаете новую цену.

В следующей таблице перечислены поля на вкладке **Общие** и странице **Быстрое создание** строки цен категории, которую следует иметь в виду при создании цен категории в прайс-листе продаж или себестоимости.

| Поле | Размещение | Описание: | Воздействие на последующие элементы |
| --- | --- | --- | --- |
| Категория проводки | Вкладка **Общие** и страницы **Быстрое создание** | Выберите категорию транзакций, для которой вы создаете цену продажи или себестоимость. | Категория транзакций во входящей оценке или фактическая стоимость для расходов будет сопоставлена с этой строкой, чтобы установить ставку себестоимости или продаж по умолчанию для категории транзакций. |
| Группа единиц измерения | Вкладка **Общие** и страницы **Быстрое создание** | Расписание единиц измерения по умолчанию устанавливается равным расписанию единиц измерения категории транзакций. | Это поле не оказывает влияния на последующую обработку. |
| Единица | Вкладка **Общие** и страницы **Быстрое создание** | Выберите единицу, для которой устанавливаются ставки. | Единица во входящей смете или фактической стоимости сопоставляется с единицей в этой строке, чтобы установить по умолчанию ставку по смете расходов или фактическим расходам. |
| Способ ценообразования | Вкладка **Общие** и страницы **Быстрое создание** | Возможные значения поля **Метод ценообразования**: **Цена за единицу**, **По себестоимости** и **Наценка сверх себестоимости**. | При установке цены при выборе значения **Цена за единицу** блокируется поле **Процент** в строке цены категории. Если выбрано значение **По себестоимости**, поля **Цена** и **Процент** блокируются в прайс-листе продаж. Например, тариф за километраж составляет 10 долларов США за милю и 8 долларов США за километр. | Тариф за пробег будет ставкой, которая по умолчанию равно цене или себестоимости единицы строки входящей оценки или фактических данных для класса транзакций "расходы".|
| Процент | Вкладка **Общие** и страницы **Быстрое создание** | Установите процент сверх себестоимости для сочетания категории транзакций и единицы. Например, ставка продажи авиабилетов должна быть на 10 процентов выше себестоимости понесенных расходов на авиаперевозки. | Этот процент сверх себестоимости применяется только для прайс-листа продаж, когда выбран метод ценообразования **Наценка сверх себестоимости**. |

## <a name="pricing-methods-for-expenses"></a>Методы ценообразования для расходов

Когда вы устанавливаете цены категорий, которые актуальны только в контексте расчета затрат, вы можете использовать один из следующих трех методов ценообразования:

- **Цена за единицу**
- **По себестоимости**
- **Наценка сверх себестоимости**

### <a name="price-per-unit"></a>Цена за единицу
Когда этот метод ценообразования выбран в строке цены категории, которая связана с прайс-листом продаж, цена по умолчанию используется для комбинации категории и единицы как в оценке, так и в фактической сумме. Под оценкой понимаются строки оценки проекта для расходов, детали строки предложения с расценками и детали строки контракта для расходов.

### <a name="at-cost"></a>По себестоимости
Когда этот метод ценообразования выбран в строке цены категории, которая связана с прайс-листом продаж, цена по умолчанию используется для комбинации категории и единицы только для фактической суммы расходов. Например,фактические суммы продаж, за которые не выставлены счета, для класса проводки расходов. Цена за единицу устанавливается на основе фактических продаж, по которым не выставлен счет, исходя из цены за единицу на основе фактических затрат по этому расходу. Установка цены по умолчанию на основе себестоимости не выполняется в оценках проекта для расходов или в строке предложения с расценками и деталях строки контракта для расходов.

### <a name="markup-over-cost"></a>Наценка сверх себестоимости
Когда этот метод ценообразования выбран в строке цены категории, которая связана с прайс-листом продаж, цена по умолчанию используется для комбинации категории и единицы только для фактической суммы расходов. Например,фактические суммы продаж, за которые не выставлены счета, для класса проводки расходов. Эта цена за единицу устанавливается на основе фактических продаж, за которые не выставлен счет, на вычисленное значение, исходя из цены единицы по фактическим затратам для этого расхода после применения определенного процента наценки. Установка цены по умолчанию на основе себестоимости не выполняется в оценках проекта для расходов или в строке предложения с расценками и деталях строки контракта для расходов.
