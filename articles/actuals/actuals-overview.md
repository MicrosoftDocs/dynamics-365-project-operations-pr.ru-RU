# <a name="actuals"></a>Фактические значения 

_**Относится к:** Project Operations для сценариев на основе ресурсов/нескладируемых запасов, упрощенное развертывание — от сделки до выставления счетов-фактур_

Фактические данные представляют собой проанализированный и утвержденный финансовый и календарный прогресс по проекту. Они создаются в результате утверждения записей о времени, расходах, использовании материалов, а также записей журналов и счетов.

## <a name="journal-lines-and-time-submission"></a>Отправка строк журналов и времени

Для получения дополнительной информации о записи времени см. тему [Обзор записи времени](../time/time-entry-overview.md).

### <a name="time-and-materials"></a>Время и материалы

Когда отправляемая запись времени связана с проектом, который сопоставлен со строкой контракта на время и материалы, система создает две строки журнала: одну для затрат и одну для продаж, за которые не выставлен счет.

### <a name="fixed-price"></a>Фиксированная цена

Когда отправляемая запись времени связана с проектом, который сопоставляется со строкой контракта с фиксированной ценой, система создает одну строку журнала для стоимости.

### <a name="default-pricing"></a>Цены по умолчанию

Логика создания цен по умолчанию находится в строке журнала. Значения полей из записи времени копируются в строку журнала. Эти значения включают дату транзакции, строку контракта, с которой сопоставлен проект, и результат валюты в соответствующем прайс-листе.

Поля, которые влияют на цены по умолчанию, например **Роль** и **Единица распределения ресурсов** используются для определения соответствующей цены в строке журнала. Вы можете добавить настраиваемое поле к записи времени. Если вы хотите, чтобы значение поля распространялось на фактические значения, создайте поле в таблицах **Фактические значения** и **Строка журнала**. Используйте настраиваемый код для распространения значения выбранного поля из "Запись времени" в "Фактические значения" по строке журнала с помощью источников транзакции. Для получения дополнительной информации об источниках транзакций и подключениях см. [Связь фактических данных с исходными записями](linkingactuals.md#example-how-transaction-origin-works-with-transaction-connection).

## <a name="journal-lines-and-basic-expense-submission"></a>Строки журнала и базовая отправка расходов

Дополнительные сведения о записи расходов см. в теме [Обзор расходов](../expense/expense-overview.md).

### <a name="time-and-materials"></a>Время и материалы

Когда отправляемая базовая запись затрат связана с проектом, который сопоставлен со строкой контракта на время и материалы, система создает две строки журнала: одну для затрат и одну для продаж, за которые не выставлен счет.

### <a name="fixed-price"></a>Фиксированная цена

Когда отправленная базовая запись расхода связана с проектом, который сопоставляется со строкой контракта с фиксированной ценой, система создает одну строку журнала для стоимости.

### <a name="default-pricing"></a>Цены по умолчанию

Логика ввода цен по умолчанию для расходов основана на категории расходов. Дата транзакции, строка контракта, с которой сопоставлен проект, и валюта все используются для определения соответствующего прайс-листа. Поля, которые влияют на цены по умолчанию, например **Категория проводки** и **Единица распределения ресурсов** используются для определения соответствующей цены в строке журнала. Однако это работает только в том случае, если в прайс-листе указан метод ценообразования **Цена за единицу**. Если метод ценообразования — **По себестоимости** или **Наценка по стоимости**, цена, введенная при создании записи расхода, используется для стоимости, а цена в строке журнала продаж рассчитывается на основе метода ценообразования. 

Вы можете добавить настраиваемое поле в запись расхода. Если вы хотите, чтобы значение поля распространялось на фактические значения, создайте поле в таблицах **Фактические значения** и **Строка журнала**. Используйте настраиваемый код для распространения значения выбранного поля из "Запись времени" в "Фактические значения" по строке журнала с помощью источников транзакции. Для получения дополнительной информации об источниках транзакций и подключениях см. [Связь фактических данных с исходными записями](linkingactuals.md#example-how-transaction-origin-works-with-transaction-connection).

## <a name="journal-lines-and-material-usage-log-submission"></a>Отправка строк журнала и журнала использования материалов

Для получения дополнительной информации о записи расходов см. [Журнал использования материалов](../material/material-usage-log.md).

### <a name="time-and-materials"></a>Время и материалы

Когда отправленная запись журнала использования материалов связана с проектом, который сопоставлен со строкой контракта на время и материалы, система создает две строки журнала: одну для стоимости и одну для продаж без выставления счета.

### <a name="fixed-price"></a>Фиксированная цена

Когда отправленная запись журнала использования материала связана с проектом, который сопоставляется со строкой контракта с фиксированной ценой, система создает одну строку журнала для стоимости.

### <a name="default-pricing"></a>Цены по умолчанию

Логика ввода цен по умолчанию для материала основана на комбинации продукта и единицы. Дата транзакции, строка контракта, с которой сопоставлен проект, и валюта все используются для определения соответствующего прайс-листа. Поля, которые влияют на цены по умолчанию, например **Артикул** и **Единица** используются для определения соответствующей цены в строке журнала. Однако это работает только для продуктов из каталога. Для вписанных продуктов цена, введенная при создании записи в журнале использования материала, используется для стоимости и цены продажи в строках журнала. 

Вы можете добавить настраиваемое поле в запись **Журнал использования материалов**. Если вы хотите, чтобы значение поля распространялось на фактические значения, создайте поле в таблицах **Фактические значения** и **Строка журнала**. Используйте настраиваемый код для распространения значения выбранного поля из "Запись времени" в "Фактические значения" по строке журнала с помощью источников транзакции. Для получения дополнительной информации об источниках транзакций и подключениях см. [Связь фактических данных с исходными записями](linkingactuals.md#example-how-transaction-origin-works-with-transaction-connection).

## <a name="use-entry-journals-to-record-costs"></a>Использование журналов записей для записи стоимости

Журналы можно использовать для записи стоимости или дохода в классах проводок по материалам, сборам, времени, расходам и налогам. Журналы могут использоваться для следующих целей:

- Перемещение фактических данных транзакций из другой системы в Microsoft Dynamics 365 Project Operations.
- Запись затрат, которые произошли в другой системе. Эти затраты могут включать затраты на закупку или субподряд.

> [!IMPORTANT]
> Приложение не проверяет тип строки журнала или соответствующие цены, введенные в строке журнала. Следовательно, только пользователь, который полностью осознает влияние на учет от фактических данных по проекту должен использовать журналы записей для создания фактических данных. Из-за влияния этого типа журнала вам следует тщательно выбирать, у кого есть доступ для создания журналов входа.
