# <a name="transaction-origins---link-actuals-to-their-source"></a>Происхождения проводки — привязка фактических значений к их источнику

_**Относится к:** Project Operations для сценариев на основе ресурсов/нескладируемых запасов, упрощенное развертывание — от сделки до выставления счетов-фактур_

Записи о происхождении проводок создаются для связывания фактических значений с их источником, таким как записи времени, записи расходов, журналы использования материалов или счета по проекту.

Следующий пример показывает типичную обработку записей времени в течение жизненного цикла проекта Project Operations.

> ![Обработка записей времени в Project Operations.](media/basic-guide-17.png)
 
1. Отправка записи времени вызывает создание двух строк журнала: одной для стоимости, второй для продаж без выставленного счета.
2. Последующее утверждение записи времени вызывает создание двух строк фактических значений: одной для стоимости, второй для продаж без выставленного счета.
3. Когда пользователь создает счет по проекту, проводка строки счета создается на основе данных из фактических данных продаж без выставления счета.
4. При утверждении счета создаются два новых фактических значения: реверсирование продаж без выставления счетов и фактические данные продаж, за которые выставлен счет.

Каждое событие в этом рабочем процессе вызывает создание записей в сущности "Происхождение проводки" для обеспечения возможности трассировки отношений между этими записями, которые создаются для ввода времени, строк журнала, фактических значений и сведений о строке счета.
