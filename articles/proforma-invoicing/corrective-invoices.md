# <a name="corrective-project-based-invoices"></a>Корректирующие счетов на основе проектов

_**Относится к:** Project Operations для сценариев на основе ресурсов/без запасов_

Подтвержденный счет по проекту может быть скорректирован для обработки изменений или кредитов по согласованию с заказчиком и менеджером проекта.

Чтобы внести изменения в подтвержденный счет, откройте подтвержденный счет и выберите **Исправить этот счет**. 

> [!NOTE]
> Этот выбор недоступен, пока счет проекта не подтвержден или в счете на основе проектов есть авансы или гонорары, или выверка авансов или гонораров.

Новый черновик счета создается из подтвержденного счета. Все детали строк счета из ранее подтвержденного счета копируются в новый черновик. Ниже приведены некоторые из ключевых моментов, которые нужно знать о деталях строк в новом исправленном счете:

- Все количества обновляются до нуля. Dynamics 365 Project Operations предполагает, что все номенклатуры с выставленными счетами, полностью кредитуются. При необходимости вы можете вручную обновить эти количества, чтобы отразить количество, по которому выставляется счет, а не количество, которое кредитуется. На основе введенного количества приложение рассчитывает кредитуемое количество. Эта сумма отражается в фактических данных, которые создаются при подтверждении исправленного счета. Если вы вносите изменения в сумму налога, вы должны ввести правильную сумму налога, а не сумму налога, которая кредитуется.
- Исправления вех всегда обрабатываются как полные кредиты.


> [!IMPORTANT]
> Для сведений строки счета, которые являются исправлением других платежей с уже выставленными счетами, поле **Исправление** задано как **Да**. Для счетов, у которых есть исправленные сведения строки счета поле **Содержит корректировки** задано как **Да**.
