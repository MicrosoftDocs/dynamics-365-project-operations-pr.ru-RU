# <a name="manage-multiple-customers-on-project-contracts"></a>Управление несколькими клиентами в контрактах по проекту

В этом разделе представлена информация об управлении несколькими клиентами в контракте по проекту. Вы можете использовать контракт по проекту, когда для финансирования сделки требуется договорное соглашение для нескольких клиентов. На страница **Контракт по проекту**, вкладка **Cводка** содержит информацию об основном клиенте сделки. Другие клиенты, участвующие в сделке, могут быть добавлены на вкладку **Клиенты**.

Все клиенты контракта на вкладке **Клиенты** в контракте по проекту по умолчанию являются клиентами строки контракта для любых новых строк контракта на основе проектов, которые создаются для контракта по проекту. Любые существующие строки контрактов на основе проектов не наследуют новые записи клиентов контракта, которые создаются позднее.

Клиентов контракта и клиентов строки контракта можно добавить, обновить или удалить в любое время до заключения контракта. Клиент контракта по проекту должен быть настроен как клиент в ответственной компании или юридическом лице на странице **Клиенты**. Юридические лица заданы в модуле **Управление и учет по проектам** Dynamics 365 Project Operations и доступны как компании в модулях Project Operations **Продажи по проекту** и **Доставка**.

## <a name="primary-customers"></a>Основные клиенты

Потенциальный клиент на вкладке **Сводка** контракта по проекту является основным клиентом контракта. Основного клиента нельзя обновить из списка клиентов контракта. При попытке удалить основного клиента из списка клиентов по контракту появится сообщение об ошибке, указывающее, что запись основного клиента в контракте не может быть удалена. Вместо этого, измените клиента в поле **Потенциальный клиент** на вкладке **Сводка** контракта по проекту. При обновлении этого поля выбранный новый клиент добавляется как новый клиент контракта с помощью флага **Основной**, установленного на **Да**. Предыдущий основной клиент по-прежнему является клиентом по контракту, однако он больше не является основным клиентом.

## <a name="create-update-or-delete-a-contract-customer-record"></a>Создание, обновление или удаление записи о клиенте по контракту

Клиента по контракту можно создать, обновить или удалить на вкладке **Клиенты** на странице **Контракт**. Следующие поля включены в записи клиента контракта для контракта по проекту.

| --- | --- | --- |

Это поле блокируется после создания записи. Чтобы обновить запись, удалите запись и создайте ее заново. Если вы записали какие-либо фактические данные или если запись клиента по контракту является основным клиентом, вы не можете удалить эту запись. При создании строки контракта клиенты контракта копируются как клиенты строки контракта. |
| Процент выставления счетов с разбиением | Редактируемая сетка на вкладке **Клиенты контракта**, а также на главной странице и странице быстрого создания для клиента контракта. | Представляет процентную долю каждой транзакции продажи без выставления счета, относящейся к данному клиенту контракта. Когда создаются новые строки контракта по проекту, процент разделения выставленных счетов копируется в новые любые созданные строки контракта и в клиентов строк контракта по проекту. |

Это значение по умолчанию берется из записи связанной организации. Имя контактного лица копируется в **Счет по названию контракта** в счете, созданном для этого клиента. |
| Имя для счета | Редактируемая сетка на вкладке **Клиенты контракта**, а также на главной странице и странице быстрого создания для клиента контракта. | Используйте это поле для идентификации контактного лица по счету для клиента. Это значение по умолчанию берется из записи связанной организации. Имя копируется в поле **Счет по названию контракта** в счете, созданном для клиента. |
| Условия оплаты | Редактируемая сетка на вкладке **Клиенты контракта**, а также на главной странице и странице быстрого создания для клиента контракта. | Это значение по умолчанию берется из записи связанной организации. Условия копируются в **Счет по названию контракта** в счете, созданном для этого клиента. |

Ответственная компания приравнивается к юридическому лицу в модуле Project Operations **Управление и учет по проектам**. ## <a name="edit-billing-split-percentages"></a>Изменить процентное соотношение разделения выставления счетов

Проценты разделения выставления счетов можно редактировать в сетке. Если проценты разделения выставления счетов в сумме не составляют 100 процентов, произойдет ошибка. После редактирования процентов разделения выставления счетов обновите страницу **Контракт по проекту**, чтобы удалить ошибку.

Вы также можете выбрать **Распределить равномерно** во вложенной сетке клиента контракта по проекту. Разделения выставления счетов равномерно распределяются между всеми клиентами контракта по проекту. Если есть какой-либо коэффициент округления, он будет добавлен к клиенту округления. У одного из клиентов контракта всегда установлен флаг **Округление** на **да**. Этот клиент является клиентом округления. Обычно клиент округления также является основным клиентом контракта, но это не обязательно.
