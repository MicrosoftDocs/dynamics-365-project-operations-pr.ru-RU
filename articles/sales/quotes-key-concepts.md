# <a name="concepts-unique-to-project-based-quotes"></a>Понятия, уникальные для предложений с расценками на основе проекта

_**Относится к:** Project Operations для сценариев на основе ресурсов/без запасов_

Прежде чем приступать к использованию предложений с расценками по проектам в Microsoft Dynamics 365 Project Operations, необходимо ознакомиться со следующими ключевыми понятиями.

## <a name="owning-company"></a>Ответственная компания

Ответственная компания представляет юридическое лицо, отвечающее за реализацию проекта. Клиент, указанный в предложении с расценками, должен быть действительным клиентом этого юридического лица в приложениях для управления финансами и операциями. Валюта ответственной компании и валюта единицы по контракту, выбранной в предложении с расценками по проекту, должны совпадать.

## <a name="contracting-unit"></a>Единица по контракту

Единица по контракту представляет собой подразделение или предприятие, которое отвечает за реализацию проекта. Вы можете настроить затраты на ресурсы для каждой контрактной единицы. Когда вы указываете стоимость ресурсов для ресурса в единице по контракту, вы можете установить разные ставки затрат для ресурсов, которые это единица по контракту заимствует, или для других подразделений или предприятий внутри компании. Эти ставки затрат называются трансфертными ценами, заимствованием ресурсов или ценами обмена. Когда вы настраиваете стоимость заимствования ресурсов из других подразделений, вы можете настроить ставки затрат в валюте подразделения, из которого заимствуются ресурсы.

## <a name="cost-currency"></a>Валюта затрат

Валюта затрат в Project Operations — это валюта, в которой затраты отражаются в отчетности. Эта валюта выводится из валюты, прикрепленной к полю **Единица по контракту** для предложения с расценками, контракта и проекта. Затраты по проекту могут учитываться в любой валюте. Однако существует конвертация валюты из валюты, в которой были учтены затраты, в валюту затрат по проекту.

Поскольку обменные курсы на платформе Dataverse не могут быть актуальными на определенную дату, итоговые суммы затрат на экране могут со временем измениться, если вы обновите курсы обмена валют. Однако затраты, учтенные в базе данных, остаются неизменными, поскольку суммы хранятся в той валюте, в которой они были понесены.

## <a name="sales-currency"></a>Валюта продаж

Валюта продаж в Project Operations — это валюта, в которой учитываются и отображаются оценочные и фактические суммы продаж. Это также валюта, в которой клиенту выставляется счет за сделку. В предложении с расценками по проекту валюта продаж по умолчанию берется из записи клиента или записи организации и может быть изменена при создании предложения с расценками. После сохранения предложения с расценками валюту продаж, однако, изменить нельзя. Прайс-листы на продукты и проекты по умолчанию основаны на валюте продажи предложения с расценками.

В отличие от затрат, значения продаж могут учитываться **только** в валюте продаж.

## <a name="billing-method"></a>Метод выставления счета

В проектах обычно используются модели контрактов с фиксированной ценой и на основе потребления. В Project Operations контрактная модель проекта представлена методом выставления счетов. Метод выставления счетов может принимать два значения: «Время и материал» и «Фиксированная цена».

- **Время и материал** — это модель контракта, основанная на потреблении, где все понесенные затраты подкрепляются соответствующей выручкой. По мере того, как вы оцениваете или несете дополнительные расходы, соответствующие оценочные и фактические продажи также увеличиваются. Вы можете указать максимальные лимиты для строк предложения с расценками, для которых используется этот метод выставления счетов. Таким образом вы можете ограничить фактический доход. На расчетный доход не влияют ограничения, наложенные максимальными лимитами.
- **Фиксированная цена** — это модель контракта с фиксированным вознаграждением, в которой значения продаж не будут зависеть от понесенных затрат. Стоимость продажи является фиксированной и не изменяется по мере того, как вы оцениваете или несете дополнительные расходы.

## <a name="project-price-lists"></a>Прайс-листы проектов

Прайс-листы проекта — это прайс-листы, которые используются для ввода цен по умолчанию, а не для ставок затрат для времени, расходов и других компонентов, связанных с проектом. Может быть несколько прайс-листов, и каждый лист может иметь свою дату вступления в силу для каждого предложения с расценками по проекту. Project Operations не поддерживает перекрывающиеся даты вступления в силу в прайс-листах проекта.

## <a name="product-price-lists"></a>Прайс-листы продуктов

Прайс-листы продуктов — это прайс-листы, которые используются для ввода цен по умолчанию, а не для ставок затрат для строк на основе продуктов в предложении с расценками. Для каждого предложения с расценками может быть только один прайс-лист на продукты.

## <a name="transaction-classes"></a>Классы проводок

Project Operations поддерживает четыре типа классов транзакций:

- Время
- Расходы
- Материал
- Сбор

Значения затрат и продаж могут быть оцениваться и учитываться с помощью проводок классов **Время**, **Расходы** и **Материалы**. **Сбор** — это класс проводок, предназначенный только для дохода.

## <a name="work-entities-and-billing-entities"></a>Сущности работы сущности выставления счетов

Проекты и задачи — это сущности, которые представляют работу. Строки предложения с расценками и строки контракта — это сущности, которые представляют выставление счетов. Вы можете связать различные сущности работы с вариантами выставления счетов, связав их со строками предложения с расценками или строками контракта.

## <a name="multi-customer-deals"></a>Сделки с несколькими клиентами

Сделки с несколькими клиентами происходят, когда в одном счете указано более одного клиента. Вот несколько типичных примеров:

- **OEM-предприятия и их партнеры**— партнеры и реселлеры продают продукт, который включает в себя услуги, создающие добавленную стоимость. Во время сделки с клиентом OEM-производитель обычно предлагает профинансировать часть проекта.
- **Проекты государственного сектора** — несколько департаментов местного правительства соглашаются финансировать проект, и им выставляются счета в соответствии с заранее согласованным разделением. Например, школьный округ и городское или местное управление соглашаются финансировать строительство плавательного бассейна.

## <a name="invoice-schedules"></a>Расписания счетов

Графики выставления счетов задаются для конкретных строк предложения с расценками и являются необязательными. Графики выставления счетов создаются на основе определенных дат начала и окончания, а также частоты выставления счетов. Они используются на стадии контракта, когда настраивается процесс автоматического создания счетов. На этапе предложения с расценками графики выставления счетов использовать необязательно. Если они создаются на этапе предложения с расценками они копируются в контракт по проекту, который создается при реализации предложения с расценками по проекту.

## <a name="differences-from-dynamics-365-sales-quotes"></a>Отличия от предложений с расценками в Dynamics 365 Sales

Предложения с расценками в Project Operations построены на основе предложений с расценками Dynamics 365 Sales. Однако есть некоторые важные различия в функциональности, о которых вам следует знать:

- Предложения с расценками в Project Operations имеют два разных типа строк: один для проектов, а другой для продуктов.
- Предложения с расценками в Project Operations имеют свою собственную страницу и элементы пользовательского интерфейса, бизнес-правила, бизнес-логику в подключаемых модулях и клиентские скрипты, которые отличают их от предложений с расценками из Sales.
- В Sales к одному предложению с расценками по продажам можно прикрепить несколько заказов. В Project Operations к предложению с расценками по проекту можно прикрепить только один контракт по проекту.
- Когда вы реализуете предложение с расценками по продажам, соответствующая возможная сделка может оставаться открытой. После реализации предложения с расценками проекта связанная возможная сделка закрывается.
- Предложение с расценками по продажам не включает в себя некоторые поля и понятия, которые входят в предложение с расценками по проекту. Эти поля включают **Единица по контракту**, **Менеджер по работе с клиентами** и **Имя контакта для счета**.
- Предложения с расценками по продажам и предложения с расценками по проектам можно отличить по значению в поле **Тип**, которое представляет собой набор параметров. У предложений с расценками по продажам это поле имеет значение **На основе товаров**. У предложений с расценками по проектам оно имеет значение **На основе работ**.

Из-за этих различий мы не рекомендуем использовать предложения с расценками в Sales и в Project Operations как взаимозаменяемые.
