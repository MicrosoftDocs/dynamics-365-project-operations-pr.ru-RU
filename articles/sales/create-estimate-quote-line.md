# <a name="create-estimates-on-a-quote-line"></a>Создание оценок в строке предложения с расценками

_**Относится к:** Project Operations для сценариев на основе ресурсов/нескладируемых запасов, упрощенное развертывание — от сделки до выставления счетов-фактур_

В предложении с расценками на основе проекта можно использовать сущность сведения строки предложения с расценками для оценки работы, которую необходимо выполнить по проекту. Затем можно поделиться этой оценкой с клиентом.

Строки предложения с расценками на основе проекта не обязательно должны содержать какие-либо сведения строки предложения с расценками. Кроме того, они могут иметь много сведений строки предложения с расценками. Сведения строки предложения с расценками используются для оценки времени, расходов и выплат. Dynamics 365 Project Operations не допускает материальных оценок в сведениях строки предложения с расценками. Они называются классами транзакции. Оценку сумм налога можно также ввести в класс транзакций.

В дополнение к классам транзакции, сведения строки предложения с расценками имеет тип транзакции. Имеется два типа транзакций для сведений строки предложения с расценками: **Стоимость** и **Контракт по проекту**.

## <a name="estimate-by-using-a-contract"></a>Оценка с помощью контракта

Если вы использовали предложение с расценками Project Operations при создании контракта на основе проекта, оценка, которая была выполнена для каждой строки предложения с расценками по предложению с расценками, копируется в контракт по проекту. Структура контракта по проекту аналогична структуре предложения с расценками, которое имеет строки, сведения по строкам и расписания выставления счетов.

Оценки можно вносить непосредственно в контракт по проекту, как в предложение с расценками проекта. Для предложения проекта оценка осуществляется с помощью строк контракта и сведений строк контракта. Сведения строк контракта также могут создаваться из плана проекта, который был создано с помощью подхода оценки "снизу вверх".

Сведения строки контракта могут использоваться для оценки времени, расходов или сборов. Оценку сумм налога можно также ввести в сведения строки контракта.

Материальные оценки не допускаются в сведениях строки контракта.

Процессы, которые поддерживаются в контракте по проекту, это создание и подтверждение счета. При создании счета создается черновик счета на основании проекта, который включает все фактические продажи, за которые еще не высатвлен счет, до текущей даты.

Подтверждение делает контракт доступным только для чтения и изменяет его состояние с **Черновик** на **Подтверждено**. После того как вы выполните этой действие, нельзя отменить его. Поскольку это действие постоянное, рекомендуем сохранять контракт в состоянии **Черновик**.

Единственные различия между черновиками контракта и подтвержденными контрактами — это их состояние и тот факт, что черновики контрактов можно изменять, в то время как подтвержденные контракты изменять невозможно. Создание счетов и отслеживание фактических значений можно делать как с черновиками контрактов, так и с подтвержденными контрактами.

Изменение заказов не поддерживается в контрактах или проектах.

## <a name="estimating-projects"></a>Оценка проектов

Вы можете оценить время и затраты на проекты, но не материалы или гонорары.

Оценки времени создаются, когда вы создаете задачу и определяете атрибуты универсального ресурса, который необходим для выполнения этой задачи. Оценки времени создаются из запланированных задач. Оценки времени не создаются при создании универсальных участников рабочей группы вне контекста расписания.

Оценки расходов вводятся в сетку на странице **Оценки**.

## <a name="customize-the-quote-line-detail-and-contract-line-detail-entities"></a>Настройка сведения сущностей строки предложения с расценками и сведений строки контракта

Если вы добавили настраиваемое поле в сведения строки предложения с расценками и хотите, чтобы система вводила значение поля в качестве значения по умолчанию в связанной строке затрат, которую она создает, используйте средства регистрации подключаемых модулей PreOperationContractLineDetailUpdate и PreOperationQuoteLineDetailUpdate. Этот подключаемые модули необходимо заново зарегистрировать после изменения сведений строки предложения с расценками и сведений строки контракта. Чтобы завершить этот процесс, выполните следующие шаги.

1. Откройте PluginRegistrationTool и подключитесь к вашему экземпляру в сети.
2. Выбор **Поиск** и выполните поиск подключаемого модуля для обновления.
3. Выберите подключаемый модуль, затем, на главной странице, выберите **Выбрать**.
4. Выберите шаг подключаемого модуля для обновления, щелкните правой кнопкой мыши, затем выберите **Обновить**.
5. В диалоговом окне **Обновить существующий шаг** в поле **Атрибуты фильтрации** выберите кнопку с многоточием (**...**):
6. В диалоговом окне **Выберите атрибуты** установите флажки для настраиваемых атрибутов.
7. Выберите **ОК**, чтобы закрыть диалоговое окно, затем выберите **Обновить шаг**.
8. Повторите шаги с 1 по 7 для второго подключаемого модуля.
9. Закройте PluginRegistrationTool.
