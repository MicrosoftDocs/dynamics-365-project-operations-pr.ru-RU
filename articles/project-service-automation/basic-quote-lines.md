# <a name="quotes-and-quote-lines"></a>Предложение с расценками и строки предложения с расценками

[!INCLUDE[cc-applies-to-psa-app-3.x](../includes/cc-applies-to-psa-app-3x.md)]

В Dynamics 365 Project Service Automation имеется два типа предложений с расценками: предложения с расценками проекта и предложения с расценками продаж. Эти два типа отличаются следующим:

- В предложении с расценками продаж имеется только одна сетка для номенклатур строк. В предложении с расценками проекта имеются две сетки для номенклатур строк: одна для строк проекта, другая для строк продуктов.
- Предложение с расценками продаж поддерживает активацию и изменения. Предложения с расценками проекта не поддерживает эти процессы.
- Можно вложить несколько заказов в предложение с расценками продаж. Можно вложить только один контракт по проекту в предложение с расценками проекта.
- Можно реализовать предложение с расценками продаж и оставить возможную сделку открытой. После реализации предложения с расценками проекта связанная возможная сделка закрывается.
- Предложение с расценками продаж не включают некоторые поля и понятия, которые включены в предложение с расценками проекта. Эти поля включают **Единица по контракту**, **Менеджер по работе с клиентами** и **Имя контакта для счета**.  
- Предложения с расценками продаж и предложения с расценками по проекту также определяются полем на основе набора параметров, которое называется **Тип**. Для предложения с расценками продаж это поле имеет значение **На основе элемента**. Для предложения с расценками проекта оно имеет значение **На основе работ**.

В этом разделе основное внимание уделяется подробным сведениям о предложениях с расценками по проекту.

Предложение с расценками по проекту в PSA может содержать несколько номенклатур строки или строк предложения с расценками. На самом деле предложение с расценками проекта имеет две сетки для номенклатур строк. Одна сетка предназначена для строк на основе проекта, которые обеспечивают подробные оценки. Другая сетка предназначена для строк на основе продукта, использующих простую цену за единицу и подход на основе количества.

- **На основе проекта** — сумма (значение в предложении) определяется после оценки необходимого объема работ. Можно оценить работу на высоком уровне или можно оценить ее напрямую как сведения по строке под каждой строкой предложения с расценками. Наконец, можно оценить работу на основе оценок с самого основания, используя проект и план проекта. Строки предложения с расценками на основе проекта присутствуют только в предложениях с расценками на основе проекта, созданных с помощью Project Service Automation. Этот тип строки предложения с расценками является настроенной формой вписываемых строк предложения с расценками, которые доступны в Microsoft Dynamics 365 Sales.
- **На основе продукта** — сумма (значение в предложении) определяется на основе проданного количества единиц и цены за единицу продукта. Продукт в строке на основе продукта может браться из каталога продуктов в Sales, или это может быть определенный вами продукт. Этот тип строки предложения с расценками также доступен в предложениях с расценками на основе проекта, созданных с помощью PSA.

Сумма в предложении с расценками является итоговой суммой по всем строкам на основе продуктов и строкам на основе проекта.

> [!NOTE]
> Предложения с расценками и строки предложений с расценками не являются обязательными в PSA. Можно начать процесс проекта с контракта по проекту (проданный проект). Однако возможная сделка всегда требуется, независимо от того, начинаете ли вы с предложения с расценками или с контракта по проекту.

## <a name="project-based-quote-lines"></a>Строки предложения с расценками на основе проекта

Строка предложения с расценками на основе проекта в PSA имеет следующие способы выставления счетов:

- Время и материал
- Фиксированная цена

### <a name="time-and-material"></a>Время и материал

Метод выставления счетов "Время и материал" основан на потреблении. Если выбран этот метод выставления счетов, клиенту выставляется счет по мере возникновения затрат по проекту. Счета в создаются периодически с частотой на основе дат. Во время процесса продаж сумма в предложении по расценкам для компонента "время и материалы" предоставляет только оценку конечной стоимости для клиента. Поставщик не дает обязательств завершить проект в точности за сумму, указанную в предложении с расценками. Компонент "время и материалы" увеличивает риск клиента. Клиенты могут захотеть обсудить дополнительные условия о не превышении определенных сумм для уменьшения своего риска. PSA не поддерживает задание условий о не превышении определенных сумм.

### <a name="fixed-price"></a>Фиксированная цена

В методе выставления счетов с фиксированной ценой поставщик обязуется выполнить проект за фиксированную стоимость для клиента. Клиенту выставляется счет на сумму из предложения с расценками по строке предложения с фиксированной ценой, независимо от затрат, понесенных поставщиком для выполнения этой строки предложения. Сумма строки предложения с расценками с фиксированной ценой выставляется в счете одном из следующих способов: 

- Как общая сумма в начале или в конце проекта либо при завершении важного этапа проекта. 
- С частотой на основе дат в равных долях от фиксированной суммы по строке предложения. Эти частичные платежи называются периодическими этапами.
- В частичных платежах, которые имеют денежное выражение, зависящее от хода выполнения работ или достижения определенных этапов проекта. В этом случае сумма каждого частичного платежа может отличаться, но вместе они должны давать фиксированную суммы из строки предложения.

PSA поддерживает все три типов расписаний счетов для строк предложения с фиксированной ценой.

## <a name="transaction-classification"></a>Классификация проводки

Организации, оказывающие профессиональные услуги, обычно выставляют счета клиентам по классификации затрат. В PSA затраты представлены следующими классификациями транзакций.

- **Время** — эта классификация представляет затраты на оплату труда или времени сотрудников на работу по проекту.
- **Расходы**: — эта классификация представляет все другие виды расходов по проекту. Поскольку расходы можно широко классифицировать, большинство организаций создает подкатегории, например транспортные расходы, аренда автомобилей, проживание или канцелярские принадлежности.
- **Сборы** — эта классификация представляет различные накладные расходы, штрафы и другие позиции, которые выставляются клиенту. 
- **Налог** — эта классификация представляет суммы налога, которые пользователи добавляют, когда они несут расходы.
- **Материальная транзакция** — эта классификация представляет фактические значения из линеек продуктов в подтвержденном счете по проекту.
- **Этап** — эта классификация используется логикой выставления счетов с фиксированной ценой в PSA.

Одна или несколько из этих классификаций транзакций может быть связана с каждой строкой предложения. Когда предложение реализовано, сопоставление между классификацией транзакции и строкой предложения переносится в строку контракта.
 
> ![Снимок экрана сопоставления типов транзакций строкам предложения и строками контракта](media/basic-guide-5.png)
  
Например, предложение может содержать следующие две строки предложения: 
- Консультационная работа, использующая метод выставления счетов "Время и материалы", где применяются классификации транзакций времени и сборов. Например, за все транзакции времени и сборов для примера проекта **Внедрение Dynamics AX** выставляется счет клиенту на основе использованного времени и материалов. 
- Связанные транспортные расходы, для которых используется метод выставления счетов с фиксированной ценой. Например, счета за все транспортные расходы для примера проекта **Внедрение Dynamics AX** выставляются в фиксированном денежном выражении.

> [!NOTE]
> Сочетание проекта и классификации транзакций **Время**, **Расходы** и **Сборы**, которые связаны со строкой предложения или строками контракта, должно быть уникальным. Если одно и то же сочетание проекта и класса транзакций связано с несколькими строками контракта или строками предложения, PSA, не будет работать правильно.

## <a name="billing-types"></a>Типы выставления счета

Поле **Тип выставления счетов** определяет концепцию возможности оплаты в PSA. Это набор параметров, который имеет следующие возможные значения:

- **Оплачивается** — затраты, понесенные этой ролью/категорией, являются прямыми затратами, которые необходимы для выполнения проекта, и клиент оплатит эту работы. Оплатой можно управлять как по времени и материалам, так и на основе фиксированной цены. Однако сотрудник, который затратил это время, получит соответствующей кредит для своей оплачиваемой загруженности.
- **Не оплачивается** — затраты, понесенные этой ролью/категорией, рассматриваются как прямые затраты, которые необходимы для выполнения проекта, хотя клиент и не признает этот факт и не будет оплачивать эту работу. Сотрудник, который затратил это время, не будет кредитован с оплачиваемой загруженностью за него.
- **Бесплатный** — затраты, понесенные этой ролью/категорией, рассматриваются как прямые затраты, которые необходимы для выполнения проекта, и клиент признает этот факт. Сотрудник, который затратил это время, будет кредитован с оплачиваемой загруженностью за него. Однако счет за эти затраты не выставляется клиенту.
- **Недоступно** — затраты на внутренние проекты, которые не требуют отслеживать доход, отслеживаются с помощью этого параметра.

## <a name="invoice-schedule"></a>Расписание счета

Расписание счета — это последовательность дат, когда выставляются счета для проекта. При желании можно создать расписание выставления счетов в строке предложения в PSA. Каждая строка предложения могут иметь собственное расписание выставления счета. Чтобы создать расписание выставления счетов, необходимо предоставить следующие значения атрибутов:

- Дата начала выставления счета 
- Дата доставки, представляющая дату окончания выставления счетов по проекту
- Периодичность выставления счета

PSA использует эти три значения атрибутов для создания предварительного набора дат, когда выставляются счета.

## <a name="invoice-frequency"></a>Периодичность выставления счета

Периодичность выставления счета — это объект, в котором хранятся значения атрибутов, которые помогают выразить частоту создания счетов. Следующие атрибуты выражают или определяют сущность частоты выставления счетов:

- **Период** — поддерживаются периоды ежемесячно, раз в две недели и еженедельно. 
- **Запусков за период** — для периодов еженедельно и раз в две недели можно определить только один запуск за период. На месячных периодов можно задать от одного до четырех запусков за период. 
- **Дни запуска** — дни, когда должно запускаться выставление счетов. Можно настроить этот атрибут двумя способами:
  - **Рабочие дни** — например, можно указать, что выставление счетов запускается каждый понедельник и каждый второй понедельник. Клиенты, которые должны задать запуск выставление счетов по рабочим дням, могут предпочитать это вид конфигурации. 
  - **Календарные дни** — например, можно указать, что выставление счетов запускается в каждый седьмой и двадцать второй день каждого месяца. Некоторые организации могут предпочитать это вид настройки, поскольку она помогает гарантировать, что выставление счетов запускается по фиксированному расписанию каждый месяц.
  
### <a name="invoice-schedule-for-a-fixed-price-quote-line"></a>Расписание выставления счетов для строки предложения с фиксированной ценой

Для строки предложения с расценками с фиксированной ценой можно использовать сетку **Расписание счета** для создания этапов выставления счетов, которые равняются сумме строки предложения с расценками.

- Для создания этапов выставления счетов, которые разделены на равные части, выберите частоту выставления счетов, введите дату начала выставления счетов по строке предложения и выберите **Запрошенная дата завершения** для предложения в разделе **Сводка** заголовка предложения с расценками. Затем выберите **Сформировать периодические вехи**, чтобы создать равномерно разделенные этапы на основе выбранной частоты выставления счетов. 
- Для создания этапа выставления счетов общей суммы, создайте веху, затем введите сумму по строке предложения как сумму этапа.
- Чтобы создать этапы выставления счетов, которые основаны на определенных задачах в плане проекта, создайте этап и сопоставьте его с элементом расписания проекта в интерфейсе пользователя этапа выставления счетов.
