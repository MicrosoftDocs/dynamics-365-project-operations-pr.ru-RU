# <a name="expense-management-mobile-workspace"></a>Мобильная рабочая область управления расходами

В этой статье приводятся сведения о мобильной рабочей области **Управление расходами**. Эта рабочая область позволяет пользователям записывать и отправлять квитанцию, чтобы потом прикрепить ее к отчету о расходах. Пользователи также могут быстро создать строку расходов с помощью прикрепленной квитанции, а также создавать свои отчеты о расходах и управлять ими. Кроме того, утверждающие могут использовать мобильную рабочую область **Управление расходами** для просмотра назначенных им отчетов о расходах и утверждения или отклонения этих отчетов о расходах.


Это мобильное рабочее пространство предназначено для использования вместе с мобильным приложением Dynamics 365 Unified Ops.


## <a name="overview"></a>Обзор

Многие организации требуют, чтобы копия квитанции была приложена к отчету о расходах, связанных с поездкой или бизнесом, который сотрудник отправляет на возмещение. Мобильная рабочая область **Управление расходами** позволяет пользователям быстро создавать новые строки расходов на мобильном устройстве по своему выбору, используя прикрепленную фотографию квитанции. Кроме того, пользователи могут сделать снимок квитанции, а затем прикрепить ее к отчету о расходах позже. Сотрудники также могут создавать отчеты о расходах и управлять ими, а затем отправлять их на утверждение и возмещение с помощью своего мобильного устройства.


В частности, мобильная рабочая область **Управление расходами** позволяет пользователям выполнять следующие задачи:

- Сделать фото чека и отправить его в Dynamics 365 Finance. Позже вы можете прикрепить эту фотографию к отчету о расходах позже.
- Отправьте файл как снятую квитанцию. Позже вы можете прикрепить этот файл к отчету о расходах.
- Создайте новую строку расходов, используя прикрепленную квитанцию. Позже вы можете добавить эту строку в отчет о расходах и отправить ее на утверждение и возмещение.

Можно также использовать следующие функции:

- Создайте новый отчет о расходах.
- Прикрепите к отчету о расходах транзакции по кредитным картам и другие ранее созданные расходы.
- Создайте новые расходы для отчета о расходах.
- Прикрепите квитанцию к любым расходам для отчета о расходах, сфотографировав квитанцию или отправив файл в качестве полученной квитанции.
- В зависимости от политики компании в отношении расходов добавьте список гостей к расходам.
- Детализируйте расходы в зависимости от политики компании в отношении расходов.
- Отправьте отчет о расходах на утверждение и возмещение.
- Утверждайте или отклоняйте отчеты о расходах, для которых вы назначены в качестве утверждающего.

## <a name="capture-a-receipt-by-using-the-expense-management-mobile-workspace"></a>Получение чека с помощью мобильной рабочей области "Управление расходами"

1. На мобильном устройстве откройте рабочую область **Управление расходами**.
2. Выберите **Получение квитанции**.
3. Выберите **Сфотографировать** или **Выбрать изображение**.
4. Выполните одно из перечисленных ниже действий.

    - Если вы выбрали пункт **Сфотографировать**, выполните следующие действия:

        1. Вы будете переключены на камеру вашего мобильного устройства, чтобы вы могли сфотографировать чек. Когда вы закончите делать фотографию, выберите **ОК**, чтобы принять фотографию.
        2. Необязательно: введите имя фотографии и введите заметки.

    - Если вы выбрали пункт **Выбрать изображение**, выполните следующие действия:

        1. Выберите изображение в списке.
        2. Необязательно: введите имя изображения и введите заметки.

5. Нажмите кнопку **Готово**.

## <a name="quickly-enter-expenses-by-using-the-expense-management-mobile-workspace"></a>Быстрый ввод расходов с помощью мобильной рабочей области "Управление расходами"
1. На мобильном устройстве откройте рабочую область **Управление расходами**.
2. Выберите **Быстрый ввод расходов**.
3. Выберите категорию для расхода. Вы увидите список категорий расходов, которые загружены в ваше приложение для автономного использования. По умолчанию загружено 50 элементов, но разработчик может изменить это число. Для получения дополнительной информации разработчики должны обратиться к разделу [Мобильная платформа](/dynamics365/fin-ops-core/dev-itpro/mobile-apps/platform/mobile-platform-home-page). Если вашей категории нет в списке, выберите **Поиск** для выполнения сетевого поиска. Выполните поиск по категории расходов или переключитесь на поиск по типу расходов.
4. Введите дату транзакции расхода.
5. Необязательно: введите продавца для расхода.
6. Введите сумму расхода.
7. Выберите валюту расхода. Вы увидите список кодов валют, которые загружены в ваше приложение для автономного использования. По умолчанию загружено 400 валют, но разработчик может изменить это число. Для получения дополнительной информации разработчики должны обратиться к разделу [Мобильная платформа](/dynamics365/fin-ops-core/dev-itpro/mobile-apps/platform/mobile-platform-home-page). Если вашей валюты нет в списке, выберите **Поиск** для выполнения сетевого поиска. Выполните поиск по валюте или переключитесь на поиск по названию.
8. Выберите **Сфотографировать** или **Выбрать изображение**.
9. Выполните одно из перечисленных ниже действий.

    - Если выбран пункт **Сфотографировать**, вы будете переключены на камеру вашего мобильного устройства, чтобы вы могли сфотографировать чек. Когда вы закончите делать фотографию, выберите **ОК**, чтобы принять фотографию.
    - Если вы выбрали **Выбрать изображение**, выберите изображение в списке.

10. Нажмите кнопку **Готово**.

## <a name="approve-an-expense-report-by-using-the-expense-management-mobile-workspace-if-you-use-the-july-2017-update"></a>Утверждение отчета о расходах с помощью мобильной рабочей области "Управление расходами" (если вы используете обновление за июль 2017 г.)
1. На мобильном устройстве откройте рабочую область **Управление расходами**.
2. **Утверждения расходов** показывают количество отчетов о расходах, которые назначены вам на утверждение. Количество обновляется примерно каждые 30 минут. Выберите **Утверждения расходов**.

    Отображается список отчетов о расходах, которые назначены вам на утверждение.
    
3. Выберите отчет о расходах, чтобы просмотреть подробную информацию о расходах.
4. Выберите расход, чтобы просмотреть подробные сведения для него. Информация, отображаемая для расходов, включает все детали квитанции, гостя и детализации.
5. Вернувшись на страницу **Отчет о затратах**, выберите утверждение или отклонение отчета о расходах.
6. Введите любые комментарии к действию утверждения.
7. Нажмите кнопку **Готово**.

## <a name="create-a-new-expense-report-and-submit-it-for-approval-by-using-the-expense-management-mobile-workspace-if-you-use-the-july-2017-update"></a>Создание нового отчета о расходах и отправке его на утверждение с помощью мобильной рабочей области "Управление расходами" (если вы используете обновление за июль 2017 г.)
1. На мобильном устройстве откройте рабочую область **Управление расходами**.
2. Выберите **Ввод расходов**.
3. Выберите **Создать отчет** или выберите существующий отчет о расходах в списке.
4. Для новых отчетов о расходах введите цель и любую доступную дополнительную информацию. Эта информация различается в зависимости от того, как настроено управление расходами в вашей компании.
5. Нажмите кнопку **Готово**.
6. Чтобы добавить в отчет о расходах существующие расходы, например операции по кредитной карте, выберите **Вложить**.
7. Выберите один или несколько расходов в списке.
8. Нажмите кнопку **Готово**.
9. Чтобы добавить новый расход в отчет о расходах, выберите **Создать расход**.
10. Выберите категорию для расхода. Вы увидите список категорий расходов, которые загружены в ваше приложение для автономного использования. По умолчанию загружено 50 элементов, но разработчик может изменить это число. Для получения дополнительной информации разработчики должны обратиться к разделу [Мобильная платформа](/dynamics365/fin-ops-core/dev-itpro/mobile-apps/platform/mobile-platform-home-page). Если вашей категории нет в списке, выберите **Поиск** для выполнения сетевого поиска. Выполните поиск по категории расходов или переключитесь на поиск по типу расходов.
11. Необязательно: введите продавца для расхода.
12. Введите дату транзакции расхода.
13. Введите сумму расхода.
14. Выберите валюту расхода. Вы увидите список кодов валют, которые загружены в ваше приложение для автономного использования. По умолчанию загружено 400 валют, но разработчик может изменить это число. Для получения дополнительной информации разработчики должны обратиться к разделу [Мобильная платформа](/dynamics365/fin-ops-core/dev-itpro/mobile-apps/platform/mobile-platform-home-page). Если вашей валюты нет в списке, выберите **Поиск** для выполнения сетевого поиска. Выполните поиск по валюте или переключитесь на поиск по названию.
15. Нажмите кнопку **Готово**.
16. Чтобы добавить дополнительные сведения к расходу, выберите **Добавить дополнительные сведения**. Доступные поля зависят от конфигурации управления расходами в вашей компании.
17. Если в соответствии с политикой компании требуется квитанция о расходах, выберите **Квитанции**, затем выполните следующие действия:

    1. Выберите **Получить квитанцию** или **Прикрепить квитанцию**.
    2. Выполните одно из перечисленных ниже действий.

        - Если вы выбрали пункт **Получить квитанцию**, выполните следующие действия:

            1. Выберите **Сфотографировать** или **Выбрать изображение**.
            2. Выполните одно из перечисленных ниже действий.

                - Если вы выбрали пункт **Сфотографировать**, выполните следующие действия:

                    1. Вы будете переключены на камеру вашего мобильного устройства, чтобы вы могли сфотографировать чек. Когда вы закончите делать фотографию, выберите **ОК**, чтобы принять фотографию.
                    2. Необязательно: введите имя фотографии и введите заметки.

                - Если вы выбрали пункт **Выбрать изображение**, выполните следующие действия:

                    1. Выберите изображение в списке.
                    2. Необязательно: введите имя изображения и введите заметки.

            3.  Нажмите кнопку **Готово**.

        - Если вы выбрали пункт **Прикрепить квитанцию**, выполните следующие действия:

            1.  Выберите в списке одно или несколько изображений.
            2.  Нажмите кнопку **Готово**.

    3. Выберите кнопку **Назад**, чтобы вернуться к сведениям о расходах.

18. Если в соответствии с политикой компании требуются гости для расходов, выберите **Гости**, затем выполните следующие действия:

    1. Выберите **Гость**, **Предыдущие гости** или **Коллеги**.
    2. Выполните одно из перечисленных ниже действий.

        - Если вы выбрали пункт **Гости**, выполните следующие действия:

            1. Введите имя гостя.
            2. Необязательно: введите организацию и/или страну гостя.
            3. Необязательно: введите должность гостя.
            4. Нажмите кнопку **Готово**.

        - Если вы выбрали пункт **Предыдущие гости**, выполните следующие действия:

            1. Выберите в списке одного или несколько предыдущих гостей. Вы увидите список предыдущих гостей, которых вы добавили в предыдущие отчеты о расходах, которые загружены в ваше приложение для автономного использования. По умолчанию загружено 50 элементов, но разработчик может изменить это число. Для получения дополнительной информации разработчики должны обратиться к разделу [Мобильная платформа](/dynamics365/fin-ops-core/dev-itpro/mobile-apps/platform/mobile-platform-home-page). Если вашего предыдущего гостя нет в списке, выберите **Поиск** для выполнения сетевого поиска. Выполните поиск по названию или переключитесь на поиск по организации, стране или должности.
            2. Нажмите кнопку **Готово**.

        - Если вы выбрали пункт **Коллеги**, выполните следующие действия:

            1. В списке выберите одного или нескольких коллег. Вы увидите список коллег, которые загружены в ваше приложение для автономного использования. По умолчанию загружено 50 элементов, но разработчик может изменить это число. Для получения дополнительной информации разработчики должны обратиться к разделу [Мобильная платформа](/dynamics365/fin-ops-core/dev-itpro/mobile-apps/platform/mobile-platform-home-page). Если вашего коллеги нет в списке, выберите **Поиск** для выполнения сетевого поиска. Выполните поиск по названию или переключитесь на поиск по компании или должности.
            2. Нажмите кнопку **Готово**.

    3. Выберите кнопку **Назад**, чтобы вернуться к сведениям о расходах.

19. Если в соответствии с политикой компании требуется детализация расхода, выберите **Детализировать**, затем выполните следующие действия:

    1. Выберите первую дату для детализации.
    2. Выберите **Добавить детализацию**.
    3. Выберите подкатегорию для детализации расходов. Вы увидите список подкатегорий расходов, которые загружены в ваше приложение для автономного использования. По умолчанию загружено 50 элементов, но разработчик может изменить это число. Для получения дополнительной информации разработчики должны обратиться к разделу [Мобильная платформа](/dynamics365/fin-ops-core/dev-itpro/mobile-apps/platform/mobile-platform-home-page). Если вашей подкатегории нет в списке, выберите **Поиск** для выполнения сетевого поиска. Выполните поиск по названию подкатегории расходов.
    4. Введите сумму транзакции для детализации.
    5. При необходимости измените дату транзакции.
    6. Нажмите кнопку **Готово**.
    7. Повторяйте предыдущие шаги, пока не закончите добавлять все элементы детализации на выбранную дату.
    8. Для дополнительных дней вы можете выбрать **Копировать на следующий день**, чтобы скопировать детализацию на следующий день. Кроме того, вы можете выбрать дату для детализации, а затем добавить элементы детализации, как вы это делали для первой даты.
    9. После того, как вы закончите детализацию расхода, выберите кнопку **Назад**, чтобы вернуться к сведениям о расходах.

20. Выберите кнопку **Назад**, чтобы вернуться на страницу **Отчет о расходах**.
21. Повторяйте предыдущие шаги, пока не закончите добавлять все расходы.
22. Выберите **Отправить**.
23. Введите любые комментарии для утверждающего.
24. Нажмите кнопку **Готово**.
