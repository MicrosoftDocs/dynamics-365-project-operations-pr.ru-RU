# <a name="financial-estimation-concepts"></a>Понятия финансовой оценки

_**Относится к:** Project Operations для сценариев на основе ресурсов/нескладируемых запасов, упрощенное развертывание — от сделки до выставления счетов-фактур_

В Dynamics 365 Project Operations можно финансово оценить свои проекты в два этапа: 
1. На этапе предпродажной подготовки до заключения сделки. 
2. На стадии исполнения после создания контракта проекта. 

Вы можете создать финансовую оценку для работы на основе проектов, используя любую из следующих 3 страниц:
- На странице **Строка предложения с расценками**, используя сведения строки предложения с расценками.  
- На странице **Строка контракта проекта**, используя сведения строки контракта. 
- На странице **Проект**, используя страницы вкладок **Задачи** или **Оценки расходов**.

## <a name="use-a-project-quote-to-create-an-estimate"></a>Использование предложения с расценками по проекту для создания оценки
В предложении с расценками на основе проекта можно использовать сущность **Сведения строки предложения с расценками** для оценки работы, которую необходимо выполнить по проекту. Затем можно поделиться этой оценкой с клиентом.

Строки предложения с расценками на основе проекта могут иметь от нуля до нескольких сведений о строках предложений с расценками. Сведения строки предложения с расценками используются для оценки времени, расходов и выплат. Microsoft Dynamics 365 Project Operations не допускает материальных оценок в сведениях строки предложения с расценками. Они называются классами транзакции. Оценку сумм налога можно также ввести в класс транзакций.

В дополнение к классам транзакции, сведения строки предложения с расценками имеет тип транзакции. Два типа транзакций поддерживаются для сведений строки предложения с расценками: **Стоимость** и **Контракт по проекту**.

## <a name="use-a-project-contract-to-create-an-estimate"></a>Использование контракта проекта для создания оценки

Если вы использовали предложение с расценками при создании контракта на основе проекта, оценка, которая была выполнена для каждой строки предложения с расценками по предложению с расценками, копируется в контракт по проекту. Структура контракта по проекту аналогична структуре предложения с расценками, которое имеет строки, сведения по строкам и расписания выставления счетов.

Оценки можно вносить непосредственно в контракт по проекту, как в предложение с расценками проекта. Для предложения проекта оценка осуществляется с помощью строк контракта и сведений строк контракта. Сведения строк контракта также могут создаваться из плана проекта, который был создано с помощью подхода оценки "снизу вверх".

Сведения строки контракта могут использоваться для оценки времени, расходов или сборов. Оценку сумм налога можно также ввести в сведения строки контракта.

Материальные оценки не допускаются в сведениях строки контракта.

## <a name="use-a-project-to-create-an-estimate"></a>Использование проекта для создания оценки 

Можно оценить время и затраты на проекты. Project Operations не поддерживает оценки материалов или сборов по проектам.

Оценки времени создаются, когда вы создаете задачу и определяете атрибуты универсального ресурса, который необходим для выполнения этой задачи. Оценки времени создаются из запланированных задач. Оценки времени не создаются при создании универсальных участников рабочей группы вне контекста расписания.

Оценки расходов вводятся в сетку на странице **Оценки расходов**.

Создание оценки для проекта рекомендуется, поскольку вы можете создавать восходящие подробные оценки трудозатрат или времени и расходов по каждой задаче в плане проекта. Затем вы можете использовать эту подробную оценку для создания оценок для каждой строки предложения с расценками и создания более надежного предложения с расценками для клиента. Когда вы импортируете или создаете подробную оценку в строке предложения с расценками с использованием плана проекта, Project Operations импортирует значения продаж и значения стоимости этих оценок. После импорта вы можете просмотреть показатели рентабельности, выгоды и технико-экономические показатели предложения с расценками по проекту.

## <a name="customize-the-quote-line-detail-and-contract-line-detail-entities"></a>Настройка сведения сущностей строки предложения с расценками и сведений строки контракта

Если вы добавили настраиваемое поле в сведения строки предложения с расценками и хотите, чтобы система вводила значение поля в качестве значения по умолчанию в связанной строке затрат, которую она создает, используйте средства регистрации подключаемых модулей **PreOperationContractLineDetailUpdate** и **PreOperationQuoteLineDetailUpdate**. Этот подключаемые модули необходимо заново зарегистрировать после изменения сведений строки предложения с расценками и сведений строки контракта. Чтобы завершить этот процесс, выполните следующие шаги.

1. Откройте PluginRegistrationTool и подключитесь к вашему экземпляру в сети.
2. Выбор **Поиск** и выполните поиск подключаемого модуля для обновления.
3. Выберите подключаемый модуль, затем, на главной странице, щелкните **Выбрать**.
4. Выберите шаг подключаемого модуля для обновления, щелкните правой кнопкой мыши, затем выберите **Обновить**.
5. В диалоговом окне **Обновить существующий шаг** в поле **Атрибуты фильтрации** выберите кнопку с многоточием (**...**):
6. В диалоговом окне **Выберите атрибуты** установите флажки для настраиваемых атрибутов.
7. Выберите **ОК**, чтобы закрыть диалоговое окно, затем выберите **Обновить шаг**.
8. Повторите шаги с 1 по 7 для второго подключаемого модуля.
9. Закройте **PluginRegistrationTool**.
