# <a name="create-vendor-invoices"></a>Создание накладных поставщиков

_**Относится к:** Project Operations для сценариев на основе ресурсов/без запасов_

Для использования функциональности, описанной в этой статье, необходимо включить функцию **Включение обработку фактических данных по субподряду с помощью Project Operations для сценариев на основе ресурсов** в рабочей области **Управление функциями**.

Выставление счетов поставщиками в Microsoft Dynamics 365 Project Operations можно использовать для учета затрат на оказанные поставщиками услуги и/или поставленные ими материалами в рамках проекта.

Когда услуги и/или материалы передаются на субподряд поставщику, субподряд ппредставляет собой договорное соглашение с этим поставщиком. Когда поставщик оказывает услуги или поставляет материалы, которые используются в задачах проекта, в проекте учитываются затраты. Затем поставщик периодически отправляет счета. Эти счета, которые проверяются и сопоставляются с затратами, учтенными в проекте. После завершения процесса проверки счет поставщика подтверждается и отправляется на оплату.

## <a name="scenarios-for-use"></a>Сценарии использования

Счета поставщиков в Project Operations можно использовать для поддержки двух разных сценариев:

- Клиенты полностью используют интерфейс для работы с субподрядами.
- Клиенты полностью используют интерфейс для работы с субподрядами, но хотят иметь единое представление затрат по проектам в Project Operations.

### <a name="customers-use-the-full-subcontracting-experiences"></a>Клиенты полностью используют интерфейс для работы с субподрядами

Интерфейсы для работ со счетами поставщиков дают возможность проверять записи времени, использование материалов и записи расходов, которые ссылаются на отданные на субподряд компоненты, и сопоставлять их со строками счета поставщика. Этот процесс можно использовать для проверки точности строк счета поставщика. После завершения процесса проверки и подтверждения счета поставщика система сторнирует фактические значения, учтенные утвержденными записями времени, расходов и журналами использования материалов, и затем создает новые фактические значения затрат путем использования строк счета поставщика.

### <a name="customers-dont-use-the-full-subcontracting-experiences-but-want-to-have-a-unified-view-of-costs-on-projects-in-project-operations"></a>Клиенты полностью используют интерфейс для работы с субподрядами, но хотят иметь единое представление затрат по проектам в Project Operations

Если вы отслеживаете процесс субподряда в сторонней системе, вы можете учитывать затраты из этой сторонней системы в Project Operations путем создания счетов поставщиков, которые не ссылаются на субподряды. Таким образом ваши руководители проектов будут иметь единое унифицированное представление обо всех затратах по конкретному проекту.

## <a name="create-vendor-invoices-in-project-operations"></a>Создание счетов поставщиков в Project Operations

Счета поставщика создаются в Dynamics 365 Finance с помощью модуля **Расчеты с поставщиками**. Вы не можете создавать счета поставщика непосредственно в Dataverse.

### <a name="set-up-vendor-invoice-verification"></a>Настройка проверки накладных поставщика

Если счет поставщика предназначен для субподряда в Dataverse, вы должны включить параметр **Требуется ручная проверка менеджером проекта**. Задание этого параметра определяет, должен ли счет поставщика автоматически подтверждаться в Dataverse или требует ручного подтверждения. Заголовок каждого счета-фактуры поставщика проекта включает одноименную опцию. По умолчанию для параметра в заголовке всех счетов поставщиков проекта установлено значение, которое вы установили здесь. Однако при необходимости можно обновить значение в заголовке отдельных счетов-фактур поставщиков.

Если для этого параметра установлено значение **Да**, счет поставщика, созданный в Finance и синхронизированный с Dataverse, имеет состояние **Черновик**. Затем счет-фактура должен быть проверен и подтвержден руководителем проекта, затем утвержден, прежде чем он будет обработан и учтен в конкретном проекте и учетных записях главной книги в Finance.

Если для этого параметра установлено значение **Нет**, счет-фактура поставщика подтверждается автоматически. Никакого дальнейшего действия не требуется.

Чтобы настроить проверку накладных поставщиков, выполните следующие действия.

1. В Finance перейдите в раздел **Управление и учет по проектам** \> **Настройка** \> **Параметры модуля "Управление и учет по проектам"**.
1. На вкладке **Финансы** установите для параметра **Требуется ручная проверка менеджером проекта** значение **Да**, если политика компании требует проверки счетов поставщиков-субподрядчиков. Если проверка менеджером проекта не требуется в Dataverse, оставьте для параметра значение **Нет**. По умолчанию настройка будет использоваться на странице **Ожидающий счет поставщика**.

> [!NOTE]
> Счета-фактуры поставщиков, созданные для субконтрактов в Dataverse, могут быть обработаны корректно только в том случае, если для параметра **Требуется ручная проверка менеджером проекта** установлено значение **Да**. Фактические затраты времени, материалов и расходы, которые создают субподрядчики, могут быть вручную сопоставлены со строками накладной поставщика руководителем проекта только в том случае, если для этого параметра установлено значение **Да**.

### <a name="set-up-a-procurement-integration-account-for-vendor-invoices"></a>Настройка учетной записи интеграции закупок для счетов поставщиков

Когда накладная поставщика разносится в Finance for Project Operations — интегрированная среда (без запасов), финансовые показатели разносятся по учетной записи интеграции «Закупки». Система формирует фактические данные в Dataverse для выставленного счета. Эти фактические значения разносятся в Finance с помощью журнала интеграции проекта. При разноске журнала интеграции проекта разносятся фактические затраты и сторнируется счет интеграции "Закупки".

Чтобы настроить учетную запись интеграции закупок для счетов поставщиков, выполните следующие действия.

1. В Finance перейдите в раздел **Управление и учет по проектам** \> **Настройка** \> **Параметры модуля "Управление и учет по проектам"**.
1. На вкладке **Project Operations в Dynamics 365 Customer Engagement** выберите **Материалы** \> **Учетная запись интеграции закупок**.

### <a name="create-and-post-subcontract-vendor-invoices"></a>Создание и разноска счетов поставщиков-субподрядчиков

Когда клерк по расчетам с поставщиками получает счет от субподрядчика, новый счет создается в Finance.

1. В Finance выберите **Расчеты с поставщиками** \> **Накладные** \> **Накладные поставщиков, ожидающие обработки**.
1. На **Панель действий** выберите **Создать** для создания счета-фактуры поставщика.
1. В шапке счета-фактуры в поле **Организация накладной** выберите **Субподрядчик**.
1. Выберите дату накладной.
1. На вкладке **Заголовок** установите для параметра **Требуется ручная проверка менеджером проекта** значение **Да**. Настройка по умолчанию для этого параметра берется со страницы **Параметры модуля "Управление и учет по проектам"**. Однако его можно изменить на уровне накладной поставщика.
1. На экспресс-вкладке **Строка счета** выберите **Добавить строку** для создания строки накладной поставщика.
1. Выберите категорию закупок, созданную для строк субподряда, и введите цену за единицу, единицу измерения и количество.
1. В разделе строк накладной поставщика на вкладке **Проект** выберите проект, для которого субподрядчик выставляет накладную субподряда.
1. Выберите категорию проекта. Это может быть любой тип из **Номенклатура**, **Расходы** , **Материалы** или **Часы**.
1. Выбирайте роль только в том случае, если выбранная категория проекта относится к типу **Час**.
1. Выберите **Разнести**, чтобы разнести накладную поставщика.

Кроме того, вы можете создать счет-фактуру поставщика, перейдя в раздел **Расчеты с поставщиками** \> **Счета** \> **Открытые счета поставщиков** и выбрав **Создать**.

После разноски накладной поставщика она становится доступной в Dataverse для проверки руководителем проекта и обработки.

## <a name="vendor-invoice-processing-in-dataverse"></a>Обработка счетов поставщиков в Dataverse

В накладной поставщика, созданной в Dataverse, некоторые значения полей берутся из накладной поставщика, записанной в Finance. Другие значения являются значениями по умолчанию, которые берутся из субконтракта.

Следующие поля и связанные записи будут скопированы из субподряда в заголовок счета поставщика:

- Валюта.
- Единица по контракту
- Условия оплаты

В строках накладной поставщика Project Operations использует следующие значения полей для ввода субконтракта по умолчанию и ссылки на строку субконтракта:

- Класс проводки
- Роль
- Категория проводки
- Поля продукта
- Project
- Задача

> [!NOTE]
> Строки субподряда с фиксированной ценой не поддерживаются в Project Operations.
