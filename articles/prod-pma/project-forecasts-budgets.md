# <a name="project-forecasts-and-budgets"></a>Прогнозы и бюджеты проектов

[!include [banner](../includes/banner.md)]

Есть два способа управления и контроля проектов: прогнозы проектов и бюджеты проектов. 

Используйте прогнозирование, если у вашей организации есть операционная перспектива и она нацелена на доходы и расходы, полученные от конкретных транзакций. Если ваша организация больше фокусируется на финансовых суммах, вы можете использовать бюджетирование. 

И прогнозы проектов, и бюджеты проектов используют модели прогнозов для хранения предполагаемых сумм транзакций. 

У каждого метода свои преимущества. Прежде чем выбрать метод для своей организации, вам следует учесть следующие моменты.

|   Метод выделения       |           Прогнозирование проекта            |        Бюджетирование проекта                           |
|---------------------------|------------------------------------------|----------------------------------------------------|
| **Распределение периода**     | Вы не можете явно распределить транзакции по финансовым периодам. Вместо этого прогноз и управление прогнозом основаны на сроке службы проекта. Поскольку прогнозы основаны на определенной дате, вы должны определить период, исходя из этой даты. | Вы не можете явно распределить транзакции по всему проекту или финансовому периоду. Если вы распределяете в течение определенного периода, вы можете перенести неиспользованные суммы на следующий финансовый период. |
| **Просмотр транзакций**  | Вы можете просматривать транзакции в формах прогнозов, где вы видите прогнозы для всей компании и для всех проектов, независимо от иерархии. Чтобы сосредоточиться на конкретном проекте, вы должны отфильтровать данные.                                       | Вы можете просмотреть бюджетные транзакции для отдельной иерархии проекта. Таким образом, вы можете просматривать детали транзакции для родительского проекта или его вложенных проектов.                 |
| **Переменные транзакции** | При вводе транзакций прогноза можно использовать все атрибуты, существующие для фактической транзакции. Это позволяет сделать прогноз более подробным. Например, вы можете ввести подробные сведения о количествах, работниках, номенклатурах или свойствах строк.         | Когда вы вводите детали бюджета, вы можете использовать только суммы, категории и действия.                    |
| **Безопасность**              | Прогнозирование основано на транзакциях, которые вы вводите в формы прогноза, и не включает механизм управления процессом. Любой работник, у которого есть разрешения для формы прогноза, может изменять информацию без одобрения.                                        | Бюджетирование использует систему рабочего процесса, которая позволяет управлять изменениями и позволяет вам вести историю изменений.         |
| **Типы записей**           | Записи транзакций прогноза основаны на количестве единиц, а также на стоимости и цене продажи единицы.  | Детали бюджета основаны на суммах, которые разделены между расходами и доходами.                                          |

| **Перерасход средств**         | Вы можете разрешить или запретить ввод только тех транзакций, которые вызовут перерасход средств.   | Составление бюджета проекта предоставляет пользователям дополнительные возможности управления. Вы можете разрешить предупреждения и перерасход.                    |
| **Элемент управления**               | Управление прогнозом осуществляется с помощью уменьшения прогноза. ## <a name="project-forecasts"></a>Прогнозы проекта
При использовании прогнозирования проекта можно вводить транзакции прогноза в формы прогноза для каждого типа транзакции. Каждый атрибут, доступный для фактической транзакции, может использоваться для прогнозируемой транзакции, например, прибыльность строки, атрибуты строки, работники или описания. Вы также можете спрогнозировать, через какое время после несения затрат вы будете выставлять счет клиенту. 

Операции прогнозирования проекта основаны на единицах и суммах. 

Вы должны связать каждый прогноз проекта с моделью прогноза. При вводе транзакции прогноза автоматически предлагается модель прогноза. Модель прогноза действует как контейнер для транзакций прогноза. 

Вы можете обозначить модели прогноза как вложенные модели для модели. Это позволяет делать прогнозы по региону, периоду времени или отделу. Вы можете скопировать транзакции прогноза в модель, чтобы создать новую модель, а также можете перенести транзакции в главную книгу. Поскольку между прогнозом и моделью существует взаимно однозначная связь, каждая модель прогноза составляет отдельный бюджет для проекта. 

Модели прогнозов могут использовать сокращение прогнозов в качестве механизма управления проектами. При сокращении прогноза фактические транзакции уменьшают прогнозируемые балансы транзакций. Однако, поскольку сокращение прогноза применяется к самому высокому проекту в иерархии, оно обеспечивает ограниченное представление изменений в прогнозе. Например, если работник связан с вложенным проектом, фактические транзакции для работника отправляются в родительский проект. Это может затруднить отслеживание изменений, потому что вы не можете легко определить, какая транзакция в каком вложенном проекте вызвала уменьшение прогнозируемой суммы. Поэтому рекомендуется создать копию модели прогноза для использования при сокращении прогноза. Затем вы можете использовать отчеты для просмотра того, что изначально прогнозировалось. 

Вы можете пересматривать, копировать, удалять или переносить прогнозы проекта в бюджет главной книги. Однако нет никакого технологического контроля. Любой работник, у которого есть разрешение для формы прогноза, может вносить изменения без рассмотрения.

-   **Пересмотреть** — вы можете пересмотреть транзакцию прогноза в тех же формах, где были сделаны исходные записи.
-   **Копировать или удалить** — при копировании транзакций прогноза вы копируете строки транзакции одной модели прогноза в другую модель прогноза. При удалении прогноза вы удаляете транзакции прогноза из модели прогноза. Чтобы ограничить количество операций прогноза, которые копируются или удаляются, выберите определенные типы операций и даты. Это позволяет копировать или удалять только определенные части прогноза.
-   **Перенести** — когда вы переносите прогноз проекта в бюджет главной книги, вы переносите операции прогноза модели прогноза в бюджет главной книги. Вы можете перезаписать любые ранее перенесенные транзакции в бюджете главной книги, в которую вы переносите прогноз проекта.

## <a name="project-budgets"></a>Бюджеты проекта
Составление бюджета по проекту — более простой метод, чем прогнозирование, хотя он интегрируется с моделями прогноза. Оно использует единую форму ввода для исходных сведений и изменений бюджета и позволяет делать прогнозы, основанные только на сумме, категории или деятельности. 

При составлении бюджета проекта все исходные бюджеты и изменения должны отправляться в рабочий процесс проекта для утверждения. Рабочие процессы дают вам больший контроль над процессом и создают запись истории изменений. 

Составление бюджета проекта похоже на составление бюджета главной книги, но его проще и быстрее настроить. Многие параметры бюджетирования в главной книге, такие как номерные серии или валюта, не нужно настраивать отдельно для проектов.

Бюджеты проекта автоматически связываются с двумя моделями прогноза: одной для исходного бюджета и одной для оставшегося бюджета. Следовательно, отчеты, основанные на моделях прогноза, могут использовать данные бюджета. Когда бюджет проекта подтвержден, система создает транзакции прогноза на основе связанных моделей, которые используются для отчетности и контроля.

## <a name="forecast-models"></a>Модели прогнозов
У модели прогнозов одноуровневая иерархия. Это означает, что один прогноз проекта должен быть связан с одной моделью прогноза.

Если вы используете прогнозирование проекта, вы можете идентифицировать модели как вложенные модели. Затем можно делать прогнозы по подразделению, периоду времени или региону. Например, вы можете создать модель прогноза на год, а затем создать вложенные модели для региональных прогнозов северо-востока, юго-востока, северо-запада и юго-запада, которые будут представлены региональными руководителями. Выбирая различные параметры в доступных отчетах, вы можете просматривать информацию по общему прогнозу или по вложенной модели.
