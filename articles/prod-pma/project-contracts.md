# <a name="project-contracts"></a>Контракты по проектам

[!include [banner](../includes/banner.md)]

Эта статья содержит примеры контрактов по проектам, которые вы можете создать для различных типов проектов и источников финансирования, а также способы управления контрактами и выставлением счетов клиентам проекта.

Тип проекта, который вы создаете для контракта по проекту, определяет метод, который используется для выставления счетов клиентам проекта. Вы можете изменить контракт проекта и связанный проект, но не можете изменить тип проекта. 

Используя контракт по проекту, вы можете выставлять счета одновременно для одного или нескольких проектов. Контракт по проекту также помогает гарантировать единообразную процедуру выставления счетов для каждого вложенного проекта в структуре проекта. 

Каждый проект, по которому будет выставлен счет, должен быть связан с контрактом по проекту. Параметры контракта по проекту применяются ко всем проектам и вложенным проектам, связанным с этим контрактом по проекту. 

В контракте по проекту можно указать один или несколько источников финансирования. Таким образом, вы можете разделить выставление счетов между несколькими спонсорами, установить лимиты финансирования, чтобы источники финансирования не выставляли счета на сумму, превышающую указанную, и настроить правила финансирования для начисления расходов.

## <a name="funding-for-project-contracts"></a>Финансирование контрактов по проекту
В некоторых контрактах по проекту указано, что несколько сторон несут ответственность за финансирование проектных затрат. Ниже приведено несколько примеров:

-   Крупный заказчик, у которого есть несколько подразделений, просит разделить финансирование проекта по подразделениям.
-   Ваша компания разделяет расходы на крупный проект с внешней организацией.
-   Дорожный проект совместно финансируется двумя муниципалитетами.
-   Проект моста финансируется за счет государственного гранта и частной корпорации.

В Dynamics 365 Finance вы можете разделить счет за одну транзакцию или весь проект между несколькими клиентами, грантами или организациями. 

В проектах, у которых есть несколько спонсоров, все стороны, которые вносят вклад в финансирование проекта расширенного финансирования, называются источниками финансирования. После того как заказчик, организация или грант определены как источник финансирования, они могут быть назначены одному или нескольким правилам финансирования. Правила финансирования содержат критерии, определяющие, как расходы распределяются между различными источниками финансирования проекта. 

Поскольку номенклатуры в запасах, такие как те, которые указаны в заявках на покупку и заказах на покупку, нельзя разделить, сумма затрат не может быть разделена между несколькими источниками финансирования во время распределения. Следовательно, значение источника финансирования остается 0 (нулем) до разноски отпуска запасов. При разноске расхода запасов сумма затрат распределяется в соответствии с правилами распределения счетов для проекта.

Вот несколько шагов, которые можно предпринять, чтобы упростить разделение счетов между несколькими источниками финансирования:

-   Укажите, что все проводки, вводимые для проекта, используют ту же валюту продаж, что и контракт по проекту.
-   Установите лимиты финансирования, чтобы источник финансирования не выставлял счет для проекта больше указанной суммы.
-   Настройте правила финансирования и лимиты финансирования для каждого работника, элемента, категории, группы категорий и типа транзакции (или для всех типов транзакций).
-   Выберите необязательные даты начала и окончания, чтобы определить период действия каждого правила финансирования.
-   Укажите процент, за который отвечает каждый источник финансирования.
-   Укажите, какой источник финансирования отвечает за разницу в округлении, вызванную расчетами распределения финансирования.
-   Установите правила, которые определяют, как затраты по проекту выставляются внешним клиентам и взимаются с внутренних организаций.
-   Записывайте транзакции на счете отложенного финансирования до тех пор, пока не будет получено дополнительное финансирование или пока вы не решите нести расходы внутри компании.

Чтобы определить, какую налоговую группу нужно связать с транзакцией, в проекте выполняется поиск присвоения налоговой группы. Если присвоение налоговой группы не было выполнено на уровне проекта, выполняется поиск контракта по проекту.

### <a name="example-multiple-funding-sources-complex"></a>Пример: несколько источников финансирования (сложный)

У вас есть три источника финансирования, которые вы хотите использовать в следующем порядке:

1.  Используйте источник финансирования 2 и источник финансирования 3 одинаково, пока источник финансирования 2 не будет исчерпан.
2.  Продолжайте использовать источник финансирования 3, пока он не будет исчерпан.
3.  Используйте источник финансирования 1 после того, как источник финансирования 3 будет исчерпан.

Для достижения этой цели следует сделать следующее:

-   Установите лимиты финансирования для источника финансирования 2 и источника финансирования 3 для соответствующих сумм.
-   Создайте следующие правила финансирования:
    -   Правило 1 (Приоритет 1): распределяйте 50 процентов транзакций на источник финансирования 2 и 50 процентов на источник финансирования 3.
    -   Правило 2 (Приоритет 2): распределяйте 100 процентов транзакций на источник финансирования 3.
    -   Правило 3 (Приоритет 3): распределяйте 100 процентов транзакций на источник финансирования 1.

Эта настройка работает, потому что транзакции проверяются на соответствие правилам и ограничениям, чтобы определить, применимы ли какие-либо из них к транзакции. Если к транзакции не применяются какие-либо особые правила или ограничения, применяется правило «Все транзакции». Правило "Все транзакции" соответствует всем транзакциям. 

Если найдено правило, соответствующее транзакции, сначала применяется процент, выделенный в этом правиле, но только после того, как совпадения будут проверены на соответствие установленным ограничениям. Если лимит был достигнут и средства источника финансирования исчерпаны, правило финансирования, связанное с лимитом финансирования, игнорируется, и программа проверяет следующее применимое правило. 

В некоторых случаях правилом может быть выделена только часть транзакции. Это может произойти из-за того, что при распределении транзакции достигнут предел. В этом случае в соответствии с этим правилом выделяется только определенная сумма, например, 50 процентов на каждый источник финансирования. Так обстоит дело в правиле 1, которое описано ранее в этом разделе. Остаток распределяется согласно следующему правилу в последовательности.

## <a name="billing-rules"></a>Правила выставления счетов
Когда вы ведете переговоры по контракту по проекту с заказчиком, вы определяете, как и когда вы можете выставить заказчику счет за работу над проектом. После настройки контракта по проекту и проекта вы можете настроить правила выставления счетов для проекта. Правила выставления счетов основываются на условиях проекта, указанных в контракте по проекту. Правила выставления счетов, которые вы можете создать, зависят от условий контракта по проекту и типа проекта, такого как "Время и материалы" или "Фиксированная цена", которые вы связываете с правилом выставления счетов. Для контракта по проекту можно создать более одного правила выставления счетов. Вы также можете назначить правило выставления счетов нескольким проектам, которые связаны с одним и тем же контрактом по проекту и с одинаковыми условиями выставления счетов. 

Вы можете настроить следующие типы правил выставления счетов:

-   **Единица доставки** — выставляйте счет-фактуру клиенту, когда вы комплектуете единицу поставки. Вы определяете единицы поставки в контракте.
-   **Ход выполнения** — выставляйте счет заказчику, когда вы выполняете определенный процент проекта. Вы можете настроить правило выставления счетов для автоматического расчета процента выполненных работ или вручную рассчитать процент выполненных работ и сумму для выставления счета клиенту.
-   **Веха** — выставляйте счета заказчику на полную сумму вехи проекта, когда она будет достигнута.
-   **Вознаграждение** — выставляйте клиенту счет за свои услуги плюс вознаграждение за управление, которое обычно составляет процент от стоимости услуг.
-   **Время и материал** — выставляйте заказчику счет на стоимость времени и материалов, которые используются в проекте.

Для всех типов правил выставления счетов вы можете указать процент удержания, который вычитается из счетов клиентов до тех пор, пока проект не достигнет согласованной стадии. Процент удержания платежей указан в контракте по проекту. Сумма рассчитывается на основе общей стоимости строк в счете клиента и вычитается из нее. 

Для правил выставления счетов **Время и материал** и **Ход выполнения** , вы можете назначать оплачиваемые категории. Оплачиваемые категории указывают транзакции, которые должны быть включены в счета-фактуры клиентов. 

Когда вы готовы выставить счет клиенту, сумма счета для проекта рассчитывается на основе правил выставления счетов, и создается предложение счета по проекту. 

В следующих разделах представлены примеры, показывающие, как настроить правила выставления счетов для проекта и управлять ими.

### <a name="example-create-a-billing-rule-that-is-based-on-the-number-of-units-delivered"></a>Пример: создание правила выставления счетов на основе количества доставленных единиц

Ваша организация заключает соглашение о проведении пяти учебных занятий для сотрудников заказчика по цене 10 000 за одно занятие. Вы выставляете счет клиенту после каждого занятия. 

При настройке правил выставления счетов для контракта вы используете следующие значения:

-   Единица доставки — одно обучающее занятие.
-   Цена за единицу — 10 000 за занятие.
-   Общее количество единиц — пять учебных занятий.

По завершении одного занятия вы можете создать счет на 10 000 за первую поставленную единицу продукции и отправить счет клиенту.

### <a name="example-create-a-billing-rule-that-is-based-on-a-specified-percentage-of-project-completion-manual-calculation"></a>Пример: создание правила выставления счетов, основанного на указанном проценте завершения проекта (расчет вручную)

Ваша организация, консалтинговая фирма по программному обеспечению, заключает с заказчиком соглашение о разработке части продукта, который разрабатывает заказчик. Ваша организация соглашается предоставить программный код в течение шести месяцев. Заказчик соглашается заплатить вашей организации в общей сложности 100 000 за работу. Вы создаете правило выставления счетов для выставления счета клиенту на основе процента выполненных работ по проекту, как указано в контракте.

-   В конце первого месяца вы встречаетесь с заказчиком, чтобы определить процент выполненных работ. После того, как вы и заказчик рассмотрите проект, вы решите, что проект выполнен на 15 процентов.
-   Вы создаете счет-фактуру для 15 000 (15 процентов от 100 000) и отправляете его клиенту.

### <a name="example-create-a-billing-rule-that-is-based-on-a-specified-percentage-of-project-completion-automatic-calculation"></a>Пример: создание правила выставления счетов, основанного на указанном проценте завершения проекта (автоматический расчет)

Ваша организация, занимающаяся разработкой программного обеспечения, соглашается разработать пакет учета заработной платы для клиента на 30 000 человек. Заказчик соглашается заплатить вашей организации на основе процента выполненной работы. По вашим оценкам, стоимость проекта составляет 20 000. В контракте по проекту указываются категории работ, которые вы используете в процессе выставления счетов. Вы устанавливаете правила выставления счетов, которые автоматически рассчитывают суммы накладных для процента работы, выполненной для каждой категории. Вы устанавливаете бюджет для каждой категории:

-   **Разработка** — стоимость 15 000, выручка 20 000
-   **Установка** — стоимость 5000, выручка 10 000

Когда вы создаете счет клиента в первый раз, сумма счета автоматически рассчитывается на основе следующей информации:

-   Через месяц работник проекта отправляет расписание для проекта. Стоимость рабочего времени 5000 часов на разработку и 1000 часов на установку. Работы по разработке завершены на 33 процента (фактическая стоимость 5000/бюджетная стоимость 15 000), а работы по установке выполнены на 20 процентов (фактическая стоимость 1000/бюджетная стоимость 5000).
-   Сумма счета-фактуры 8667 вычисляется автоматически (33 процента от 20 000 + 20 процентов от 10 000).
-   Вы создаете счет-фактуру для 8667 и отправляете его клиенту.

### <a name="example-create-a-billing-rule-that-is-based-on-agreed-upon-milestones"></a>Пример: создание правила выставления счетов на основе согласованных вех

Ваша организация, консалтинговая компания по вопросам управления, соглашается провести исследование рынка потребительского продукта, который заказчик планирует продать. Клиент соглашается использовать ваши услуги в течение трех месяцев начиная с марта и соглашается заплатить вашей организации 50 000. У проекта есть три вехи:

-   Веха 1. Сбор данных о потребителях — 31 марта
-   Веха 2. Анализ данных о потребителях — 30 апреля
-   Веха 3. Представление предложения о жизнеспособности продукта — 31 мая

Заказчик соглашается заплатить вашей организации 10 000 за первый этап, 20 000 за второй этап и 20 000 за третий этап. 

Создавая контракт по проекту, вы соглашаетесь выставить счет заказчику на основе выполненной вехи. Настройка правила выставления счетов включает следующие шаги:

-   Определите вехи проекта.
-   Определите сумму, на которую будет выставлен счет клиенту по завершении каждой вехи.

Когда 31 марта завершается первая веха, вы отмечаете ее как завершенную, а затем создаете счет на 10 000 и отправляете его клиенту. Вы не можете создать счет для вехи, пока не отметите веху как завершенную.

### <a name="example-create-a-billing-rule-that-is-based-on-services-plus-a-management-fee"></a>Пример: создание правила выставления счетов, основанного на услугах плюс вознаграждение за управление

Ваша организация, консалтинговая компания по вопросам управления, соглашается провести исследование рынка для оценки жизнеспособности продукта, разрабатываемого клиентом, розничной компанией. В условиях соглашения указано, что вы будете предоставлять услуги трех своих лучших консультантов по вопросам управления, которые будут проводить исследования в соответствии с условиями и временем. Заказчик соглашается платить 100 в час, плюс 10 процентов вознаграждения за консультационные часы, отнесенные к проекту. 

При настройке контракта по проекту создайте правило выставления счетов, чтобы добавить 10-процентное вознаграждение за управление к часам консультаций, взимаемым с проекта. 

Когда вы создаете счет для клиента, ему выставляется счет за управление в размере 10 процентов вознаграждения плюс стоимость часов консультации. Например, если три консультанта проработали в общей сложности 200 часов над проектом, счет на 22 000 создается на основе следующего расчета:

-   200 часов по 100 часов в час = 20 000
-   Вознаграждение за управление 10 процентов = 2000
-   Общая сумма счета = 22 000

Если сборы облагаются налогом для клиента и вы выбираете налоговую группу в контракте по проекту, налоговая группа автоматически вводится в правило выставления счетов для сборов.

### <a name="example-create-a-billing-rule-for-the-value-of-time-and-materials"></a>Пример: создание правила выставления счетов для времени и материалов

Ваша организация, консалтинговая компания по программному обеспечению, соглашается предоставить пять технических консультантов для работы над проектом разработки программного обеспечения для клиента в течение следующих шести месяцев. Заказчик соглашается платить 150 за каждый час консультации, плюс стоимость канцелярских принадлежностей. Ваша организация отправляет клиенту счет в конце каждого месяца. 

При оформлении контракта по проекту вы соглашаетесь ежемесячно выставлять заказчику счет за время и материалы по проекту. Вы создаете правило выставления счетов, которое включает следующую информацию:

-   Срок контракта — шесть месяцев.
-   Время консультации рассчитывается как 150 в час.
-   Счета на канцелярские товары выставляются по себестоимости, и общая стоимость проекта не должна превышать 10 000.
-   Вы создаете счет-фактуру клиента в конце каждого календарного месяца в течение проекта.

В течение первого месяца консультанты по проекту регистрируют в общей сложности 800 часов. Стоимость канцелярских принадлежностей, внесенных в счет проекта, составляет 2000. Таким образом, в конце месяца вы создаете счет-фактуру на 122 000, что рассчитывается как 800 часов из расчета 150 часов в час плюс 2000 для канцелярских принадлежностей.
