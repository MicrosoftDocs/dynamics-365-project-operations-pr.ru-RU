# <a name="manage-project-invoice-proposals"></a>Управление предложениями по счетам по проекту

_**Относится к:** Project Operations для сценариев на основе ресурсов/без запасов_

Предложения по счетам по проекту могут обрабатываться вашим отделом выставления счетов при соблюдении следующих двух условий:

  - Менеджер проекта подтверждает счет-проформу в Microsoft Dataverse.
  - Все проводки продаж без выставления счета с учетом времени и материала, которые включены в счетом-проформу, разносятся с использованием журнала **интеграции Project Operations** Dynamics 365.

Выполните следующие действия, чтобы заполнить предложение по счету по проекту в Dynamics 365 Finance.

1. Просмотрите данные для выставления счетов на предмет проводок с учетом времени и материала и разнесите журнал **Интеграция Project Operations**.
2. Просмотрите данные для выставления счетов для вех выставления счетов с фиксированной ценой.
3. Просмотрите и отформатируйте предложение по счету по проекту.
4. Разнесите и распечатайте счета по проекту.

## <a name="manage-billing-information-in-the-project-operations-integration-journal"></a>Управление данные для выставления счетов в журнале интеграции Project Operations

Фактические данные по проекту созданы в Dataverse проверяются и разносятся в Finance бухгалтером проекта. Дополнительные сведения о работе с журналом интеграции см. в разделе [Журнал интеграции в Project Operations](../project-accounting/project-operations-integration-journal.md).

Фактические данные стоимости и фактические данные продаж без выставленного счета добавляются в журнал интеграции отдельными строками:

  - Себестоимость единицы и сумма затрат по умолчанию фактические данные стоимости из транзакции фактических затрат по проекту в Dataverse.
  - Цена продажи за единицу и сумма продаж по проводки продаж без выставления счета по умолчанию взяты из фактической проводки продаж без выставления счета по проекту в Dataverse.

### <a name="billing-sales-tax"></a>Выставление счетов налог с продаж

Расчет налога с продаж для выставления счетов определяется полями **Налоговая группа выставления счетов** и **Налоговая группа номенклатур выставления счетов** в записи о продажах без выставления счета в строке журнала **интеграции Project Operations**. Система устанавливает значения этих полей по умолчанию на основе настроек на вкладке **Финансы** на страница **Параметры управления и учета по проектам**.

- **Метод налоговой группы** определяет логику налоговой группы выставления счетов по умолчанию:
  
  - **Проект**: всегда будет использоваться по умолчанию налоговая группа выставления счетов из проекта. Вы можете просмотреть или изменить налоговая группа выставления счетов по умолчанию по проекту, выбрав **Показать учет по умолчанию** на страница **Все проекты**.
  - **Контракт по проекту**: всегда будет использоваться по умолчанию налоговая группа выставления счетов из контракта по проекту. Вы можете просмотреть или изменить налоговая группа выставления счетов по умолчанию по контракту по проекту, выбрав **Показать учет по умолчанию** на страница **Контракты по проекту**.
  - **Клиент**: всегда будет использоваться по умолчанию налоговая группа выставления счетов из клиента.
  - **Поиск**: выполнит поиск по всем указанным выше объектам и выберет первое доступное значение. Поиск начинается с объекта **Проект**, затем объект **Контракт по проекту**, и, наконец, объект **Клиент**.

- **Метод налоговой группы номенклатуры** определяет логику налоговой группы номенклатур выставления счетов по умолчанию:
  
  - Для типов транзакций по времени, расходам и сборам налоговая группа номенклатур выставления счетов всегда будет по умолчанию из объекта **Категория проекта**.
  - Для типов транзакций по материалам налоговая группа номенклатур выставления счетов принимает значения по умолчанию на основе настройки **Метод налоговой группы номенклатуры** в **Параметры управления и учета по проектам**. Номер позиции по умолчанию представляет собой налоговую группу номенклатуры выставления счетов из объекта **Выпущенный продукт**. Категория по умолчанию представляет собой налоговую группу номенклатуры выставления счетов из объекта **Категория проекта**.

### <a name="financial-dimensions"></a>Финансовые аналитики

Финансовые аналитики для записей о продажах без выставления счетов в журнале **интеграции Project Operations** по умолчанию из объекта **Проект**. Финансовые аналитики можно просмотреть и скорректировать, выбрав **Распределить суммы**. Если вам нужно изменить финансовые аналитики записи о продажах без выставления счета после разноски журнала интеграции, но до подтверждения счета-проформы, перейдите к **Все проекты** > **Управление** > **Разнесенные транзакции**, выберите транзакцию, а затем выберите **Обработать** > **Корректировка учета**.

### <a name="exchange-rates"></a>Валютные курсы

Валюта проводки без выставления счета в Dataverse используется в качестве валюты транзакции в Finance и преобразуется в валюту учета компании с использованием валютных курсов, определенных в Finance.


## <a name="manage-the-financial-attributes-of-billing-milestones"></a>Управляйте финансовыми атрибутами вех выставления счетов 

Строки контракта по проекту с использованием метода выставления счетов с фиксированной ценой выставляются через [Вехи фиксированных цен](../sales/invoice-schedules-contract-line.md#create-a-fixed-price-invoice-schedule-for-a-contract-line). Бухгалтер проекта может просмотреть вехи выставления счетов в Finance, перейдя к **Управление и учет по проектам** > **Все проекты** > **Управление** > **Проводки по промежуточным накладным**.

### <a name="billing-sales-tax"></a>Выставление счетов налог с продаж

Значения **Налоговая группа** и **Налоговая группа номенклатур** по умолчанию из параметров, когда новая веха выставления счетов создается в Dataverse. Система устанавливает значения этих полей по умолчанию на основе параметров на вкладке **Финансы** на страница **Параметры управления и учета по проектам**.

- **Метод налоговой группы** определяет логику **налоговой группы выставления счетов** по умолчанию:

    - **Проект**: всегда будет использоваться по умолчанию налоговая группа выставления счетов из проекта. Вы можете просмотреть или изменить налоговая группа по умолчанию по проекту, выбрав **Показать учет по умолчанию** на страница **Все проекты**.
    - **Контракт по проекту**: всегда будет использоваться по умолчанию налоговая группа выставления счетов из контракта по проекту. Вы можете просмотреть или изменить налоговая группа выставления счетов по умолчанию по контракту по проекту, выбрав **Показать учет по умолчанию** на страница **Контракты по проекту**.
    - **Клиент**: всегда будет использоваться по умолчанию налоговая группа выставления счетов из клиента.
    - **Поиск**: выполнит поиск по всем указанным в этом списке объектам и выберет первое доступное значение. Поиск начинается с объекта **Проект**, затем объект **Контракт по проекту**, и затем объект **Клиент**.

- **Налоговая группа номенклатуры вехи фиксированных цен** используется по умолчанию для значения поля **Налоговая группа номенклатуры**.

### <a name="financial-dimensions"></a>Финансовые аналитики

Финансовые аналитики по умолчанию для вехи выставления счетов с фиксированной ценой устанавливаются в строках контракта по проекту. Перейти к **Контракты по проекту** > **Показать учет по умолчанию** и на вкладка **Строки контракта** выберите **строка контракта с ценой**, а затем установите значения финансовой аналитики, которые вы хотите использовать по умолчанию.

Бухгалтер проекта может редактировать информацию о налогах с продаж и финансовых аналитиках по вехам выставления счетов до тех пор, пока не будет создано предложение по счету по проекту.


## <a name="create-project-invoice-proposals"></a>Создание предложений по счетам по проекту

Предложения по счетам по проекту можно просмотреть в модуле **Управление и учет по проектам**, перейдя в **Счета по проекту** > **Предложения по счетам по проекту**.

Заголовок предложения по счету по проекту создается в Finance, когда счет-проформа подтверждается в Dataverse. Для упрощения выверки система устанавливает номер предложения по счету по проекту в Finance на тот же номер, что и идентификатор счета-проформы в Dataverse. Поскольку счета-проформы не обязательно подтверждаются в том же порядке, в котором они создаются, номерная последовательность предложения по счету по проекту в Finance должна допускать изменение более низких и более высоких номеров. Настройте номерные серии, выполнив следующие действия:

1. В Finance перейдите к **Администрирование организации** > **Номерные серии** > **Номерные серии**.
2. В фильтр **Область** выберите **Проекты**.
3. В фильтр **Ссылка** выберите **Предложение по счету**.
4. Использовать поле **Компания** для фильтрации каждого юридического лица с включенной интеграцией Project Operations Dataverse.
5. Откройте **Детали номерной серии** и на вкладке **Общее** задайте:

    - **Разрешить изменения пользователя: на меньшее число** = **Да**
    - **Разрешить изменения пользователя: на большее число** = **Да**

Строки предложения по счету по проекту добавляются системой с использованием периодического процесса. **Импорт из промежуточной таблицы** (**Управление и учет по проектам** > **Периодический** > **Интеграция Project Operations** > **Импорт из промежуточной таблицы**). Этот процесс можно запускать вручную или с помощью периодического расписания. Система не будет добавлять строки в документ предложения по счету, пока все строки не будут готовы к выставлению счета. Проводки с учетом времени и материала готовы к выставлению счетов только тогда, когда они разнесены с помощью журнала **интеграции Project Operations**.

## <a name="format-and-print-invoice-proposals"></a>Форматирование и печать предложений по счетам

Бухгалтер проекта может настроить распечатку счета по проекту с помощью страницы **Форматировать предложение по счету** и возможности управления печатью.

### <a name="format-invoice-proposals"></a>Форматирование предложений по счетам

Страница **Форматирование предложений по счетам** позволяет настраивать группировку транзакций для отображения в счете по проекту клиента.

1. На страница **Предложение по счету по проекту** выберите **Печать** > **Форматировать предложение по счету**.
2. Выберите **Создать** для создания новой группирования для распечатки счета по проекту.
3. В поле **Подробно/Сводка** выберите параметры для этого группирования:

    - Выберите **Подробно** для распечатки сведения о транзакции в счете клиента.
    - Выберите **Сводка** для распечатки сводки по транзакции в счете клиента.

> [!NOTE]
> Выбор в поле **Подробно/Сводка** на странице **Форматировать предложение по счету** переопределяет параметр, выбранный в поле **Формат счета** на странице **Предложения по счетам**, чтобы распечатать подробный счет или сводный счет.

- Выберите строки проводки для включения в этот раздел на вкладка **Доступные транзакции** и выберите **Включить транзакции**, чтобы переместить их на вкладку **Выбранные транзакции**.
- Выберите **Переместить вверх** и **Переместить вниз**, чтобы изменить порядок разделов.
- Выберите **Предварительный просмотр** для предварительного просмотра отформатированного счета.

### <a name="print-management"></a>Управление печатью

Управление печатью использует различные файлы отчетов для печати, указания адресатов и настройки текста нижнего колонтитула для счета. Управление печатью можно настроить на уровне модуля, однако эти настройки можно переопределить для конкретного клиента, контракта или предложения по счету. Чтобы получить доступ к этой функции на странице **Предложение по счету по проекту**, выберите **Печать** > **Управление печатью**.

Настройка управления печатью отображается в виде дерева, где на каждом уровне узла отображаются доступные для настройки документы. Настраиваемые распечатки можно назначить на уровне модуля, клиента, контракта или документа предложения по счету. Чтобы изменить исходную распечатку документа, разверните нужный узел и выберите **Исходный элемент**. В поле **Формат отчета** выберите формат отчета, который будет использоваться для печати. Вы можете использовать настраиваемые форматы отчетов, используя [Платформу управления бизнес-документами](https://docs.microsoft.com/dynamics365/fin-ops-core/dev-itpro/analytics/er-business-document-management).

## <a name="post-invoice-proposals"></a>Разноска предложений по счетам

После того, как счет будет просмотрен и отредактирован, а строки предложения по счету признаны удовлетворительными, проверьте итоговые суммы счета и налог с продаж. В группе **Подробно**, выберите **Итоги** а затем выберите **Разнести**, чтобы разнести счет.

Чтобы просмотреть счет перед разноской, снимите флажок **Разноска**. Надпись **Проформа** будет напечатана на счете, чтобы указать, что это образец счета. Чтобы распечатать счет, установите флажок **Распечатать счет**.

В добавок к странице **Предложение по счету**, предложения по счетам также можно разнести, выполнив периодическое задание, **Разноска предложений по счетам**. Чтобы найти это задание, перейдите к **Управление и учет по проектам** > **Периодический** > **Счета по проекту** > **Разноска предложений по счетам**.

На этой странице показаны все предложения по счетам, готовые к разноске. Вы можете запланировать разноску предложений по счетам, выбрав **Пакет**. Установите **Параметр пакетной обработки** на **Да** и задайте повторение пакетной обработки, выбрав **Повторение**.
