# <a name="create-corrective-project-based-invoices"></a>Создание корректирующих счетов на основе проектов 

_**Относится к:** Project Operations для сценариев на основе ресурсов/без запасов_

Подтвержденный счет по проекту может быть скорректирован для обработки изменений или кредитов по согласованию с заказчиком и менеджером проекта.

Чтобы внести изменения в подтвержденный счет, откройте подтвержденный счет и выберите **Исправить этот счет**. 

> [!NOTE]
> Этот выбор недоступен, если счет по проекту не подтвержден.

Новый черновик счета создается из подтвержденного счета. Все детали строк счета из ранее подтвержденного счета копируются в новый черновик. Ниже приведены некоторые ключевые моменты, которые помогут вам лучше понять сведения строки в новом исправленном счете:

- Все количества обновляются до нуля. Предполагается, что все номенклатуры с выставленными счетами, полностью кредитуются. При необходимости вы можете вручную обновить эти количества, чтобы отразить количество, по которому выставляется счет, а не количество, которое кредитуется. На основе введенного количества приложение рассчитывает кредитуемое количество. Эта сумма отражается в фактических данных, которые создаются при подтверждении исправленного счета. Если вы вносите изменения в сумму налога, вы должны ввести правильную сумму налога, а не сумму налога, которая кредитуется.
- Исправления вех всегда обрабатываются как полные кредиты.
- Суммы предоплаты или авансы могут быть исправлены, если клиенту выставлен счет на неправильную сумму.
- Выверки предоплаты и авансов можно исправить, если неверная сумма была использована для сверки с расходами в ранее подтвержденном счете.

> [!IMPORTANT]
> У сведений строки счета, которые являются исправлением других платежей с уже выставленными счетами, поле **Исправление** задано как **Да**. В счетах, в которых есть исправленные данные строк счетов, есть поле с именем **Есть исправления**, в котором также установлено значение **Да**.
