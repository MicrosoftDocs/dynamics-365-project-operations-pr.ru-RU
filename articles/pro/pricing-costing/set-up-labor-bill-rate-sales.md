# <a name="set-up-labor-bill-rates---lite"></a>Настройка ставок оплаты трудозатрат — облегченное развертывание

_**Относится к:** развертывание Lite — от сделки до счетов-проформ_

В каждом прайс-листе есть цены ролей или ставки оплаты труда, которые действуют для контекста и дат, включенных в заголовок прайс-листа. Ставки оплаты за время в Dynamics 365 Project Operations можно устанавливать только в одной валюте, которая указана в заголовке прайс-листа.

1. Чтобы настроить ставки счетов за оплату труда для прайс-листов продаж, создайте прайс-лист на основе заголовка прайс-листа. 
2. На вкладке **Цены ролей** во вложенной сетке выберите **+Создать цену роли**. 
3. В области **Быстрое создание** введите комбинацию роли и подразделения, для которой необходимо настроить ставку выставления счетов.

  Следующая таблица включает поля на вкладке **Общие** и в области **Быстрое создание** строки цен ролей, которую следует иметь в виду при создании цен ролей в прайс-листе продаж:

  | Поле | Размещение | Описание: | Воздействие на последующие элементы |
  | --- | --- | --- | --- |
  | Роль | Вкладка **Общие** и область **Быстрое создание** | Выберите роль, для которой вы устанавливаете ставку выставления счетов. | Роль во входящей оценке или фактической стоимости будет сопоставлена с этой строкой, чтобы установить ставку выставления счетов для роли по умолчанию. |
  | Единица распределения ресурсов | Вкладка **Общие** и область **Быстрое создание** | Выберите организационное подразделение или подразделение компании, из которой эта роль. Например, разработчик из подразделения робототехники компании Fabrikam India или разработчик из подразделения программного обеспечения компании Fabrikam USA. | Единица распределения ресурсов во входящей оценке или фактическая стоимость будет сопоставлена с этой строкой, чтобы установить ставку выставления счетов для роли по умолчанию. |
  | Цена | Вкладка **Общие** и область **Быстрое создание** | Установите ставку выставления счетов для комбинации роли, компании распределения ресурсов и подразделения распределения ресурсов. Например, разработчик из Fabrikam India имеет ставку счета 100 долларов США, а разработчик из Fabrikam USA — ставку счета 150 долларов США. | Эта цена — это ставка выставления счетов по умолчанию, которая основана на цене единицы строки входящей оценки или фактических данных для класса транзакций "Время". |
  | Валюта | Вкладка **Общие** и область **Быстрое создание**| По умолчанию значение валюты берется из валюты в заголовке прайс-листа продаж. В прайс-листе продаж валюта не может быть переопределена. | Эта валюта является валютой по умолчанию для цены единицы строки входящих фактических продаж для класса транзакций "Время". |
  | Группа единиц измерения | Вкладка **Общие** и область **Быстрое создание** | По умолчанию для этого расписания единиц выбрано время, и его нельзя изменить в сущности цены роли, потому что в ней используются экспресс-ставки по единицам времени. | Это поле не оказывает влияния на последующую обработку. |
  | Единица | Вкладка **Общие** и область **Быстрое создание** | Значение единицы берется из поля **Единицы времени** в заголовке прайс-листа продаж. Но это значение можно переопределить. Например, разработчик из Fabrikam India имеет ставку для выставления счетов 1000 долларов США за **Индийский день**. Разработчик из Fabrikam USA имеет ставку выставления счетов 1500 долларов США за **День США**. | Когда цена за единицу по умолчанию устанавливается по входящей оценке или строке фактических данных, система использует систему единиц и преобразование в базовые единицы для вычисления цены за единицу. Например, оценка указана за 10 **индийских дней** работы разработчика из Индии, и единица "Индийский день" определяется как 10 часов. При ценообразовании для этой строки сметы приложение вычисляет цену за единицу по смете как 1000 долларов США / 10 часов = 100 долларов США в час. |


## <a name="transfer-pricing-or-set-up-bill-rates-for-resources-from-other-organizational-units-or-divisions"></a>Трансфертное ценообразование или настройка ставок выставления счетов для ресурсов из других организационных единиц или подразделений 

Компании на основе проектов могут использовать сотрудников других подразделений компании для работы над проектами. Проекты могут выполняться из одного подразделения, в то время как сотрудники или консультанты приходят из того же или другого подразделения компании. В проекте также могут участвовать люди из разных подразделений. В Project Operations компания, которой принадлежит реализация проекта, называется **Единица по контракту**. Все остальные подразделения, предоставляющие ресурсы, называются **Единицы распределения ресурсов**. Из-за различий в стоимости рабочей силы в разных регионах и на рынках труда по всему миру ставки выставления счетов за трудозатраты также устанавливаются по-разному для разных регионов.

Например, за разработчика из Fabrikam India, работающего над проектом в США, выставляются счета из расчета 100 долларов США в час. За разработчика из Fabrikam US, работающего над проектом в США, выставляются счета из расчета 150 долларов США в час.

### <a name="example-set-up-a-bill-rate"></a>Пример: настройка ставки выставления счетов

1. Создайте прайс-лист продаж под названием *Ставки выставления счетов Fabrikam US* и установите дату вступления в силу.
2. В прайс-листе продаже введите следующие сведения о ставках:

    | Роль | Подразделение | Ставка по счету |
    | --- | --- | --- |
    | Developer | Fabrikam India | 100 долларов |
    | Developer | Fabrikam Philippines | 90 долларов США |
    | Developer | Fabrikam US | 150 долларов США |

3. Прикрепите прайс-лист продаж **Ставки выставления счетов Fabrikam US** к прайс-листу проекта контракта по проекту или к определенному счету.
