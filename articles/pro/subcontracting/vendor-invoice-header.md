# <a name="header-details-for-vendor-invoices"></a>Сведения в заголовке для счетов поставщиков

_**Относится к:** развертывание Lite — от сделки до счетов-проформ_

В этой статье рассматривается функциональность, предоставляемая в заголовке счета поставщика в Microsoft Dynamics 365 Project Operations.

Планируя и выполняя проекты, руководители проектов могут нанимать субподрядчиков и приобретать продукты и услуги у поставщиков. Во время выполнения проекта возникают затраты из услуг, материалов и категорий расходов, которые закупаются по субподрядам с поставщиками. Поставщики выставляют счета по этим затратам на проекты путем создания счетов поставщиков.

В следующей таблице представлена информация о полях в заголовках счетов поставщиков в Project Operations.

| Поле | Description | Функциональное воздействие |
| --- | --- | --- |
| Имя. | Название счета поставщика. | Во всех раскрывающихся списках счетов поставщиков название счета указано в первом столбце, чтобы вам легче было идентифицировать счет поставщика. По умолчанию, когда счет поставщика создается из субподряда, в поле **Имя** счета поставщика устанавливается значение, которое состоит из имени субподряда и текущей даты. |
| Description | Краткое описание услуг и продуктов, за которые выставляется счет. | None |
| Производитель | Название компании, которая выставляет счет за товары и услуги. Эта компания должна представлять собой запись организаации с типом отношения **Продавец** или **Поставщик**. | <p>В зависимости от выбранного поставщика значения по умолчанию автоматически вводятся в следующие поля:</p><ul><li>Валюта.</li><li>Прайс-листы</li><li>Условия оплаты</li><li>Адрес платежа</li></ul> |
| Субподряд | Ссылка на субподряд для счета поставщика. | <p>В зависимости от выбранного субподряда значения по умолчанию автоматически вводятся в следующие поля:</p><ul><li>Валюта.</li><li>Прайс-листы</li><li>Условия оплаты</li><li>Адрес платежа</li></ul><p>Субподряд, выбранный в заголовке счета поставщика, по умолчанию вводится в строки счета поставщика, и изменить его там нельзя.</p> |
| Дата счета | Дата для фактических значений затрат, которые будет созданы при подтверждении счета поставщика. | Дата счета также используется для выбора правильного прайс-листа либо из прайс-листов, прикрепленных к соответствующему поставщику, либо из параметров проекта. |
| Причина состояния | Состояние счета поставщика. | <p>Статус определяет, где находится счет поставщика в бизнес-процессе и можно ли его редактировать. Вот некоторые из доступных значений:</p><ul><li>**Черновик** — счет поставщика можно редактировать.</li><li>**Подтвержденно** — счет поставщика был проверен и подтвержден. Счета поставщиков в этом состоянии нельзя изменить или удалить.</li><li>**В процессе** — счет поставщика проверяется. Счета поставщиков в этом состоянии можно редактировать, но невозможно удалять.</li><li>**Отменено** — счет поставщика был отменен. Счета поставщиков в этом состоянии нельзя изменить или удалить.</li></ul> |
| Валюта. | Валюта, в которой поставщик выставляет счета за продукты и услуги, указанные в счете поставщика. | В счете поставщика, который ссылается на субподряд, валюта субподряда вводится по умолчанию в качестве валюты счета поставщика. В счете поставщика, который не ссылается на субподряд, значение по умолчанию вводится из записи организации поставщика и может быть изменено. После сохранения счета поставщика изменить валюту больше нельзя. |
| Единица по контракту | Подразделение компании, отвечающее за получение услуг и/или продуктов от поставщика. | None |
| Условия оплаты | Условия оплаты по выставляемым счетам поставщика. Значение по умолчанию вводится автоматически из записи счета поставщика, но его можно изменить. | Условия оплаты из субподряда копируются во все счета поставщика, связанные с этим субподрядом. Условия оплаты могут быть обновлены, если счет поставщика находится в состоянии **Черновик**. |
| Адрес платежа | Адрес поставщика, которому должны быть отправлены платежи. Значение по умолчанию вводится автоматически из записи счета поставщика, но его можно изменить. | Адрес для оплаты из субподряда копируется как адрес платежа во все счета поставщика, которые создаются для этого субподряда. Адрес для оплаты можно обновить, если счет поставщика находится в состоянии **Черновик**. |
| Промежуточный итог | Если в счете поставщика нет строк, введите промежуточный итог счета до налогов. Если в счете поставщика есть строки, это поле доступно только для чтения. В этом случае отображаемая сумма — это промежуточный итог по всем строкам счета поставщика. | None |
| Итого налогов | Если в счете поставщика нет строк, введите итоговую сумму налогов по субподряду. Если в счете поставщика есть строки, это поле доступно только для чтения. В этом случае отображаемая сумма — это сумма с сумм налогов по всем строкам счета поставщика. | None |
| Общая сумма | Это вычисляемое поле показывает итоговую сумму по счету поставщика после включения налогов. | None |

> [!NOTE]
> Следующие поля в накладной поставщика не могут быть изменены после сохранения накладной поставщика: **Поставщик**, **Субподряд**, **Валюта**, **Единица по контракту** и **Условия оплаты**. Если нужно внести какие-либо изменения в этих полях в счете поставщика, вы должны удалить счет поставщика и создать новый счет поставщика.
