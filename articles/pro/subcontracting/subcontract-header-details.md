# <a name="header-details-for-subcontracts"></a>Сведения о заголовке для субподрядов

[!include [banner](../../includes/dataverse-preview.md)]

_**Относится к:** развертывание Lite — от сделки до счетов-проформ_

Этот тема объясняет функциональные возможности, предоставляемые в заголовке субподряда в Dynamics 365 Project Operations.

По мере того, как руководитель проекта планирует и выполняет проекты, он может нанимать субподрядчиков и приобретать продукты и услуги у поставщиков. Когда руководителям проекта необходимо приобрести продукты или услуги, они могут создать субподряд в Project Operations.

Чтобы создать субподряд, выполните указанные ниже действия.

1. На панели навигации выберите **Субподряды**, и на странице **Субподряд** выберите **Создать**.
2. Введите обязательные сведения, затем выберите **Сохранить**.

    В следующей таблице представлена информация о полях на странице **Заголовок субподряда**.

    | Поле | Описание |Функциональное воздействие |
    |---|------|---| 
    | Имя. | Имя субподряда. | Во всех раскрывающихся списках субподряда название субподряда указано в первом столбце, чтобы помочь вам идентифицировать субподряд. | 
    | Описание | Краткое описание услуг и продуктов, приобретаемых по субподряду. | Без доступа |
    | Производитель | Название компании, у которой покупаются товары и услуги. У этой записи организации тип отношений — **Продавец** или **Поставщик**. | В зависимости от выбранного поставщика значения по умолчанию автоматически вводятся в следующие поля:<br/> **• Валюта** </br> **• Прайс-листы** </br> **• Условия оплаты**</br> **• Адрес платежа**</br> **• Адрес выставления счета**</br> **• Имя для счета** </br>**• Менеджер счета поставщика**|
    | Дата по субподряду | Дата создания субподряда. | Дата субподряда используется для выбора правильного прайс-листа либо из прайс-листов, прикрепленных к соответствующему поставщику, либо из параметров проекта. |
    | Причина состояния. | Состояние субподряда. | Статус определяет, где находится субподряд в бизнес-процессе и можно ли его редактировать. <br/>Доступные значения:<br>• **Черновик**: субподряд можно редактировать. Вы можете редактировать субподряды только со статусом **Черновик**.<br/>• **Подтвержден**: переговоры с поставщиком завершены, и субподряд утвержден для поставки. <br/>• **Закрыт**: поставка по субподряду завершена.<br/>• **Отменен**: субподряд отменен, поставки не планируется.  | 
    | Валюта. | Валюта, в которой приобретаются товары и услуги. Значение по умолчанию вводится автоматически из счета поставщика, но его можно изменить. | Валюта субподряда используется для выбора прайс-листа либо из прайс-листов, прикрепленных к соответствующему поставщику, либо из параметров проекта. Прайс-листы в другой валюте не могут быть связаны с субподрядом. Стоимость времени, затрат и материалов, которые ресурсы поставщика доставляют по этому субподряду, записываются в этой валюте по проекту. После сохранения записи субподряда валюту субподряда изменить нельзя.|
    | Единица по контракту | Подразделение компании, заключающее договор купли-продажи или субподряд с поставщиком. | Без доступа |
    | Условия оплаты | Условия оплаты по счетам поставщика, выставляемым по этому субподряду. Значение по умолчанию вводится автоматически из записи счета поставщика, но его можно изменить. | Условия оплаты из субподряда копируются во все счета поставщика, связанные с этим субподрядом. Условия оплаты могут быть обновлены, если субподряд имеет статус **Черновик**. | 
    | Адрес платежа | Адрес поставщика, которому должны быть отправлены платежи. Значение по умолчанию вводится автоматически из записи счета поставщика, но его можно изменить. | Адрес платежа из субподряда копируется как адрес платежа во все счета поставщика, которые создаются для этого субподряда. Адрес платежа может быть обновлен, если субподряд имеет статус **Черновик**.|
    | Название плательщика | Имя человека или подразделения в компании поставщика, которое отправит накладную. Значение по умолчанию вводится автоматически из записи счета поставщика, но его можно изменить. | Значение **Имя для счета** из субподряда копируются во все счета поставщика, связанные с этим субподрядом. Это поле может быть обновлено, если субподряд имеет статус **Черновик**.|
    | Адрес для выставления счета | Адрес, который используется в счетах от поставщика. Значение по умолчанию вводится автоматически из записи счета поставщика, но его можно изменить. | Этот адрес является адресом «счет от» в счетах поставщика, которые создаются для этого субподряда. |
    | Промежуточный итог | Если у субподряда нет строк, введите промежуточную итоги заказа до вычета налогов. Если субподряд содержит строки, это поле доступно только для чтения. Отображаемая сумма — это промежуточная сумма по всем строкам субподряда. | Без доступа |
    | Итого налогов | Если у субподряда нет строк, введите общий налоги по этому субподряду. Если субподряд содержит строки, это поле доступно только для чтения. Отображаемая сумма — это сумма налога по всем строкам субподряда. | Без доступа |
    | Итоговая сумма | В этом вычисляемом поле отображается общая сумма субподряда после учета налогов. | Без доступа |
    | Дата подтверждения | Дата подтверждения субподряда. | Без доступа |
    | Кем запрошено | По умолчанию в этом поле указывается имя пользователя, создавшего субподряд. Однако создатель субподряда может изменить значение, чтобы указать человека, от имени которого он создает субподряд. | Без доступа |
    | Менеджер счета поставщика | Имя основного контактного лица организации поставщика. Значение по умолчанию вводится автоматически из записи счета поставщика, но его можно изменить. Вы можете выбрать другое контактное лицо в качестве руководителя по работе с поставщиками по субподряду. | Вы можете настроить оповещения по электронной почте, чтобы уведомить контакт об изменениях, внесенных в договор субподряда в результате переговоров о цене. |
