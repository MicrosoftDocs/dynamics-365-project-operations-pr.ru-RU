# <a name="vendor-invoice-lines-for-expense-categories"></a>Строки счета поставщика для категорий расходов

_**Относится к:** развертывание Lite — от сделки до счетов-проформ_

Счет поставщика в Microsoft Dynamics 365 Project Operations может содержать строки счета поставщика для категорий расходов. Руководители проектов могут использовать строки счета поставщика для категорий расходов, чтобы учитывать стоимость услуг, закупаемых как категории расходов.

Строки счета поставщика для категорий расходов могут ссылаться или не ссылаться на строку субподряда для категорий расходов. Если строка счета поставщика для категорий расходов ссылается на субподряд, руководители проектов могут сопоставлять и проверять расходы, за которые выставляется счет, путем сравнения строки счета поставщика с расходами, учтенными в этих категориях расходов и утвержденных руководителями проекта.

В следующей таблице представлена информация о полях в строках счета поставщика для категорий расходов.

| Поле | Description | Функциональное воздействие |
| --- | --- | --- |
| Имя. | Название строки счета поставщика для удобства идентификации. | Это название будет отображаться как первый столбец во всех подстановках, основанных на строках счета поставщика. |
| Description | Краткое описание услуг и продуктов, за которые требует оплату поставщик в строке счета поставщика. | None |
| Субподряд | Субподряд, по которому изначально были заказаны услуги. | При выборе субподряда для счета поставщика все строки в счете поставщика будут наследовать этот субподряд. В счете поставщика не может быть строк, котрые ссылаются на разные субподряды. |
| Строка субподряда | Строка субподряда, по которой были заказаны услуги. Список строк субподряда, которые можно выбрать, ограничен строками выбранного субподряда. | Когда в строке счета поставщика для категорий расходов выбрана строка субподряда, значения по умолчанию для полей **Проект**, **Задача** и **Категория проводки** вводятся из соответствующих полей в строке субподряда. Если выбранная строка субподряда имеет значения в полях **Проект**, **Задача проекта** и **Категория проводки**, значения соответствующих полей в строке счета поставщика не могут отличаться от этих значений. |
| Дата транзакции | Дата, когда фактическое значение затрат по строке счета поставщика будет учтено в проекте. |None |
| Класс проводки | Выберите **Расход**, чтобы учесть счет поставщика для категории расходов. | Значение **Расход** указывает, что строка счета поставщика используется для учета суммы счета за услуги, которые были закуплены как категории расходов. |
| Project | Название проекта, в котором использовались услуги, за которые выставлен счет. | Это поле является обязательным, и его нельзя оставлять пустым. |
| Задача | Название задачи проекта, в которой использовались услуги, за которые выставлен счет. Это поле доступно, только если выбран проект. Выбор задачи проекта необязателен. | Если это поле оставлено пустым, руководитель проекта может сопоставить строку счета поставщика с расходами, учтенными в любой задаче проекта. Если строка счета поставщика не ссылается на строку субподряда, а это поле оставлено пустым, фактическое значение затрат, созданное строкой счета поставщика, не будет связано ни с какими фактическими значениями продаж, по которым не выставлены счета. В этом случае, если настроено выставление счетов на основе задач, может получиться так, что счет по затратам не будет выставлен конечному клиенту. |
| Категория проводки | Категория проводки, по которой выставлен счет. Для выбранной категории проводки должна быть создана соответствующая категория расходов. | Сочетание значений **Категория проводки** и **Единица** будет использоваться в качестве значения по умолчанию или рассчитанного значения для поля **Цена за единицу** в строке счета поставщика. |
| Количество | Введите количество, за которое поставщик требует оплату в строке счета. |None|
| Группа единиц измерения | Значение по умолчанию вводится на основе группы единиц измерения выбранной категории проводки. | None |
| Единица | Значением по умолчанию является базовая единица выбранной группы единиц измерения. Вы можете изменить это значение, чтобы покупать услуги в любой единице из выбранной группы единиц измерения. | Сочетание значений **Категория проводки** и **Единица** будет использоваться в качестве значения по умолчанию или рассчитанного значения для поля **Цена за единицу** в строке счета поставщика. |
| Цена за единицу | Цена за единицу по умолчанию использует сочетание значений **Категория проводки** и **Единица** из прайс-листа проекта, который применим к дате проводки строки счета поставщика. | Если цена для применимого прайс-листа проекта настроена в единицах, которые отличаются от единиц в строке счета поставщика, система использует преобразование единиц для расчета цены за единицу. |
| Промежуточный итог | Это доступное только для поля чтение рассчитывается как *Количество* &times; *Цена за единицу*, если значения введены и в поле **Количество**, и в поле **Цена за единицу**. Если одно или оба эти поля пусты, вы можете ввести значение в это поле.| None |
| Налог с продаж | Введите сумму налога. | None |
| Общая сумма | Общая сумма по строке счета поставщика, включая налоги. Это поле рассчитывается как *Промежуточный итог* +  *Налог с продаж*. | None |
