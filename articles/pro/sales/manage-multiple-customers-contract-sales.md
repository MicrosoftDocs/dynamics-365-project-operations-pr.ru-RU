# <a name="manage-multiple-customers-on-project-contracts---lite"></a>Управление несколькими клиентами в контрактах по проекту — облегченное развертывание

_**Относится к:** развертывание Lite — от сделки до счетов-проформ_

Контракты по проектам в Dynamics 365 Project Operations поддерживают сценарий, в котором договорное соглашение включает нескольких клиентов, финансирующих сделку. Вкладка **Сводка** на странице **Контракт по проекту** включает поле **Клиент**. В этом поле указывается основной клиент сделки. Других клиентов для сделки можно настроить на вкладке **Клиенты** страницы **Контракт по проекту**.

Все клиенты контракта, перечисленные в контракте по проекту, по умолчанию являются клиентами строки контракта для любых новых строк контракта на основе проекта, которые создаются для контракта по проекту. Существующие строки контрактов на основе проектов не наследуют новых клиентов по контракту при создании новых записей.

Строки контрактов на основе продуктов автоматически связываются с основным клиентом.

Клиентов по контракту и клиентов по строкам контрактов можно добавить, обновить или удалить в любое время до заключения контракта.

## <a name="primary-customer"></a>Основной клиент

Клиент, указанный на вкладке **Сводка** контракта по проекту в качестве потенциального клиента, является основным клиентом контракта. При попытке удалить основного клиента из списка клиентов в контракте вы получите сообщение об ошибке, указывающее, что запись основного клиента в контракте не может быть удалена.

Основного клиента нельзя обновить из списка клиентов контракта. Вместо этого измените потенциального клиента на вкладке **Сводка** контракта. Когда это поле обновляется на странице **Сводка по контракту**, новый клиент добавляется как новый клиент контракта с установленным флагом **Основной**. Предыдущий основной клиент по-прежнему будет клиентом по контракту.

## <a name="create-update-or-delete-a-contract-customer-record"></a>Создание, обновление или удаление записи о клиенте по контракту

Клиента по контракту можно создать, обновить или удалить на вкладке **Клиенты** на странице **Контракт по проекту**. Поля в следующей таблице относятся к записи о клиенте контракта в контракте по проекту, и их следует помнить при работе с контрактом.

Это поле блокируется после создания записи. Чтобы обновить учетную запись, удалите запись и создайте ее заново. ## <a name="edit-billing-split-percentages"></a>Изменить процентное соотношение разделения выставления счетов

Процентные доли разделения счетов можно редактировать с помощью встроенного интерфейса редактирования сетки. Если процентные доли разделения счетов в сумме не составляют 100 процентов, вы получите сообщение об ошибке. После редактирования процентов разделения счетов обновите страницу, чтобы убрать ошибку.

Вы также можете выбрать **Равномерно распределить** во вложенной сетке **Заказчики по контракту** для равномерного распределения счетов по всем клиентам контракта. Если есть коэффициент округления, он будет добавлен к клиенту округления. Один из клиентов контракта всегда помечается как клиент **округления**, что означает, что для записи клиента по контракту установлен флаг округления **Да**. Обычно это основной клиент контракта, но он также может быть изменен.
