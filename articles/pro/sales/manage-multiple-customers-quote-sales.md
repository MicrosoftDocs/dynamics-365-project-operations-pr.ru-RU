# <a name="manage-multiple-customers-on-project-quotes---lite"></a>Управление несколькими клиентами в предложениях с расценками по проекту — облегченное развертывание

_**Относится к:** развертывание Lite — от сделки до счетов-проформ_

Предложения с расценками по проекту поддерживают сценарий, в котором предложение включает нескольких клиентов, которые будут финансировать сделку. На вкладке **Сводка** предложения с расценками имеется поле **Потенциальный клиент**, идентифицирующее основного клиента сделки. Других клиентов для сделки можно настроить на вкладке **Клиенты** предложения с расценками по проекту.

Все клиенты предложения с расценками на вкладке **Клиенты** предложения с расценками проекта по умолчанию задаются в качестве клиентов строки предложения с расценками во всех **новых** строках предложения с расценками на основе проекта, созданных для предложения с расценками по проекту. Любые существующие строки предложения с расценками по проекту не наследуют новые записи клиентов предложения с расценками, созданные после них.

Строки предложений с расценками на основе продуктов автоматически связываются с основным клиентом, который также является клиентом в поле **Потенциальный клиент** на вкладке **Сводка** предложения с расценками.

Клиенты предложения с расценками и клиенты строк предложения с расценками можно добавить, обновить или удалить в любое время до того, как предложение с расценками будет выиграно.

## <a name="concept-of-a-primary-customer"></a>Понятие основного клиента

Клиент, который находится на сводной вкладке предложения с расценками по проекту в качестве потенциального клиента, является основным клиентом предложения с расценками. Когда вы пытаетесь удалить основного клиента из списка клиентов предложения с расценками, вы увидите сообщение об ошибке, что запись основного клиента предложения с расценками не может быть удалена.

Основного клиента не следует обновлять из списка клиентов в предложении с расценками. Однако вы можете повлиять на основного клиента, изменив потенциального клиента на вкладке **Сводка** предложения с расценками. Когда это поле обновляется в **Сводке предложения с расценками**, новый выбранный потенциальный клиент добавляется как новый клиент предложения с расценками с установленным флагом **Основной**. Старый потенциальный клиент по-прежнему будет клиентом в предложении с расценками.

Клиент предложения с расценками может быть создан, обновлен или удален на вкладке **Клиенты предложения с расценками** на странице **Предложение с расценками**. В следующей таблице указаны поля, имеющиеся в записи клиента предложения с расценками для предложения с расценками по проекту.

## <a name="editing-billing-split-percentages"></a>Изменение процентного соотношения разделения выставления счетов

Вы можете редактировать процентные доли разделения счетов с помощью встроенного интерфейса редактирования сетки. Когда процентные доли разделения выставления счетов не составляют в сумме 100%, возникнет ошибка. После обновления процентов разделения счетов обновите страницу, чтобы удалить ошибку.

Вы также можете попробовать выбрать **Распределить равномерно** во вложенной сетке клиента предложения с расценками. Это действие распределяет разделение выставления счетов для всех клиентов предложения с расценками. Если есть какой-либо коэффициент округления, он будет добавлен к клиенту округления. Один из клиентов предложения с расценками всегда помечается как клиент округления. Это означает, что запись клиента предложения с расценками имеет флаг **Округление**, установленный в **Да**. Обычно это основной клиент предложения с расценками, но это можно изменить.
