# <a name="schedule-a-project-with-a-work-breakdown-structure-project-service"></a>Планирование проекта со структурной декомпозицией работ (Project Service)

[!INCLUDE[cc-applies-to-psa-app-1x-2x](../includes/cc-applies-to-psa-app-1x-2x.md)]

Расписание проекта определяет, какую работу необходимо выполнить, какие ресурсы будут выполнять работу и период времени, за который необходимо выполнить работу. Расписание проекта отражает все работы, связанные с выполнением проекта в срок. Одним из первых шагов в стадии начала проекта — составление расписания проекта. Чтобы составить расписание проекта необходимо создать структурную декомпозицию работ.  
  
 Создание структуры проекта со структурной декомпозицией работ, которая поможет сделать следующее:  
  
- Разбить работу на управляемые задачи  
  
- Оценить время, требуемое для выполнения задачи  
  
- Задавать зависимости задач и длительность задач  
  
- Определять роли, необходимые для выполнения каждой задачи  
  
  Расписание проекта в структурной декомпозиции работ похож на интерактивную диаграмму Ганта.  
  
## <a name="create-a-work-breakdown-structure-for-a-project"></a>Создание структурной декомпозиции работ для проекта  
 Создание структурной декомпозиции работ для представления последовательности задач в проекте. Структурная декомпозиция работ включает задачи, требования для каждой задачи и сведения о выручке и расходах. В структурной декомпозиции работ можно добавить:  
  
-   Последовательность задач в иерархии  
  
-   Другие задачи (если есть), которые необходимо выполнить перед началом задачи  
  
-   Дата начала, дате окончания и длительности задачи  
  
-   Количество часов, требуемых для задачи  
  
-   Требуемые навыки работников и образование  
  
-   Рабочие, назначенные для задачи  
  
-   Предполагаемые выручка и расходы  
  
## <a name="task-types"></a>Типы задач  
Будут использоваться следующие типы задач при создании своей структурной декомпозиции работ:  

| | | 
|---------------------------------------|-----------------------------------------------------------------| 
| **Корневой узел проекта** | Сводная задача верхнего уровня для проекта. Все другие задачи проекта создаются под ней. Название корневой задачи — имя проекта. Усилия, даты и длительность корневого узла основаны на значениях в иерархии под ним. Нельзя изменять свойства корневого узла или удалять корневой узел. | 
| **Сводные задачи или задачи контейнера** | Сводная задача — это задача, в которой есть подзадачи. В сводной задачи нет своих усилий работы или стоимости. Ее усилия по работе и стоимость являются сверткой подзадач. Можно изменить имя сводной задачи, но нельзя изменить усилия, даты и длительность, поскольку они автоматически рассчитываются. При удалении сводной задачи удаляется задача и все ее подзадачи.|  
| **Задачи листового узла** | Задача листового узла представляет наиболее подробную работу над проектом. В ней содержится расчетное усилие, плановое количество ресурсов, запланированные даты начала и окончания и продолжительность.|

  
## <a name="task-hierarchy"></a>Иерархия задач  
 Имеются следующие параметры при создании иерархии задач:  
  
- **Добавить задачу**.   Можно добавить задачу в расположении, выбранном в иерархии задач. Если не выбрать расположение, новая задача появляется в конце.  
  
- **Понизить уровень задачи**.   Понизьте уровень задачи, чтобы сделать ее дочерней относительно задачи непосредственно над ней.  
  
- **Повысить уровень задачи**   Повысьте уровень задачи, чтобы она больше не была дочерней относительно исходной родительской задачи.  
  
- **Переместить вверх и вниз**.   Перемещайте задачи вверх и вниз по иерархии родительской задачи. Перемещение задачи вверх или вниз не влияет на ее усилие, стоимость, даты и длительность.  
  
## <a name="task-attributes"></a>Атрибуты задачи  
 Название задачи описывает работу, которая должны быть выполнена. Следует использовать разные атрибуты задач для описания расписания и требований к персоналу для задачи.  
  
### <a name="schedule-attributes"></a>Атрибуты расписания

 - Назначение значений **Рабочие часы** , **Количество ресурсов** , **Дата начала** , **Дата окончания** и **Длительность** для определения расписания для задачи. 
 - **Усилия** — это оценка часов, которые потребуются для выполнения задачи.
 - **Количество ресурсов** — оценка, которую менеджер проекта помещает в задачу для составления наилучшего расписания. 
 - **Длительность** (в днях) указывает число рабочих дней, которые потребуются для выполнения задачи.  
  
### <a name="staffing-attributes"></a>Атрибуты персонала

 - **Роль** , **Подразделение ресурса** , **Количество ресурсов** и **Ресурсы** описывают требования к персоналу для задачи. 
 - **Роль** описывает тип ресурса, необходимого для выполнения задачи. 
 - **Подразделение ресурса** указывает подразделение, из которого ресурсы должны быть укомплектованы персоналом для этой задачи; это также влияет на оценку стоимости и продаж по задаче, так как это учитывается при определении цены продажи единицы для этого ресурса. 
 - **Ресурсы** содержит общий ресурс или именованный ресурс при нахождении ресурса.  
  
## <a name="task-dependencies"></a>Зависимости задач  
 Можно создавать отношения предшественников между одной или несколькими задачами в структурной декомпозиции работ. Может потребоваться задать одно или несколько значений для поля предшественника по задаче, чтобы указать задачи, от которых он будет зависеть. При назначении значения предшественника для задачи задача может начаться, только когда все задачи предшественника будут завершены. Установка этой зависимости по задаче приведет к пересчету запланированной даты начала задачи как последнего окончания всех предшественников. Влияния на расписание, связанные с предшественниками, не ограничиваются режимом задачи, указанным для задачи.  
  
## <a name="task-mode"></a>Режим задачи  
 Режим задачи — один из важных факторов, определяющие планирование задач листового узла. Есть два режима задач по каждой задачи: режим автоматического планирования и режим ручного планирования.  
  
-   **Автоматические создание расписания**.   Когда вы задаете режим задачи "автоматическое планирование", модуль планирования задачи использует правила планирования для следующих атрибутов задачи для определения расписания для задачи:  
  
    -   Предшественники  
  
    -   Усилия  
  
    -   Количество ресурсов  
  
    -   даты начала и окончания;  
  
-   **Правила планирования**.   Дата начала задачи листового узла не имеет предшественников по умолчанию относительно даты начала планирования проекта. Длительность задачи листового узла всегда рассчитывается как число рабочих дней между датами начала и окончания. Если задача планируется автоматически, модуль планирования соответствует правилам ниже:  
  
    -   Даты начала и окончания задачи всегда должны быть рабочими днями относительно календаря планирования проекта  
  
    -   Дата начала задачи, у которой есть предшественник, по умолчанию является последней датой окончания ее предшественников  
  
    -   Усилия = число людей * длительность * часы в стандартном рабочем дне календаря проекта  
  
-   **Ручное планирование**.   В некоторых случаях может потребоваться отклониться от этих правил. В этих случаях можно задать режим ручного планирования задачи. При этом модуль планирования не рассчитывает значения для других атрибутов планирования. Установка предшественников по задачам всегда влияет на дату начала зависимой задачи.  
  
## <a name="create-a-work-breakdown-structure"></a>Создание структурной декомпозиции работ  
  
1.  Перейдите к разделу **Project Service > Проекты**.  
  
2.  Щелкните проект, над которым вы хотите работать.  
  
3.  В строке в верхней части экрана щелкните стрелку вниз рядом с именем проекта и щелкните "Структурная декомпозиция работ".  
  
4.  Для добавления задачи щелкните **Добавить задачу**. Заполните поля для задачи, затем щелкните **Сохранить**.  
  
5.  Продолжите добавлять задачи до завершения структурной декомпозиции работ. При создании своей структурной декомпозиции работ можно выполнить следующие действия для упорядочивания своих задач:  
  
    -   Выберите задачу и щелкните **Понизить уровень** , чтобы переместить ее под другую задачу или щелкните "Повысить уровень" для повышения.  
  
    -   Выберите задачу и щелкните **Вверх** или **Вниз** , чтобы переместить ее вверх или вниз в списке.  
  
    -   Щелкните **Скрыть Ганта** , чтобы скрыть диаграмму Ганта, и щелкните **Показать Ганта** , чтобы показать ее.  
  
    -   Выберите другой период времени диаграммы Ганта в области **Шкала времени**.  
  
6.  Для добавления ролей, указанных в структурной декомпозиции работ, для участников рабочей группы своего проекта щелкните **Создать рабочую группу проекта**.  
  
7.  По окончании внесения изменений щелкните **Сохранить** в правом нижнем углу экрана.
