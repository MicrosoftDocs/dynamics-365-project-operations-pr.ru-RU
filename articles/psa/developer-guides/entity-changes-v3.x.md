<span data-ttu-id="bc46f-107">В версиях Dynamics 365 Project Service Automation (PSA), выпущенных до версии 3.0, некоторые отношений между сущностями документов продажи, строк документов продаж и сведений строк документов продаж были реализованы через строковые поля, которые должны были содержать строковое представление GUID связанной сущности.</span><span class="sxs-lookup"><span data-stu-id="bc46f-107">In versions of Dynamics 365 Project Service Automation (PSA) released prior to version 3.0, some of the relationships between sales documents, sales document lines, and sales document line detail entities were implemented through string fields that would hold a string representation of GUID of the related entity.</span></span> <span data-ttu-id="bc46f-108">Это было связано с ограничениями платформы, которая требовала значительного объема настраиваемого кода на сторонах сервера и клиента в решении, чтобы эти отношения работали аналогично типичным отношениям сущностей Dynamics CRM и чтобы строковые поля вели себя как поля подстановки.</span><span class="sxs-lookup"><span data-stu-id="bc46f-108">This was due to platform limitations that required significant custom code on the server and client sides of the solution to make those relationships work similar to typical Dynamics CRM entity relationships and to make string fields act like lookup fields.</span></span>
