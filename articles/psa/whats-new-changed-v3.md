# <a name="whats-new-or-changed-in-project-service-automation-version-3"></a>Что нового или измененного в Project Service Automation версии 3

[!include [banner](../includes/psa-now-project-operations.md)]

[!INCLUDE[cc-applies-to-psa-app-1x-2x](../includes/cc-applies-to-psa-app-1x-2x.md)]

[!include [rename-banner](~/includes/cc-data-platform-banner.md)]

В этом разделе приведены сведения об изменениях в пользовательском интерфейсе (UI), функциональных возможностях и терминологии в Project Service Automation между версией 2 или версией 1 и версией 3.

## <a name="project-scheduling"></a>Планирование проекта
Расписание проекта, которое было известно как структурная декомпозиция работ (WBS) как в предыдущих версиях, было переименовано в "Расписание", и к нему можно перейти, щелкнув вкладку **Расписание**. 

![Расписание проекта.](media/psa-schedule-01.png)

У расписания теперь новая поверхность для взаимодействия — современная и доступная. Базовая подсистема планирования Project Service Automation, однако, не изменилась. Кнопки управления на ленте сетки расписания позволяют взаимодействовать с расписанием аналогично тому, как это делалось в предыдущих версиях Project Service Automation. Дополнительные изменения в расписании включают:

- **Диаграмма Ганта** — диаграмма Ганта больше не представлена. Новая визуализация Ганта будет возвращена в будущем обновлении.
- **Заголовки столбцов** — можно скрывать заголовки столбцов в сетке щелкнув вниз индикатор рядом с заголовком столбца. 
- **Столбцы** — можно отобразить скрытые столбцы, нажав **Добавить столбец**. 
- **Категория проводки** — Поиск **Категория проводки** добавлен в сетку расписания и отображен по умолчанию. 
 
## <a name="project-templates"></a>Шаблоны проектов
Следующие изменения были внесены в функциональность шаблона проекта.

### <a name="create-a-project-template"></a>Создание шаблона проекта 
В версии 3 можно создавать шаблоны проектов аналогично тому, как это делалось в предыдущих версиях Project Service Automation. Шаблон может содержать только расписание, а расписание может включать назначения, но они не обязательны. Если расписание имеет назначения, они могут быть только для общих ресурсов. Вы можете сформировать требования к ресурсам для общих ресурсов, но они не могут быть зарезервированы с реальными ресурсами в шаблоне. Вы не можете забронировать реальный ресурс для группы в шаблоне. 

### <a name="create-a-template-from-an-existing-template"></a>Создание нового шаблона из существующего шаблона
Когда вы создаете новый шаблон проекта из существующего шаблона в Project Service Automation версии 3, происходит следующее: 

- Расписание исходного проекта копируется в шаблон. 
- Универсальные ресурсы копируются в группу, любые назначения универсальных ресурсов копируются. Требования для универсальных ресурсов не копируются. 

### <a name="create-a-template-from-an-existing-project"></a>Создание нового шаблона из существующего проекта
Когда вы создаете новый шаблон проекта из существующего проекта, происходит следующее: 

- Расписание исходного проекта копируется в шаблон. 
- Универсальные ресурсы копируются в группу, любые назначения универсальных ресурсов сохраняются. Требования для универсальных ресурсов не копируются.    
- Именованные ресурсы, назначенные или неназначенные, удаляются из группы и заменяются универсальными ресурсами.
- При наличии, информация о клиенте удаляется. 
- При наличии ссылки на предложения с расценками и контракты удаляются. 

### <a name="create-a-project-from-a-template"></a>Создание проекта из шаблона
В Project Service Automation версии 3, когда вы создаете новый проект на основе шаблона, происходит следующее:

- Расписание, группа, и назначения копируются в новый проект.   
- Дата начала — это либо дата копирования, либо дата, выбранная пользователем.   
- Для всех универсальных участников рабочей группы с потребностями в ресурсах в шаблоне требования не копируются и не генерируются автоматически. Понадобится создать их. 

## <a name="copy-a-project"></a>Копировать проект
В Project Service Automation версии 3 при копировании проекта происходит следующее: 

- Предполагаемая дата начала копируется, но может быть изменена.  
- Расписание и задачи по проекту скопированы. 
- Универсальные ресурсы и их назначения скопированы. Требования к ресурсам для универсальных ресурсов не копируются. Понадобится повторно создать их. 
- Реальные ресурсы и их назначения не копируются. Вместо этого они заменяются универсальными ресурсами. 
- Фактические значения не копируются в новый проект. 

## <a name="move-a-scheduled-project"></a>Переместить запланированный проект
При перемещении расписания существующего проекта вперед, происходит следующее: 

- Даты задачи автоматически переносятся в соответствии с периодом движения. 
- Назначенные универсальные ресурсы будут сохранены назначенными.   
- Если они создаются до перемещения проекта, требования к универсальному ресурсу остаются прежними и автоматически не создаются повторно. Понадобится создать их заново, чтобы отразить новые назначения из-за перемещения задачи. 
- Назначения по реальным ресурсам меняются в соответствии с датой перемещения задачи. Резервирование на реальные ресурсы не меняются. Потребуется изменить бронирование, используя представление выверки. 
- Ресурсы группы с бронированием, но без назначений не меняются. 
- Фактические не перемещать. 

## <a name="estimates"></a>Оценки
Оценки были разделены на две вкладки, **Назначение ресурсов** и **Оценки**. Вкладка **Назначение ресурса** содержит оценки трудозатрат и показывает назначения ресурса для задач в поэтапном представлении. Вы можете редактировать оценки на основе того, что сгенерировал механизм планирования.

![На вкладке «Назначения ресурсов» отображаются оценки трудозатрат и назначения ресурсов для задач.](media/resource-assignments-tab-02.png)

Вкладка **Оценки** показывает значения стоимости и продаж для назначений ресурса. Суммы только для чтения. Калькуляция затрат и продаж теперь определяются на основе назначений участников рабочих групп по расписанию. Это означает, что если у вас есть задача без какого-либо назначения, она будет отображаться в неназначенной группе. Это также означает, что без **роли**, которая является измерением цены по умолчанию, не будет никакой предполагаемой стоимости или продаж, если у вас есть клиент или контракт/предложение с расценками, связанные с проектом. 

![Вкладка «Оценки» с указанием значения стоимости и продаж.](media/estimates-tab-03.png)
  
Категория также поддерживается для задач в представлении расписания. Группировка по категориям для представления фаз времени оценок обеспечит лучшее взаимодействие, особенно если у вас также есть оценки расходов в вашем проекте. Оценки расходов вводятся с помощью сетки на отдельной вкладке. 

Оценки расходов можно вводить в сетку на вкладке **Оценки расходов**. 

![Вкладка оценок расходов отображает сетку оценок расходов.](media/expense-estimates-tab-04.png)

## <a name="resource-management"></a>Управление ресурсами
В Project Service Automation версии 3, из-за нового единого пользовательского интерфейса клиента и изменений в отношениях между резервированиями и назначениями, подбор персонала для проекта с использованием универсальных или реальных ресурсов значительно изменился по сравнению с версией 2 и версией 1. Однако основные понятия резервируемых ресурсов, как **реальных**, так и **универсальных**, остаются такими же, как и участники рабочей группы, требования, назначения и резервирования.   

![Использование средства выбора ресурсов.](media/resource-management-05.png)

### <a name="assign-a-real-bookable-resource"></a>Назначьте реальный резервируемый ресурс 
В Project Service Automation версии 3 резервирования и назначения задач не так тесно переплетены, как в предыдущих версиях Project Service Automation. Можно использовать сетку группы чтобы забронировать **реального** участника рабочей группы, как на рынке.

Используя средство выбора ресурсов по расписанию, вы можете выбрать участника рабочей группы, созданного в представлении группы, а затем назначить их задачам. Вы можете продолжать назначать им задачи даже после их бронирования. Используйте вкладку **Выверка** для выверки участников рабочей группы, которые имеют различия в бронировании и назначениях.

Подборщик ресурса отобразит участников рабочей группы для проекта. Можно также использовать средство выбора ресурсов для поиска и просмотра других доступных для бронирования ресурсов, которые не являются частью рабочей группы проекта. Можно назначить их задаче и они станут частью рабочей группы проекта. Вам нужно будет забронировать их, используя **Таблицу расписания** или вкладку **Выверка**.

### <a name="assign-a-generic-bookable-resource-on-a-task-and-project-team-and-then-fulfill-with-a-real-resource-via-schedule-board"></a>Назначьте универсальный резервируемый ресурс для задачи и проектной группы, после чего выполните посредством реального ресурса через таблицу расписания 
В Project Service Automation версии 3 функциональность создания рабочей группы не используется для универсальных ресурсов. Напротив, можно создавать и напрямую назначить универсальный ресурс из расписания, введя имя позиции универсального ресурса в ячейке ресурса расписания. Или можно выбрать значок ресурса в ячейке и затем, используя средство выбора ресурса, ввести имя универсального ресурса, который вы хотите создать. Это откроет панель быстрого создания, которая позволит вам установить роль и подразделение члена рабочей группы универсального ресурса. После создания ресурса он назначается задаче, и вы можете продолжить назначать этот универсальный ресурс другим задачам в расписании.    
 
При назначении ресурса всем соответствующим задачам можно создать требование к ресурсу, а затем выполнить его, непосредственно бронируя с помощью **Таблицы расписания** или путем передачи запроса ресурса. Вы также можете добавить универсальные ресурсы непосредственно в сетку участников рабочей группы. 

Универсальные ресурсов были добавлены в рабочую группу проекта без требований к ресурсам и с датами начала/окончания проекта, пока не будут сгенерированы требования к ресурсам. Чтобы сгенерировать требование, выберите универсальный ресурс и нажмите **Создать**. Теперь отображается ссылка на требование, и необходимые часы будут заполнены назначенными часами. Можно нажать на ссылку для открытия и для обновления требования.
  
Когда резервирование завершено и полностью выполнено именованным ресурсом, универсальный ресурс заменяется на именованный ресурс, а назначение в расписании обновляется с помощью именованного ресурса. 

Предлагаемые ресурсы для требований теперь хранятся на вкладке, а не в отдельном разделе.

### <a name="multiple-named-resources-fulfilling-a-generic-resource"></a>Несколько именованных ресурсов, выполняющих универсальный ресурс
Когда требование выполняется с несколькими ресурсами, универсальный ресурс остается в рабочей группе и назначается задаче. Именованные участники рабочей группы, которые забронированы, не назначены как часть позиции. Руководитель проекта может назначить работу реальным ресурсам по мере необходимости.  Представление **Выверка** обеспечивает разбивку резервирований по нескольким ресурсам на несколько назначений для задачи. Это не делается автоматически, поскольку в более сложных сценариях, например, где имеется набор задач, составляющих требование, намерение, которое руководитель проекта хочет назначить. Поскольку система не может понять намерения, предположения, вероятно, будут отличаться от запланированных, и произойдет неправильный или непредсказуемый результат. Прогнозируемый результат заключается в том, что универсальный ресурс остается назначенным, пока руководитель проекта явным образом не назначит ресурсы с помощью представления **Выверка**.

### <a name="reconciliation"></a>Выверка
Вкладка **Выверка**, которая показывает резервирования и все назначения для каждого участника проектной группы. Представление показывает часы в ячейках, которые могут представлять моменты времени от месяцев до дней. Эти представления помогают руководителям проекта выверить резервирования участников рабочей группы и их назначения для их проектной группы. Это полезно поскольку резервирования и назначения задачи не тесно связаны, что обеспечивает большую гибкость при планировании проекта. 

![Вкладка «Выверка», которая показывает резервирования и все назначения для каждого участника проектной группы.](media/resource-reconciliation-tab-06.png)

Для каждого из ресурсов представление учитывает разницу между бронированиями участника рабочей группы и сверткой о назначениях их задач и показывает следующие два различия, которые могут возникать при бронировании и назначениях в проекте: 

- **Недорезервирование** — недостаточные резервирования возникают, когда ресурс имеет больше назначений, чем резервирований. Поскольку вся его производительность не была зарезервирована, руководитель проекта может исправить это, расширив резервирования, чтобы покрыть дефицит. 
- **Перерезервирование** — избыточные резервирования происходящих, когда ресурс зарезервирован на проект, но ему не назначены задачи.  Это может быть приемлемым, если, например, ресурс был зарезервирован до появления назначения задачи. Однако в других случаях ресурс может быть не запланирован для назначения, и руководитель проекта должен рассмотреть возможность отмены резервирования ресурса, чтобы позволить использовать ресурс для другого проекта. 

Когда имеются назначения задач для ресурса, но нет резервирований (недорезервирование), можно агрегировать недорезервирование, затем выбрать **Продлить резервирование**. В этой области можно просмотреть бронирование, необходимое для решения проблемы нехватки ресурса и его доступности. 
 
## <a name="time-and-expense"></a>Время и расход
В этом разделе представлена информация об изменениях во времени, расходах и утверждении в Project Service Automation версии 3. В рамках решения Dynamics 365 Project Service Automation, функция **Запись времени** была обновлена для использования платформы единого интерфейса. Это позволяет обеспечить согласованный, единый пользовательский интерфейс, отвечающий гибкому дизайну, для оптимального просмотра на любом размере экрана или устройстве. 

### <a name="landing-page"></a>Целевая страница
Нерасширяемые настраиваемые записи времени устарели в версии 3. Вместо этого теперь есть расширяемый и доступный собственный опыт работы с сеткой. Можно получить доступ к функциям записи времени, используя карту сайта слева. После этого изменения вы больше не сможете вводить время для одной недели за раз. Напротив, необходимо создать запись времени в каждый день в сетке. После создания нескольких записей времени пользователи могут массово создавать записи времени с помощью функции **Копировать**, объясненной далее в данном разделе. 

![Целевая страница записи времени.](media/time-entry-landing-page-07.png)
 
### <a name="create-new-time-entries"></a>Создание новых записей времени 
Щелкните **Создать** в ленте для открытия страницы быстрого создания для ввода времени, где вы вводите продолжительность в минутах, часах или днях. Чтобы сделать это, просто начните вводить h, m или d вместе с количеством.  

![Быстрое создание записи времени.](media/quick-create-time-entry-08.png)

Поля подстановки поддерживаются системными представлениями. Например, после ввода данных проекта, поле **Задача проекта** установлены по умолчанию на представление **Мои открытые задачи проекта**. Чтобы создать записи времени для задач, которые не назначены пользователю, выберите **Изменить представление** в поиске и выберите **Все активные задачи проекта**. После того, как запись времени была создана и показана в сетке, можно изменять любые значения строк непосредственно в сетке.  

### <a name="bulk-createcopy"></a>Массовое создание/копирование 
После создания нескольких записей времени можно использовать функцию копирования для массового создания дополнительных записей времени. Щелкните **Копировать**, чтобы открыть диалоговое окно **Копировать**. В **Из периода: Дата начала** установите диапазон дат, из которого должны копироваться периоды времени. В **В период: Дата начала** укажите дату, для которой необходимо создать записи времени. Щелкните **Копировать** для копирования записей времени в соответствующий день недели в **В период**. Например, запись времени понедельника с прошлой недели будет скопирована в понедельник для недели, указанной в поле **В период**. 

![Массовое копирование записей времени.](media/bulk-copy-time-entry-09.png)
 
### <a name="import-data"></a>Импортировать данные 
Назначения и обмен выполняются по одному и тому же шаблону пользовательского интерфейса, что позволяет пользователю указать диапазон дат, когда необходимо импортировать резервирования. Затем необходимо явно выбрать резервирования, которые должны быть скопированы в записи времени **Черновик**. В версии 3 вы больше не видите шаблон **Предлагаемые** записи времени в сетке и календаре.  

### <a name="change-in-calendar-control"></a>Изменение в элементе управления календаря
В версии 3 мы отошли от пользовательского элемента управления календаря и теперь используем календарь UC для отображения записей времени за неделю. С помощью этого календаря вы можете просматривать день, неделю и месяц. 

> [!NOTE]
> Ограничение для календаря состоит в том, что этот элемент управления не поддерживает действия с отдельными элементами календаря. Например, пользователь не сможет выбрать один или несколько элементов календаря и отправить или удалить эти элементы. При нажатии на элемент календаря откроется страница **Сущность записи времени** для дополнительных действий. 

### <a name="extensibility"></a>Расширяемость
**Собирать данные в настраиваемых полях только в сущностях записи времени и затрат** — запись времени использует редактируемую сетку, сетку только для чтения и элементы управления календарем с платформы. Все эти элементы управления являются родственными и, следовательно, будут поддерживать настройки. В Project Service Automation версии 3 можно добавить дополнительные настраиваемые поля, настроить поля поиска и создать для них резервные копии с настраиваемыми представлениями. Можно также задать настраиваемую бизнес-логику на основе выбранных значениях в настраиваемых полей.  

**Сбор данных в настраиваемых полях в записи времени и затрат и их распространение через сущности, поддерживающие поток отправки и утверждения** — типичная обработка записей времени показана в следующей схеме.

![Поток обработки записи времени.](media/process-time-entries-10.png)

Если бизнес-требования предусматривают, что сущности времени и затрат должны собирать настраиваемые аналитики ценообразования и распространять значения, заданные ресурсом времени и записи в настраиваемой аналитике ценообразования, на все сущности на предыдущем рисунке, см. раздел [Настройка настраиваемых полей в качестве аналитики ценообразования](set-up-pricing-dimensions.md)

Для поддержки бизнес-требований, когда сущности времени и затрат должны собирать настраиваемые измерения, не связанные с ценами, и распространять значения, можно использовать настройку аналитики ценообразования и выражать настраиваемые аналитики как аналитики ценообразования без стоимости или ставок по счету. Другим сценарием будет добавление настраиваемого поля для каждого из сущностей, используя одно и то же имя поля для всех сущностей. Настраиваемые подключаемые модули могут быть созданы для связывания записей в сущностях, которые участвуют в потоке отправки/утверждения, используя сущности подключения проводки.  

### <a name="delegate-time-and-expense-entry"></a>Делегирование записи времени и расходов
Платформа Common Data Service не поддерживает одного пользователя, выдающего себя за другого, что означает, что в Project Service Automation версии 3 нет поддержки делегированной записи времени и расходов. Однако партнеры и клиенты используют обходное решение, чтобы включить поддержку делегированного опыта записи времени в версии 3. Это только обходной путь, а не полное решение вопроса, поэтому важно понимать ограничения и использовать этот подход, только если ограничения приемлемы. 

> [!IMPORTANT]
> Эта информация должна рассматриваться только как рекомендуемое руководство для реализации партнером или клиентами. Группа по продукту не будет предлагать официальную поддержку этой функции через любой из наших каналов поддержки.

### <a name="customization-details"></a>Сведения по настройке 
Настройка позволяет добавлять **Резервируемый ресурс** в процессы создания и редактирования, которые позволят пользователям выступать в качестве представителя, изменив поле **Резервируемый ресурс** на другого пользователя, для которого необходимо делать записи времени и расходах. Делегирование времени записи в следующие шагах. Такая же сведения применяются к делегированию записи расходов. 
 
1.  Убедиться в том пользователь-представитель имеет доступ к проектам и задачам по проекту. 
1.  Поскольку **Резервируемый ресурс**, который является полем сущности **Запись времени**, не отображается на странице **Быстрое создание**, понадобится добавить его.

    или

    Создание настраиваемого представления, которое включает столбец **Резервируемый ресурс** для просмотра только записи времени созданные для данного ресурса. Публикация настроек в конструкторе модуля приложений для этого представления для отображения в **Селектор представления** на странице **Записи времени**. Есть два подключаемые модули которые поддерживают параметр руководителя для записей времени, не связанных с проектом:

    - PreValidateTimeEntryCreate
    - PreValidateTimeEntryUpdate
 
1. Создание нового подключаемый модуль, чтобы перезаписать поле **Руководитель** на руководителя пользователя, назначенного в поле **Резервируемый ресурс**. Используйте тот же **Этап выполнения** что и внешний подключаемый модуль (OOB) (предварительная проверка) и используйте **Порядок выполнения** который выше, чем подключаемые модули OOB (более 1). Это обеспечит выполнение настраиваемого подключаемого модуля после подключаемых модулей OOB.  
 
### <a name="end-user-experience"></a>Взаимодействие с конечным пользователем
1.  При создании записи времени на странице быстрого создания, введите сведения о проекте и задаче проекта, а затем выберите пользователя в поле **Резервируемый ресурс**, для которого необходимо записать записи времени. 
2.  По умолчанию это поле принимает значение вошедшего в систему пользователя, однако, учитывая, что пользователь переопределил это поле, теперь создается запись времени для выбранного **Резервируемого ресурса**.
3.  Когда вы отправляете записи времени, которые вы создали для этих записей, записи будут поставлены в очередь для утверждающего в проекте, как и ожидалось. 
4.  Когда вы отзываете записи времени, созданные для другого пользователя, записи времени будут возвращены в состояние **Черновика** с полем **Резервируемый ресурс**, установленным на другой пользователь. 
5.  При необходимости можно переключиться к настраиваемому представлению для фильтрации записей времени, созданных для другого пользователя. 
 
### <a name="limitations"></a>Ограничения
Функции **Копирование** и **Импорт** работают только в контексте пользователя, вошедшего в систему. Это означает, что невозможно скопировать или импортировать записи времени, созданные для пользователя, вошедшего в систему как резервируемый ресурс.

Записи времени, не относящиеся к проекту, будут перенаправлены на утверждение руководителю резервируемого ресурса только если будет выполнен шаг 4 в разделе **Сведения о настройке**. В противном случае не связанные с проектом записи времени для другого пользователя будут неправильно перенаправлены руководителю вошедшего в систему пользователя. 

### <a name="other-changes"></a>Другие изменения 
Функция **Резервирования и задачи** удалена. 

## <a name="multidimensional-pricing"></a>Многомерное ценообразование
Для обеспечения максимальной гибкости и удовлетворения различных бизнес-требований, Project Service Automation версии 3 поддерживает дискретное применение наборов измерения цены и ставок по счету. Значения измерений могут быть установлены по умолчанию, а затем распространяться на весь процесс определения расходов и цены от профилирования ресурсов до записи времени в фактические данные проекта. Конфигурация, модификация или расширение для конкретного клиента использует стандартную инфраструктуру настраиваемости.

Project Service Automation поставляется с установленным по умолчанию набором измерений цен, ролей и единиц ресурсов, а также позволяет устанавливать цены и расходы для каждой комбинации роли и подразделения.

Для клиентов Project Service Automation, которые будут продолжать использовать эти готовые поля в качестве измерений цен в версии 3, заметных изменений не будет. Вы можете продолжить использовать Project Service Automation, как обычно. Если, однако, вам нужно рассчитывать цены или стоимость для ваших ресурсов с использованием других дополнительных атрибутов, версия 3 позволяет добавить в Project Service Automation ваши собственные настраиваемые измерения ценообразования. Расширение настраиваемых аналитик ценообразования представляет собой сложный процесс настройки. 

## <a name="quotes-and-contracts"></a>Предложения с расценками и контракты
В Project Service Automation версии 3 изменились аспекты настройки и управления предложениями с расценками и контрактами. Следующие разделы содержат более подробная информация.

### <a name="set-up-chargeability-options"></a>Настройка параметров возможности оплаты
В версиях 1 и 2 настройка возможности оплаты для ролей и категорий для конкретных предложений с расценками и контрактов выполнялась с использованием представления **Возможность оплаты**, которое находилось в верхней области переходов строки предложения с расценками или строками контрактов. Здесь также где можно настраивать цены для этих ролей и категорий расходов.

Начиная с версии 3, настройка возможности оплаты по ролям и категориям расходов осуществляется на уровне предложения с расценками или строки контракта. Настройка ценообразования является отдельной от настройки возможности оплаты. Можно найти **Оплачиваемые роли** и **Оплачиваемые категории** в качестве вкладки на страницах **Строка предложения с расценками** и **Строка контракта** без использования верхней области переходов.

![Оплачиваемые роли.](media/chargeable-12.png)
 
Настройка оплачиваемых ролей и оплачиваемых категорий также использует готовой редактируемый элемент управления сеткой. Для каждой роли и категории, поддерживаемые параметры для выставления счетов на этапе предложения с расценками и заключения контракта остаются неизменными по сравнению с предыдущими версиями как **Оплачиваемые** и **Не подлежащие оплате**. **Бесплатные** не поддерживается на этапе предложения с расценками и заключения контракта. **Бесплатные** поддерживаются только на этапе утверждения времени или расходов.  
 
### <a name="create-and-edit-custom-pricing-for-a-project-service-automation-quote-and-project-contract"></a>Создание и изменение настраиваемого ценообразования для предложения с расценками и контракта по проекту в Project Service Automation
В версиях 1 и 2 использование настраиваемого прайс-листа для конкретных предложений с расценками и контрактов осуществлялось с использованием **Изменить цены** на представлении **Возможность оплаты**. Представление **Возможность оплаты** было расположено в верхней области переходов строки предложения с расценками или строки контракта. Здесь также где можно настраивать параметры возможности оплаты для ролей и/или категорий расходов.

Начиная с версии 3 создание и использование настраиваемого прайс-листа по проекту для предложения с расценками Project Service Automation и контракта по проекту Project Service Automation отделено от настройки возможности оплаты. В предложении с расценками в Project Service Automation и в контрактах по проектам в Project Service Automation есть новая вкладка, которая называется **Прайс-листы проекта**. Эта вкладка содержит связанное представление всех прайс-листов проекта, которые прилагаются к предложению с расценками или контракту по проекту в Project Service Automation. Для создания настраиваемого прайс-листа из существующего прайс-листа, который уже связан с предложением или контрактом по проекту, щелкните **Создать настраиваемые ценообразования**. Это сделает копию всех связанных прайс-листов и их прикрепит к предложению с расценками или контракту. Теперь можно открывать прайс-лист и изменять цену категории роли или расходов, чтобы изменения в ценах были применимы только к этому предложению или контракту.
