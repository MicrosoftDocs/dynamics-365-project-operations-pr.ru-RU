# <a name="sample-data-installation-for-the-project-service-application"></a>Установка демонстрационных данных для приложения Project Service

[!include [banner](../includes/psa-now-project-operations.md)]

Чтобы помочь вам развить собственные демонстрационные среды, корпорация Microsoft предоставляет загружаемые пакеты демонстрационных данных, которые демонстрируют возможности ваших приложений. Существует два типа пакетов демонстрационных данных:
- справочные данные/данные настройки
- демонстрационные данные (справочные данные/данные настройки и данные о транзакциях, такие как заказы на работу и проекты)

Пакеты демонстрационных **справочных** данных доступны для загрузки в трех различных пакетах, поэтому вы можете установить данные только для Project Service или только для Field Service, а также установить демонстрационные данные для обоих приложений одновременно.

К пакетам демонстрационных справочных данных/данных настройки относится:

- [**V902PSMasterData** — только Project Service версии 3.x](https://go.microsoft.com/fwlink/?linkid=2026540&clcid=0x409)

- [**V902FSMasterData** — только Field Service версии 8.x](https://go.microsoft.com/fwlink/?linkid=2026536&clcid=0x409)

- [**V902FPSMasterData** — Field Service 8.x и Project Service 3.x](https://go.microsoft.com/fwlink/?linkid=2026041&clcid=0x409)

Последний пакет **демонстрационных** данных:

 - [**FPSDemoData** — Field Service 8.x и Project Service 3.x](https://aka.ms/fpsdemodatapackage)

   Инструкции по установке немного отличаются в разделе создания и настройки, но в остальном аналогичны инструкциям в предыдущей [**записи блога**](https://aka.ms/fpsdemodatablog). Этот пакет включает уменьшенный набор демонстрационных данных, и его установка занимает приблизительно 3 часа.

Эти пакеты демонстрационных данных доступны только на английском языке.

> [!IMPORTANT]
> **Демонстрационные данные удалить невозможно.** Вы должны устанавливать эти пакеты только в демонстрационных, оценочных, обучающих или тестовых системах. Также обратите внимание, что установка одного отдельного пакета с последующей установкой другого отдельного пакета не поддерживается. (Иначе говоря, нельзя установить **FSMasterData**, а затем **PSMasterData** или наоборот.) Если вы обнаружите, что требуются демонстрационные данные для обоих приложений в какой-либо момент в будущем, необходимо установить пакет **v902FPSMasterData**.

При установке любых пакетов демонстрационных данных, процесс установки выполняет следующие действия:

- Создание или задание параметров по умолчанию для использования Project Service, Field Service или обоих приложений (если применимо).

- Импорт демонстрационных данных для приложений, таких как резервируемые ресурсы, специальные роли для приложения, списки цен продажи и себестоимости, подразделения, записи процесса продаж и другие объекты для демонстрации основных возможностей.  

Пакет **демонстрационных данных** включает вышеуказанные и дополнительные данных о транзакциях, такие как заказы на работу и проекты.

Интересуетесь, какие возможности можно демонстрировать с демонстрационными данными? См. вымышленный сценарий Fabrikam Robotics в разделе [Технические примечания](#technical-notes).

При возникновении вопросов по установке этих пакетов демонстрационных данных [отправьте нами сообщение по электронной почте по адресу fpsdemodata@microsoft.com](mailto:fpsdemodata@microsoft.com).

## <a name="requirements"></a>Требования

Протокол установки предполагает следующее о данном целевом экземпляре (организации):

- Базовый язык — английский, базовая валюта — доллар США (USD, $).

- У организации нет данных Project Service или Field Service либо есть только базовые данные по умолчанию, которые идут с любой новой организацией.

- Правильная версия бизнес-приложения уже установлена:
       
    - **Для FPSDemoData или v902FPSMasterData:** в организации установлены приложения Field Service версии 8.x и Project Service версии 3.x.

    - **Для v902PSMasterData:** в организации установлено приложение Project Service версии 3.x.

    - **Для v902FSMasterData:** в организации установлено приложение Field Service версии 8.x.

> [!NOTE]
> Если необходимо установить демонстрационные данные поверх существующей пробной или демонстрационной среды Project Service и Field Service, уже содержащие данные (не рекомендуется), потребуется приостановить предварительные проверки безопасности, выполняемые установщиком. Дополнительные сведения см. в технических замечаниях ниже.

## <a name="prepare-for-installation"></a>Подготовка к установке

Требуется запустить программу установки на компьютере с последней версией Windows (предпочтительно Windows 10).

Необходимо запланировать для компьютера оставаться на связи с сетью и для установки **справочных данных/данных настройки** установить время выполнения **1 час**. (Обычно установка занимает около 30 минут для пакета **FPSMasterData**, содержащего демонстрационные данные для обоих приложений.) Установка **FPSDemoData** займет приблизительно **3 часа**.

На компьютере должна быть отключена функция заставки экрана. В противном случае учетные данные сеанса для установки могут быть потеряны при включении заставки (если не будет сохраняться сеанс активным).

## <a name="download-and-unpack"></a>Загрузка и распаковка

Установщик демонстрационных данных Project Service и Field Service распространяется как самораспаковывающийся исполнимый файл. Имена файлов могут отличаться в зависимости от пакета демонстрационных данных, но в остальном шаги совпадают независимо от устанавливаемого пакета.

После загрузки пакета запустите EXE-файл, затем примите условия для распаковки сжатого ZIP-файла. Затем требуется извлечь содержимое файла в папку на компьютере.

В зависимости от используемой операционной системы и параметров безопасности, можно выполнить следующие действия после распаковки ZIP-файла:

1. Найдите и щелкните правой кнопкой мыши файл **FPSDemoData.dll** в папке **v902FPSMasterData** / **PackageDeployer_FPSDemoData**.

2. Выберите **Разблокировать**.

3. Выберите **Применить**.

4. Нажмите **ОК**.

Пакету **FPSDemoData** требуется шесть пользователей, а пакету **FPSMasterData** — один. См. правильный пакет демонстрационных данных.

## <a name="create-or-configure-users---setupreference-data-packages"></a>Создание и настройка пользователей — пакеты справочных данных/данных настройки

Пакет **FPSMasterData** предназначен для установки с одним пользователем с именем Spencer Low с описанными здесь настройками. Для правильной установки пакета необходимо создать (либо временно переименовать) пользователей в среде в соответствии с конфигурацией входящих демонстрационных данных.

Чтобы создать или настроить пользователей, перейдите в раздел **Параметры** > **Безопасность** > **Пользователи**, и выполните следующее:

1. Задайте UserFullname="Spencer Low" с именем пользователя "spencerl" (**в нижнем регистре**) для ролей руководителя проекта и руководителя по методикам.

2. Выберите пользователя **Spencer Low** и щелкните **Управление ролями**. Найдите и выберите роль **Системный администратор**, а затем выберите **ОК** для предоставления полных прав администратора пользователю Spencer Low. Этот шаг необходим для обеспечения того, что демонстрационные записи создаются с правильным типом собственности пользователя и, поэтому, заполняют представления правильно.

3. Из загруженного пакета необходимо обновить файл сопоставления данных с адресами электронной почты контекста пользователя по умолчанию. Для этого откройте папку **PkgFolder**, затем найдите и откройте файл **ImportUserMapFile.xml** в Блокноте (или в Visual Studio или другом редакторе XML). Задайте в поле **DefaultUserToMapTo=** адрес электронной почты пользователя Spencer Low.

4. Если вы не используете пользователя Spencer Low с именем пользователя **spencerl**, необходимо обновить дополнительный файл. Откройте файл **DemoDataPreImportConfig.xml**, затем найдите тег **userstocreateandconfigure**. Обновите тег **\<login\>** с использованием имени пользователя вашего пользователя Spencer Low. Дополнительные сведения см. в разделе [Технические примечания](#technical-notes).

## <a name="create-or-configure-users---demo-data-package"></a>Создание и настройка пользователей — пакет демонстрационных данных

Пакету демонстрационных данных требуется шесть пользователей. Для правильной установки пакета выполните следующие действия:

 1. Создайте или временно переименуйте существующих пользователей для соответствия конфигурации входящих демонстрационных данных, перейдя в раздел **Параметры** > **Безопасность** > **Пользователи**.
 
    Эти роли необходимые только для демонстраций, основанных на личностях.  
    - User Fullname="David So" — техник Field Service   
    - User Fullname="Jamie Reding" — диспетчер Customer Service и Field Service   
    - User Fullname="Molly Clark" — менеджер по работе с клиентами   
    - User Fullname="Spencer Low" — руководитель по методикам и руководитель проекта  
    - User Fullname="Veronica Quek" — участник рабочей группы   
    - User Fullname="William Contoso"
  
2. Для импорта демонстрационных данных назначьте шести пользователям выше роль "Администратор" для правильного импорта примеров записей. 

3. Откройте **PkgFolder**, затем найдите и откройте **ImportUserMapFile.xml**. Обновите поля **Создать=** для добавления адресов электронной почты соответствующих пользователей в системе.

   > [!div class="mx-imgBorder"]
   > ![Снимок экрана UserMapFile.](media/sample-data-7.png)

4. Если полное имя пользователя "Spencer Low" имеет идентификатор пользователя, отличный от **spencerl**, необходимо обновить дополнительный файл. Откройте **DemoDataPreImportConfig.xml** и найдите тег **userstocreateandconfigure**. Обновите тег **\<login\>** для отображения loginId (с учетом регистра). 

5. Календарь первого пользователя (в теге **userstocreateandconfigure** tag) используется для автоматического заполнения рабочих часов для всех резервируемых ресурсов при импорте демонстрационных данных. Перейдите в раздел **Параметры** > **Безопасность** > **Пользователи**, найдите пользователя "Spencer Low" и откройте параметр "Рабочие часы". Измените существующие рабочие часы, выбрав параметр **Весь недельный цикл с начала до конца**. Убедитесь, что **для рабочих часов задан период с 8:00 до 17:00 (9 часов), с понедельника по пятницу, и что задан часовой пояс "Тихоокеанское время (США и Канада)"**. Это необходимо для обеспечения правильного отображения таблицы расписаний и проекта.

**Рекомендация.** Рекомендуется создать резервную копию вашей организации теперь, в случае если будет необходимо вернуться к отправной точке, если что-либо пойдет не так во время установки демонстрационных данных. Дополнительные сведения см. в разделе [Резервное копирование и восстановление экземпляров](/dynamics365/customer-engagement/admin/backup-restore-instances).

## <a name="run-the-package-deployer"></a>Запустите Package Deployer

1. Найдите и запустите файл **PackageDeployer.exe** в папке **v902FPSMasterData** ИЛИ **PackageDeployer_FPSDemoData**.

2. Примите условия соглашения.

3. В следующем окне:

   a. Выберите тип развертывания **Office 365**.

   b. Используйте имя пользователя и пароль пользователя системного администратора, настроенный в разделе "Создание или настройка пользователей" ("Spencer Low" с именем пользователя "spencerl").

   c. Убедитесь, что установлен флажок **Отображать список доступных организаций**.

      > [!div class="mx-imgBorder"]
      > ![Снимок экрана окна Package Deployer с установленным флажком "Отображать список доступных организаций"](media/sample-data-2.png)

4. Выберите организацию, в которую требуется установить демонстрационные данные.

5. Выберите **Далее**, пока не увидите диалог **Настройка демонстрационных данных**.

   > [!div class="mx-imgBorder"]
   > ![Снимок экрана окна состояния установщика демонстрационных данных.](media/sample-data-3.png)

6. Перед тем как продолжить, обратите внимание, что установка демонстрационных данных может занять до одного часа (обычно ~10 минут). Необходимо убедиться, что компьютер остается соединенным с сетью в течение процесса установки, и что сеанс остается активным.   

7. Когда будете готовы, нажмите кнопку **Далее**, чтобы начать процесс установки демонстрационных данных. После загрузки демонстрационных данных щелкните **Готово**.

После полной загрузки демонстрационных данных войдите в систему как пользователь Spencer Low и убедитесь в следующем:

- Если приложение Project Service установлено, перейдите в раздел **Project Service** > **Параметры** > **Прайс-листы**. Убедитесь, что ставки по счету и ставки стоимости существуют с соответствующей валютой для каждой страны/региона в наборе данных.

## <a name="technical-notes"></a>Технические примечания

См. ниже дополнительные технические сведения по установке этих данных.

### <a name="installing-sample-data-on-top-of-existing-data-not-recommended"></a>Установка демонстрационных данных поверх существующих данных (не рекомендуется)

Если необходимо установить демонстрационные данные поверх существующей пробной или демонстрационной среды Field Service или Project Service, уже содержащие данные, потребуется приостановить предварительные проверки безопасности, выполняемые установщиком.

Для этого откройте папку **PkgFolder**, чтобы найти и открыть файл **DemoDataPreImportConfig.xml** в Блокноте (или в другом редакторе XML).

Найдите следующее значение, а затем измените параметр с True на False:

```alias
<TerminateOnPreCheckFailure>true</TerminateOnPreCheckFailure>
```

Это изменение заставляет установщик обходить некоторые важные проверки безопасности, в том числе:

- Проверка, что имеется не более одной активной записи **Подразделение**, а затем переименование ее в **Fabrikam US**.

- Проверка, что имеется не более одной активной записи **Рабочий шаблон**.

- Проверка, что имеется не более одной активной записи **Параметр проекта**, а затем переименование ее в **Параметры**.

### <a name="configuration-components"></a>Компоненты конфигурации

Имеется несколько других компонентов конфигурации в этом файле конфигурации перед импортом. Для технических пользователей некоторые из них включают:

- **\<RequiredSolutions\>** указывает обязательные предварительные установки решений и номера их версий.

- **\<InstallSampleData\>** управляет тем, установлены ли готовые демонстрационные данные для приложений.

    - false — установка этих встроенных данных пропускается (что может быть удалено).

    - true — установка встроенных данных одновременно с установкой демонстрационных данных FS и PSA

- **\<PreImportDataCollection\>** указывает плоский файл сопоставления данных и связанные записи, которые следует импортировать перед установкой основных демонстрационных данных.

- **\<EntitiesToEnableScheduling\>** указывает, какие объекты необходимо включить для резервирования в Microsoft Dynamics Scheduling (также называется Universal Resource Scheduling).

- **\<UsersToCreateAndConfigure\>** указывает резервируемые ресурсы, которые будут созданы (если они еще не существуют) перед выполнением импорта демонстрационных данных. Обратите внимание, что резервируемые ресурсы демонстрационных данных исходной системы соответствуют записям резервируемого ресурса целевой системы по FullName и данным для входа каждого из ресурсов. Поэтому НЕВОЗМОЖНО изменить имена в этом файле предварительной конфигурации, если только не импортировать сначала демонстрационные данные в целевую систему с использованием этих имен, а затем переименовать резервируемые ресурсы на предпочитаемый вами набор имен вместе с записями разрешенных пользователей, а затем экспортировать данные снова для импорта в вашу окончательную целевую систему (с обновлением старых и новых записей **ImportUserMapFile.xml** соответственно).

- **\<PluginsToDisable\>** указывает очень специальные подключаемые модули товаров строки, которых следует отключить при импорте демонстрационных данных, а затем снова включить.

### <a name="change-the-work-hours-for-sample-resources"></a>Изменение рабочих часов для демонстрационных ресурсов

По умолчанию все резервируемые ресурсы имеют календарь с 24 рабочими часами.

Если требуется изменить рабочие часы для демонстрационных резервируемых ресурсов, перейдите в раздел **Universal Resource Scheduling** > **Планирование** > **Ресурсы**.

Выберите пользователя (например, Spencer Low) и измените рабочие часы пользователя Spencer на часы, которые требуется применить к нескольким пользователям. Перейдите к пункту **Universal Resource Scheduling** > **Настройки** > **Шаблоны рабочих часов** и измените запись **Рабочий шаблон по умолчанию**. В поле **Ресурс шаблона** выберите пользователя с расписанием работы, которое требуется применить к другим ресурсам. Выберите **Universal Resource Scheduling** > **Планирование** > **Ресурсы** > **Активные резервируемые ресурсы**. Выберите ресурсы, которые требуется изменить, затем выберите **Задать календарь**. В раскрывающемся списке **Рабочий шаблон**, выберите шаблон **Рабочие часы по умолчанию** или другой шаблон с ресурсом с правильным шаблоном. При переходе к доске расписаний вы должны видеть, что у ресурсов обновились рабочие часы.
