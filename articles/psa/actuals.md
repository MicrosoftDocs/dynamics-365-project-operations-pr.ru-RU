# <a name="actuals-overview"></a>Обзор фактических значений

[!include [banner](../includes/psa-now-project-operations.md)]

[!INCLUDE[cc-applies-to-psa-app-3.x](../includes/cc-applies-to-psa-app-3x.md)]

Фактические значения — это сумма работы, которая была выполнена по проекту. Фактические данные проекта можно отслеживать обратно к их исходным документам. Эти исходные документы включают времени, расходы и записи журнала, а также счета.

![Как фактические показатели проекта отслеживаются к исходным документам](media/basic-guide-18.png)

## <a name="submitting-a-time-entry"></a>Отправка записи времени

В PSA когда запись время отправлена для проекта, который сопоставляется со строкой контракта по времени и материалам, создаются две строки журнала. Одна строка для стоимости и другая строка для продаж, за которые не выставлены счета. Когда запись время отправлена для проекта, который сопоставляется со строкой контракта с фиксированной ценой, строка журнала создается только для стоимости. 

Логика для ввода цен по умолчанию находится в строке журнала. Все значения полей из записи времени копируются в строку журнала. Эти поля включают дату транзакции, строку контракта, с которой сопоставлен проект, и результат валюты в соответствующем прайс-листе. 

Поля, которые влияют на цены по умолчанию, например **Роль** и **Подразделение**, вызывают ввод соответствующей цены по умолчанию в строке журнала. Если вы добавили настраиваемое поле в запись времени и хотите, чтобы значение поля распространялось на фактические данные, создайте это поле в сущности фактических данных и используйте сопоставления полей для копирования поля из записи времени в фактические данные.

## <a name="submitting-an-expense-entry"></a>Отправка записи о расходах

В PSA когда запись расходов отправлена для проекта, который сопоставляется со строкой контракта по времени и материалам, создаются две строки журнала. Одна строка для стоимости и другая строка для продаж, за которые не выставлены счета. Когда запись расхода отправлена для проекта, который сопоставляется со строкой контракта с фиксированной ценой, строка журнала создается только для стоимости.

Логика для ввода цен по умолчанию для расходов основана на категории расходов, которая выбрана на странице **Запись расхода**. Дата транзакции, строка контракта, с которой сопоставлен проект, и валюта все используются для определения соответствующего прайс-листа. Однако для самой цены сумма, которую ввел пользователь, задается непосредственно в связанных строках журнала расходов для стоимости и продаж по умолчанию.

В текущей версии PSA основанная на категории запись цен по умолчанию за единицу в записях расходов недоступна.

## <a name="using-entry-journals-to-record-costs"></a>Использование журналов записей для записи стоимости

Журналы записей в ПСА позволяют записывать стоимость или доход в классах проводок по материалам, сборам, времени, расходам и налогам. Журнал имеется заголовок, строки и действие **Подтвердить**. Ниже приведены некоторые сценарии, в которых можно использовать журнал:

- Необходимо записывать фактические затраты на материалы и продажи в проекте.
- Необходимо переместить фактические данные транзакций из другой системы в PSA.
- Необходимо записывать затраты, возникшие в другой системе, такие как стоимость закупок или стоимость субподряда.

> [!IMPORTANT]
> Использовать журналы записей для создания фактических значений должны только пользователи, которые полностью понимают влияние учета фактических значений на проект. Это связано с тем, что приложение не проверяет тип строки журнала или связанные цены, вводимые в строку журнала. Из-за влияния, которое может оказать этот тип журнала, будьте осторожны при предоставлении доступа для создания журналов записей.
